# Zero-click delivery (анализ)

Этот заметочный файл используется контрактными тестами репозитория. Он фиксирует,
что при рабочем процессе единственный авторитетный источник статуса пакетов и
контрактов — файл `doc/backlog_registry.yaml`.

Производный weekly-view в `doc/tasklist.md` может использоваться для обзора очереди,
но не заменяет реестр при разрешении конфликтов статусов или формулировок DoD.

При старте автоматизации предпочитайте роутер `scripts/workflow.py` и текст
единого промпта из `scripts/workflow_strings.py`, чтобы не дублировать формулировки
в нескольких местах.
