# E5: Personalization & Long-term Learner

Closed: **2026-04-08**

Focus: persistent learner profile, migration-safe learner state, and stronger
mastery/spaced-repetition continuity across index evolution.

## Closure Summary

E5.0-E5.8 closed the learner-state migration contour after E4 quality gates:

- **E5.0:** versioned learner profile history in
  `personalized_learner_model_history_json`.
- **E5.1:** migration-safe `mastery_vector` rehydrate from profile history when
  current mastery becomes orphaned after index changes.
- **E5.2:** `/kb/learner/profile-history` diagnostics API for current
  `index_context`, `state_migration`, and versioned history.
- **E5.3:** learner migration rollups through `/metrics/learner` and
  `get_learner_profile_migration_metrics`.
- **E5.4:** learner migration SLO alert via `slo_max_learner_rehydrated_rate`.
- **E5.5:** ready quality-gate command
  `scripts/check_learner_migration_gate.py`.
- **E5.6:** smoke workflow in `scripts/smoke_learner_migration_gate.py`.
- **E5.7:** CI merge-gate
  `.github/workflows/learner-migration-smoke-gate.yml`.
- **E5.8:** eager learner DB lineage sync on index activation through
  `apply_index_activation_hooks` -> `run_learner_state_lineage_sync`.
