# E8: User Value Delivery

Closed: **2026-04-10**

Focus: CJM-driven learner value from activation through learning loop and
retention, with product-visible outcomes preferred over new infrastructure.

## Closure Summary

E8.1-E8.2 were closed as outcome slices and then folded into later learner-facing
workstreams. The durable rule from E8 is discipline, not an active backlog item:

- keep each user-value epoch bounded to a small number of outcomes;
- attach each outcome to a CJM moment and an existing user story when possible;
- prefer pytest/API/e2e evidence over new standalone benchmark scripts;
- archive closed playbooks outside the active daily backlog.

The downstream product value from this direction is represented by later closed
epochs and registry waves, including E9 continuity/trust, E10 learning-loop
completion, E11 guided start, E12 flashcards/SRS, and E13 home mode selection.
