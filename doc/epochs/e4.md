### Итерация 24: Learning Kernel

Зачем: превратить tutor из «умного режима ответа» в управляемый учебный цикл `goal -> plan -> session -> check -> recap -> next step`, не переоткрывая уже закрытый retention foundation из `19.3–19.5`.

**Критерий закрытия итерации 24:** пользователь получает не только tutor-ответ, но и явную учебную цель, plan-aware сессию, накопительный recap, mastery state и осмысленный `next best action`; orchestration становится видимой частью продукта, а не скрытым поведением prompt-слоя. **Важно:** scope ниже описывает именно `v2 delta` поверх уже закрытого foundation `19.3–19.5`, а не greenfield-реализацию заново.

**Must:**
- [ ] Learner Goal Contract: `learning_goal_id`, тема/подтема, `target_level`, `desired_outcome`, `time_budget`, `preferred_style`.
- [ ] Goal-aware Learning Plan v2: расширить существующий learning-plan path до goal-aware декомпозиции темы на шаги, prerequisites, suggested order, estimated effort, `next_session_recommendation`.
- [ ] Session Recap Artifact: поверх текущего tutor/session foundation сохранять `что изучили`, `ключевые понятия`, `ошибки/пробелы`, `что повторить`, `следующий шаг` в contract `markdown/json`.
- [ ] Topic Mastery State v2: расширить текущие mastery/review signals до явных состояний `unknown`, `introduced`, `in_progress`, `uncertain`, `strong`, `mastered` на уровне `topic / subtopic / concept`.
- [ ] Next Best Action Engine v2: поднять текущий tutor next-step из session-local logic до goal-aware выбора между `explain prerequisite`, `ask check question`, `give mini exercise`, `summarize`, `deepen topic`, `switch to practice`, `recommend review`.
- [ ] Goal-aware multi-turn memory: расширить существующую session/tutor memory активной целью, текущим модулем плана, уже объяснёнными concepts и последними ошибками.

**Should:**
- [ ] Quiz Checkpoint Lite v2: усилить уже существующий quiz/review foundation до 1–3 коротких question/checkpoints после объяснения как явной части tutor-cycle.
- [ ] Study Modes v1: явные режимы `Learn`, `Review`, `Practice`, `Explain`.
- [ ] Difficulty adaptation v2: перед следующим tutor-ходом передавать сводку по теме (`correct/incorrect/partial`, попытки, `last_error_type`), чтобы LLM менял глубину и сложность без отдельной ML-модели пользователя.
- [ ] Topic prioritization lite: сортировать suggested topics / next steps по сочетанию `low_mastery * recency_weight`, чтобы слабые и часто всплывающие темы поднимались выше.
- [ ] Explicit style adaptation: использовать `preferred_style` как явную настройку tutor policy/prompt (`concise`, `guided`, `Socratic`, `example-first`), а не пытаться магически детектировать стиль в раннем scope.
- [ ] Time-bounded plans: пресеты `15 минут`, `30 минут`, `1 час`, `неделя`.
- [ ] Session intensity presets: `gentle`, `normal`, `intensive`.

**Could:**
- [ ] Export recap to study notes в Markdown.
- [ ] Отдельный compact contract для `next session card` в UI.
- [ ] Локальный recap archive поверх уже существующих session/learning artifacts.

### Итерация 25: Tutor Evaluation & Trust

Зачем: после появления learning kernel следующий шаг — сделать tutor-поведение измеримым, контролируемым и устойчивым по качеству; без этого дальнейшее развитие останется “на глаз”.

**Критерий закрытия итерации 25:** tutor измеряется отдельным eval-контуром, trust-поля единообразно видны в API/UI, misconception-handling и anti-overhelp policy проходят regression gates, а развитие teaching behavior перестаёт быть неформальным prompt-polish. **Важно:** это `v2 delta` над existing tutor regression/gates из `19.5`, а не новый eval-контур “с нуля”.

**Must:**
- [ ] Tutor Eval Suite v2: сценарии `beginner ask`, `confused learner`, `partial understanding`, `misconception correction`, `exam prep`, `interview prep`.
- [ ] Grounded Teaching Checks: tutor различает `источник говорит`, `это интерпретация`, `поддержка слабая`, не уходит далеко от source-grounded path.
- [ ] Answer Confidence / Trust Signals v2: расширить существующий trust UX до `grounded`, `partially_grounded`, `inferred`, `weak_support`, `conflicting_evidence`, плюс `coverage`, `uncertainty hints`, `missing evidence note`.
- [ ] Regression Gates for Tutor Modes: `Learn`, `Review`, `Practice`, `Explain` не должны ломаться при изменении prompt/policy/orchestration.
- [ ] Misconception Handling: различать `unknown topic`, `wrong belief`, `mixed concepts`, `fragment remembered incorrectly` и выбирать `correction-first`, `prerequisite-first`, `gentle clarification`, `challenge question`.
- [ ] Longitudinal Eval v1: расширить existing tutor eval от одного ответа к серии из 3–5 шагов с метриками `progress`, `context retention`, `repeat error reduction`.
- [ ] Provenance-rich recaps: в recap явно добавлять, на какие документы/темы опирались, что покрыто надёжно и что стоит дочитать.
- [ ] Feedback loop v1: собирать явный thumbs up/down и reformulation detection в рамках одной сессии; warm-path должен использовать эти сигналы для измеримого улучшения на повторяющихся паттернах запросов.

**Should:**
- [ ] Cost-aware teaching orchestration: cheap path для recall/check questions, full path для deep synthesis/teaching.
- [ ] Judge panel v2: `factual`, `pedagogical`, `actionable`, `concise-vs-overlong`.
- [ ] Anti-overhelp policy: отдельный gate против “слишком готовых ответов” в обучающем режиме.
- [ ] Failure taxonomy для tutor/trust behavior: over-answering, weak grounding, bad next step, source conflict, low coverage.
- [ ] Conflict-aware trust behavior: если top evidence приходит из конфликтующих источников/версий, ответ явно маркирует расхождение вместо молчаливого выбора одного источника.
- [ ] Retrieval weight adjustment: ввести `quality_modifier` / аналогичный коэффициент для документов/тем по накопленным негативным сигналам; использовать как мягкий множитель финального retrieval score, а не как hard blacklist.
- [ ] Tutor behavior adaptation по feedback: если reformulation rate по теме устойчиво высок, tutor переключается на более подробный / example-first / lower-abstraction mode.
- [ ] Knowledge gap report: запросы с устойчиво низким quality/trust outcome и слабым retrieval складывать в отдельный локальный report для пополнения корпуса, а не пытаться “додумать” ответ.
- [ ] Cold vs warm eval: сравнивать quality/trust scores в режимах без накопленного feedback и с накопленным feedback на одинаковых query patterns.

**Could:**
- [ ] A/B-compatible hooks для сравнения tutor strategies (`Socratic vs direct`, `recall-first vs explain-first`).
- [ ] Local developer dashboard `top regressions / top improvements`.
- [ ] LLM summary по quality/trust drift не чаще 1 раза в сутки.

### Итерация 26: Structured Knowledge & Personal Tutor

Зачем: только после закрытия learning kernel и tutor eval/trust имеет смысл подключать graph и personalization туда, где они реально усиливают обучение, а не усложняют систему ради архитектурной красоты.

**Критерий закрытия итерации 26:** система умеет показать место темы в карте знаний, prerequisites, персональный learning path и review queue; personalization прозрачно управляется через learner profile и не превращается в скрытую “магию”. **Важно:** use-case слой ниже допустим только поверх уже закрытого graph foundation и не подменяет scope `17 Core`.

**Must:**
- [ ] Prerequisite Graph v1: использовать production-grade graph foundation для `concept -> prerequisite concept`, `module -> prerequisite module`, `topic overlap / related topics` в use-cases `что знать до X`, `почему застрял`, `с чего начать`.
- [ ] Topic Map / Concept Map: поверх production-grade graph foundation показывать ключевые concepts, связи, hierarchy, nearby concepts, missing foundations.
- [ ] Learner Profile v1: `preferred_depth`, `pace`, `current_level by topic`, `weak concepts`, `recent mistakes`, `active goals` как прозрачный и управляемый contract.
- [ ] Personalized Review Loop v2: усилить existing retention foundation до персонального выбора `что повторить сегодня`, `что почти забыто`, `где были ошибки`, `какие 3 concepts нужно закрепить`.
- [ ] Study Dashboard Contract: `active goals`, `mastered topics`, `weak areas`, `recent sessions`, `suggested next session`, `review_queue`, плюс per-topic `coverage`, `mastery`, `last_active`.
- [ ] Retention Loop surfaces v2: `micro-loop`, `daily-loop`, `context-loop` как ненавязчивые сценарии recall/review поверх существующего retention foundation.

**Should:**
- [ ] Structure-aware ingestion v2: refinement поверх закрытого `17 Core`, а не замена его foundation; `headings hierarchy`, `theorem/example/definition blocks`, `exercise blocks`, `code explanation blocks`, `section roles`.
- [ ] Research Session Mode: `hypothesis`, `compare sources`, `unresolved questions`, `evidence map`, `reading queue`.
- [ ] Concept-level provenance: какой concept из какого раздела выведен, где strong/weak support, какие связи inferred.
- [ ] Personalized quiz generation по weak topics и recent mistakes.
- [ ] Graph-assisted `next topic` recommendation через prerequisite/related nodes.
- [ ] Flashcards contract v1: после успешного explanation/quiz генерировать карточки `{front, back, topic_id, difficulty}` и сохранять как learning artifact, не превращая систему в LMS.
- [ ] Mind map / concept map payload v1: JSON-структура `{nodes, edges}` с ограничением глубины/размера (например, 20–30 узлов), ранжированная по важности связей.

**Could:**
- [ ] Flashcards export / packaging для внешних repeat-tools без превращения системы в LMS.
- [ ] Topic/prerequisite map UI поверх production-grade graph backend.
- [ ] Light gamification overlay поверх уже существующего XP/gamification слоя: streaks, мягкие badges, `повторено сегодня`.
- [ ] Export learning artifacts / review summaries в Markdown.

### Явно не в scope итераций 24–26

- [ ] Multi-user / collaboration.
- [ ] Cross-device sync / WebDAV / локальная сеть как core-path.
- [ ] Full UKG platform как отдельный graph-owner.
- [ ] Heavy multi-agent orchestration.
- [ ] Миграция на `Neo4j` / `NebulaGraph` без срабатывания growth triggers.
- [ ] Новая SRE-модель “с нуля”: использовать и усиливать existing retention foundation из `19.3–19.5`.
- [ ] Геймификация как основной механизм удержания; она допустима только как optional overlay.

- [ ] **26** закрывает structured/personal tutor layer: prerequisites, topic map, learner profile, personalized review.
- [ ] Ни одна из итераций **24–26** не дублирует незакрытый scope `17 Core / 18 Core / 19 tail`.
