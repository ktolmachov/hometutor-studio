# 🎬 Killer Demo: "От вопроса до mastery за 3 минуты"

> **Цель:** Доказать, что hometutor — не очередной RAG-чат, а **полный учебный цикл**  
> **Формат:** Live demo + storytelling  
> **Время:** 3 минуты  
> **Аудитория:** Академическая комиссия / Инвесторы / Технические эксперты

---

## 🎯 Главная идея

**Не показывать функции. Показать трансформацию.**

```
❓ Студент с вопросом
        ↓
📚 Получает проверяемый ответ
        ↓
🎓 Понимает через диалог
        ↓
✅ Проверяет знание квизом
        ↓
🗂️ Запоминает через SRS
        ↓
📅 Получает персональный план
        ↓
🚀 Готов к следующему шагу
```

**Ключевой месседж:**  
> "Я показываю не генерацию текста, а **замкнутый контур обучения**: система не только отвечает, но и помогает понять, проверить, запомнить и спланировать повторение."

---

## 🎭 Драматургия: три акта за три минуты

### Акт I: Проблема (0:00-1:20) — "Обычный RAG недостаточен"

**Завязка:** Студент загружает материалы и задаёт сложный вопрос  
**Конфликт:** Ответ есть, но как убедиться, что он правильный? Как превратить его в знание?  
**Кульминация:** Показываем источники и confidence — ответ **проверяемый**

### Акт II: Решение (1:20-2:25) — "Учебный цикл в действии"

**Поворот:** Нажимаем "Учить эту тему" — начинается **настоящая магия**  
**Развитие:** Tutor → Quiz → Flashcard — три шага от понимания к запоминанию  
**Эмоция:** "Вот где assistant становится учителем"

### Акт III: Результат (2:25-3:00) — "Персональный план"

**Кульминация:** Mastery dashboard показывает прогресс  
**Развязка:** Adaptive plan предлагает следующий шаг  
**Финал:** "За 3 минуты мы прошли путь от вопроса до персонального плана обучения"

## 🎬 Покадровый сценарий

### 🎞️ Кадр 1: Открытие (0:00-0:20)

**Экран:** Streamlit UI, главная страница, папка `data/` с материалами

**Действие:** Показываем локальную папку с учебными материалами

**Реплика:**
> "Начнём с обычной ситуации: у студента есть учебные материалы — PDF, markdown, заметки. Не специально размеченная база знаний, а **его собственные файлы**. Система уже проиндексировала их локально — никакого облака, всё на машине."

**Визуальный акцент:**
- 📁 Папка `data/` с 3-5 файлами
- ✅ Статус индекса: "Ready · 42 documents · 156 chunks"
- 🔒 Иконка "Local-first" в углу

**Что доказывает:** Работает с реальными материалами пользователя, не требует специальной подготовки

---

### 🎞️ Кадр 2: Сложный вопрос (0:20-0:55)

**Экран:** Поле ввода вопроса

**Действие:** Вводим вопрос, требующий синтеза из нескольких источников

**Вопрос (на экране):**
> "Объясни, почему RAG-ответ сам по себе недостаточен для обучения, и как связка tutor, quiz, flashcards и mastery plan закрывает этот разрыв. Укажи источники."

**Реплика:**
> "Вопрос специально **не фактологический**. Он требует собрать ответ из нескольких мест, синтезировать идею и не потерять источники. Именно такие вопросы задают студенты, когда учатся."

**Визуальный акцент:**
- ⏳ Анимация обработки: "Classify → Rewrite → Retrieve → Rerank → Generate"
- 🔍 Показываем pipeline в реальном времени (если есть debug mode)

**Что доказывает:** Система работает с концептуальными вопросами, не только с "Что такое X?"

---

### 🎞️ Кадр 3: Проверяемый ответ (0:55-1:20)

**Экран:** Ответ с источниками, confidence chip, expandable citations

**Действие:** Показываем ответ и раскрываем источники

**Реплика:**
> "Вот ответ. Но **главное не текст, а то, на чём он стоит**. Смотрите: три источника, каждый кликабельный. Retrieval confidence показывает, насколько сильно найденные фрагменты покрывают вопрос. Для учебного инструмента это критично — студент должен учиться по материалу, а не по галлюцинации модели."

**Визуальный акцент:**
- 📊 Confidence chip: "High confidence · 3 sources"
- 📎 Три expandable источника с подсветкой релевантных фрагментов
- 🔗 Кликабельные ссылки на файлы

**Действие на экране:**
1. Кликаем на первый источник → раскрывается фрагмент
2. Наводим на confidence chip → показывается tooltip с объяснением

**Что доказывает:** Ответ **проверяемый**, не магический. Это не ChatGPT, где источник — "trust me bro"

---

### 🎞️ Кадр 4: Поворотный момент (1:20-1:40)

**Экран:** CTA-кнопка "🎓 Учить эту тему 5 минут"

**Реплика:**

**Визуальный акцент:**
- 🎯 Кнопка пульсирует или подсвечивается
- ⚡ Transition animation: "Q&A Mode → Learning Mode"

**Что доказывает:** Это не конец сценария, а **начало учебного цикла**

## 🛡️ Риски и страховки

### Риск 1: LLM отвечает нестабильно

**Страховка:**
- Заранее протестировать вопрос 5 раз
- Иметь fallback-вопрос, если первый не сработал
- Использовать deterministic seed для воспроизводимости (если модель поддерживает)

### Риск 2: Интернет/API недоступны

**Страховка:**
- Показать режим с локальной моделью (Ollama)
- Иметь записанное видео демо как backup
- Подготовить скриншоты всех ключевых экранов

### Риск 3: UI завис или медленно отвечает

**Страховка:**
- Перейти на скриншоты и проговорить сценарий
- Объяснить: "В production это быстрее, сейчас показываю логику"
- Иметь pre-recorded GIF-анимации ключевых моментов

### Риск 4: Вопросы из зала прерывают демо

**Страховка:**
- В начале сказать: "Прошу вопросы после демо, чтобы не потерять нить"
- Если вопрос критичный — ответить кратко и вернуться к сценарию
- Иметь таймер на экране: "Demo time: 2:15 / 3:00"

- Проект, который **решает реальную проблему фрагментации обучения**

**Рассказывайте историю, а не показывайте функции.**
