# Скрипт выступления на академической защите — home-rag_v2

> **Формат:** три минуты, академическая комиссия, цель — показать постановку задачи, архитектурное решение, результаты и процесс разработки.
>
> **Подсказка выступающему:** скрипт на одну страницу. Ориентируйся на таймкоды, говори своими словами, не зачитывай.

---

## 0:00 – 0:30 · Постановка задачи

> «Проблема самостоятельного обучения: студент работает с 30 PDF-лекциями, но вынужден использовать **четыре разных инструмента**. ChatGPT для вопросов — но он не помнит, что вы уже освоили. Anki для повторений — но он не отвечает по вашим материалам. Obsidian для заметок — но без квизов и плана. NotebookLM знает ваши файлы — но без интервального повторения.
>
> **Задача проекта:** объединить весь учебный цикл в один локальный инструмент. Не просто чат с документами, а полный цикл: ответ → объяснение → проверка → повторение → план на завтра. Всё локально, на вашей машине, с вашими материалами.»

🎯 **Ключевая фраза:** *«Четыре инструмента → один. Учебный цикл, а не просто поиск.»*

---

## 0:30 – 1:15 · Архитектурное решение

> «Архитектура построена на трёх принципах: **local-first**, **полный учебный цикл** и **измеримый прогресс**.
>
> **Стек технологий:**
> - **FastAPI** (порт 8000) — HTTP API для всех операций
> - **Streamlit UI** (порт 8501) — основной интерфейс пользователя
> - **Chroma + llama-index** — векторный поиск по документам
> - **SQLite** — хранение учебного состояния (`user_state.db`)
> - **Ollama или OpenAI-совместимые LLM** — переключение через `.env` без изменения кода
>
> **Слои архитектуры:**
> 1. Интерфейсный: Streamlit UI, CLI, Telegram-бот — все работают на общем state
> 2. API: FastAPI с роутерами по функциям (query, tutor, flashcards, dashboard)
> 3. Сервисный: `query_service`, `tutor_orchestrator`, `learner_model_service`, `adaptive_plan`
> 4. Persistence: SQLite для учебного состояния + Chroma для векторного индекса
>
> **Ключевое решение:** `app/provider.py` — единая точка подключения LLM. Меняешь одну строку в `.env` — переключаешься между локальным Ollama (0 ₽, приватность) и облачным GPT ($1–5/мес, качество). Код не трогаешь.»

🎯 **Ключевая фраза:** *«Local-first — это не ограничение, это фича. Твои материалы, твоя машина, твой прогресс.»*

---

## 1:15 – 2:15 · Демонстрация ключевых функций

> «Три функции, которые отличают home-rag от аналогов:
>
> **1. Learning Cycle (учебный цикл)**
> Студент бросает папку с лекциями в `data/`, запускает `ingest.py` — получает поисковую систему по своим материалам. Задаёт вопрос — получает ответ с **confidence 87%** и **тремя кликабельными фрагментами** из его собственных лекций. Но главное — кнопка «Учить эту тему 5 минут». Один клик — и ответ превращается в **сократический диалог**: объяснение → мини-квиз → флэшкарты → план на завтра. Контекст сохраняется, студент не теряет нить.
>
> **2. Concept Graduation (завершение темы)**
> Проблема всех SRS-систем: карточка преследует вас вечно. Anki не говорит «ты освоил тему». home-rag говорит. Если концепт стабильно удержан на transfer-уровне **больше 7 дней** — он получает статус `graduated` и больше не возвращается в ежедневный план. Это **единственная механика на рынке**, где продукт даёт ощущение финиша темы.
>
> **3. Course Workspace (курс под ключ)**
> Папка `data/ML-Course/` с 25 лекциями — один клик «Активировать как курс» — и за минуту у вас synthesis, learning plan, флэшкарты и отдельная панель прогресса для этого курса. Всё локально. Отключите интернет — продукт продолжит работать. State живёт в `user_state.db` на вашем ноутбуке.»

🎯 **Ключевая фраза:** *«Не просто ответы. Полный цикл: понял → проверил → повторил → освоил.»*

## 2:15 – 2:45 · Процесс разработки

> «Проект достиг зрелости: **14+ закрытых итераций**, стабильная архитектура, развитая документация.
>
> **Командный конвейер:** Product Owner → Analyst → Architect → Designer → Developer → Tester. Каждая роль работает через AI-агента с заточенным промптом (Cursor AI, Claude Code).
>
> **Ключевой инструмент:** Smart Router — скрипт `workflow.py`. Он читает состояние backlog, определяет где находится проект (6 состояний) и выдаёт готовую команду следующего шага. Режим **авто-цикл `--loop`**: одна команда в терминале ведёт пакет от `proposed` до `closed`, автоматически делая всё, что не требует решения человека. Ручных шагов остаётся два: принять контракт и выполнить задачу в IDE.
>
> **Результат:** процесс формализован, воспроизводим, автоматизирован. Каждый пакет проходит через одинаковый конвейер с одинаковыми гарантиями качества.»

🎯 **Ключевая фраза:** *«Не хаос, а конвейер. Не ручная работа, а автоматизация с AI-агентами.»*

## Страховочные ответы на вопросы комиссии

| Вопрос | Короткий ответ | Артефакт |
|---|---|---|
| «Как гарантируете качество ответов?» | Каждый тезис — с кликабельным фрагментом из документов; guardrails проверяют PII, prompt injection, пустые ответы; async judge оценивает каждый ответ. | [doc/user_scenarios.md](doc/user_scenarios.md) · [doc/architecture.md](doc/architecture.md) |
| «Почему не облачное SaaS?» | Учебные материалы приватны. Local-first — это фича, не баг. Облачный режим тоже поддерживается (OpenAI-совместимые), но state — ваш. | [doc/vision.md](doc/vision.md) |
| «Сколько стоит для пользователя?» | 0 ₽ с локальным Ollama. $1–5/мес с облачным LLM при ежедневном использовании. Переключение через `.env` без изменения кода. | [doc/user_guide_details.md](doc/user_guide_details.md) |
| «Как масштабируется на большие объёмы?» | Two-stage retrieval: doc→chunk. Partial reindex для мутаций. Тестировалось на 12k файлов. Chroma-индекс обновляется инкрементально. | [doc/architecture.md](doc/architecture.md) |
| «Какие метрики качества?» | Встроенный Eval Loop: hit-rate, MRR, latency p95, cost, judge-score; SLO-гейты. Каждый ответ оценивается async judge. | [doc/eval_experimenter_runbook.md](doc/eval_experimenter_runbook.md) |
| «Как организован процесс разработки?» | Командный конвейер: PO → Analyst → Architect → Designer → Developer → Tester. Smart Router автоматизирует переходы. 14+ закрытых итераций. | [doc/team_workflow/process.md](doc/team_workflow/process.md) |
| «Telegram и CLI — зачем?» | Все каналы работают на общем `user_state.db`; начал ответ в Telegram — продолжил в UI. Единый state, разные интерфейсы. | [doc/user_scenarios.md](doc/user_scenarios.md) |

## Технические страховки (если что-то пойдёт не так)

1. **Интернет отвалился во время демо** → «И это нормально. Локальный Ollama продолжит отвечать.» Переключись на локальный режим в `.env`: `OPENAI_BASE_URL=http://localhost:11434/v1`.
2. **Streamlit не стартует** → открой [doc/quickstart_demo.md](doc/quickstart_demo.md) с автоматически снятыми скриншотами — продукт уже задокументирован в снятом виде.
3. **Вопрос про конкретный сценарий** → все 14 сценариев задокументированы в [doc/user_scenarios.md](doc/user_scenarios.md) с e2e-тестами и скриншотами.
4. **Вопрос про архитектурные решения** → ADR (Architecture Decision Records) в [doc/adr.md](doc/adr.md) — каждое решение обосновано и задокументировано.
