## ❓ 5 главных вопросов и ответы

### 1. Почему это не просто "ChatGPT с загрузкой файлов"?

**Ответ:** ChatGPT даёт ответы, но не закрывает учебный цикл: нет автоматических квизов по ответу, нет SRS-карточек, нет mastery tracking, нет адаптивного плана повторения. Плюс ChatGPT не хранит учебный state локально.

---

### 2. Почему не NotebookLM?

**Ответ:** NotebookLM силён в document Q&A, но не закрывает учебный цикл: нет quiz generation, нет flashcards, нет SRS, нет mastery tracking, нет адаптивного плана. Плюс NotebookLM — облачный сервис, а здесь local-first с возможностью полного offline.

---

### 3. Где доказательства качества?

**Ответ:** Есть eval-инфраструктура и методика: deterministic checks, LLM-as-Judge, human feedback, regression gate. Численные метрики показываются по конкретному eval-run. Следующий шаг — зафиксировать eval dataset и baseline JSON.

---

### 4. Это production-ready?

**Ответ:** Production-oriented для локального one-user сценария. Проект закрывает ключевые инженерные риски RAG: гибридный поиск, self-correction, двухуровневая индексация, трёхслойная оценка, архитектурные инварианты. Для internet-scale/multi-tenant production потребуются отдельные SLO, auth, tenancy и ops-инфраструктура.

---

### 5. Для кого этот проект?

**Ответ:** Основная целевая аудитория — студенты и специалисты с техническим бэкграундом, которые хотят учиться эффективно по своим материалам. Также подходит для корпоративных пользователей с конфиденциальными данными (при локальных LLM). Не подходит для нетехнических пользователей без помощи (пока нет desktop installer) и преподавателей (пока нет групповых функций).
