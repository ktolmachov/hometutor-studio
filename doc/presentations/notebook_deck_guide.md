## 0. TL;DR — новый разбор за 5 шагов

1. **Собери факты по коду** (не по памяти прошлых разборов) — см.
   [`evolutionary_analysis_guide.md`](evolutionary_analysis_guide.md) §2 за промпт-шаблоном.
2. **Скопируй** `notebook_template.html` → `evolutionary_analyses/NN_slug.html`.
3. **Замени плейсхолдеры** в `<header>` и восьми `<section>` — палитру `:root` и
   классы компонентов не трогать (раздел 2 фиксирует их окончательно).
4. **Проверь по чек-листу** (раздел 8).
5. **Добавь строку** в `evolutionary_analyses/README.md` и, если разбор тянет
   реализацию, detail-plan `.md` в `doc/next/` (граница — `evolutionary_analysis_guide.md` §4).

## 1. Почему это отдельный формат, а не вариация слайд-дека

`html_deck_guide.md` описывает раскадровку 16:9 для защиты/питча: тёмная сцена,
индиго-циан градиент, один слайд на экран, JS-навигация стрелками. Notebook-стиль
решает другую задачу — **стратегический текст с разделами**, который:

- читается сверху вниз в браузере или печатается как документ (не листается);
- держит на странице таблицы, длинные списки вердиктов, диаграммы среднего
  размера — то, что не влезает в 16:9-карточку;
- визуально ближе к эссе/тетради, чем к презентации: серия для заголовков,
  кремовая бумага, а не тёмная сцена.

Общий принцип у обоих форматов один — «факты из источника → design-токены →
компоненты → самодостаточность» (см. `html_deck_guide.md` §0) — но CSS, палитра и
JS у них разные и **не смешиваются** в одном файле.

## 2. Design-токены — зафиксированы, не редактируются

Это ядро визитной карточки. Значения ниже финальны с 2026-07-12; меняются только
через явное решение владельца, задокументированное здесь же (раздел 9, «История версий»).

```css
:root {
  /* локоно: тёплая бумага, а не белый лист */
  --paper: #f7f3ea;        --paper-2: #fffdf7;
  --ink: #17211b;          --muted: #5c6a61;
  /* акцент: терракота — единственный «яркий» цвет страницы */
  --terra: #b95631;        --terra-deep: #7d351f;
  /* второй акцент: лесной зелёный — вердикт «оставить», номера разделов на глубоких картах */
  --forest: #243b2c;
  /* третий акцент: старое золото — номера ступеней, P2-тег, синтезы противоречий */
  --gold: #c98a3d;
  --line: rgba(23, 33, 27, 0.14);
  --card: rgba(255, 255, 255, 0.72);
  --code-bg: rgba(36, 59, 44, 0.07);
  /* семантика вердиктов — используется ТОЛЬКО в .v-keep/.v-fix/.v-no, не как акцент */
  --verdict-keep: #1e6e46;  --verdict-fix: #a05010;  --verdict-no: #8c2f39;
  --shadow: 0 14px 40px rgba(49, 29, 15, 0.08);
}
```

**Тёмная тема — не инверсия, а отдельно подобранная палитра** (тот же тёплый
характер на тёмном грунте): `--paper: #141a16`, `--ink: #ece5d4`,
`--terra: #d97a52` и т.д. — полный блок в `notebook_template.html`. Обязательны
**три** копии токенов: `@media (prefers-color-scheme: dark)`,
`:root[data-theme="light"]`, `:root[data-theme="dark"]` — вьюер Claude Artifacts
переключает тему атрибутом `data-theme`, который обязан побеждать медиа-запрос
в обе стороны (см. `artifact-design` skill).

**Опциональное расширение — статус-токены** (`--done`/`--silent`/`--hole`),
для сводных таблиц вида «сделано / работает-но-не-видно / дыра» (см. разбор №10).
Не обязательны для всех разборов — добавлять только если в тексте реально есть
такая таблица.

**Типографика — тоже часть локона, не варьировать:**
- Заголовки — `Georgia, "Times New Roman", serif` (единственное место с засечками
  на странице — это и делает разборы узнаваемыми на скриншоте).
- Текст — системный sans (`"Segoe UI", -apple-system, …`) — самодостаточность:
  никаких `@import` шрифтов с Google Fonts.
- `strong` красится в `--terra-deep`, не жирным на чёрном — акцент через цвет, не только вес.

## 3. Анатомия файла

```
<title>          ─ "Разбор №NN — Заголовок-тезис"
<style>          ─ :root (light/dark×3) + компоненты, полностью из notebook_template.html
<div class="wrap">
  <header>       ─ .kicker, h1, .lede, .meta
  <section>×8    ─ 1 СУТЬ … 8 ПЛАН, каждая начинается с .sec-label (номер + имя)
  <div class="footer">
</div>
```

Никакого `<html>/<head>/<body>` — Claude Artifacts оборачивает файл сам (см.
`artifact-design`). Для локальной копии в `evolutionary_analyses/` оборачивать
вручную в `<!doctype html><html lang="ru"><head>…</head><body>…</body></html>`
(так сделаны все текущие файлы серии — открываются напрямую в браузере).

## 4. Восемь секций — фиксированная структура, не сокращать без причины

Секции 1:1 соответствуют пунктам промпта из
[`evolutionary_analysis_guide.md`](evolutionary_analysis_guide.md) §2:

| № | `id` | Название | Обязателен? |
|---|---|---|---|
| 1 | `essence` | Суть | да — без него разбор превращается в аудит фич |
| 2 | `reality` | Реальность | да — инвентаризация с `файл:строка` |
| 3 | `tensions` | Противоречия | да, если в теме есть реальное напряжение сторон |
| 4 | `pain` | Боль-якорь | да — минимум одна `.anchor-card` |
| 5 | `verdicts` | Вердикты | да — решение, не обзор |
| 6 | `value` | Ценность | да |
| 7 | `path` | Путь | да |
| 8 | `plan` | План | да — P0 ≤ два хода |

Разрешено добавлять доп. секции (напр. «Контекст» между header и §1, как в
разборе №10) — но восемь номерных не переставлять и не удалять без явной причины.

## 5. Библиотека компонентов (все — в `notebook_template.html`)

| Компонент | Класс | Когда |
|---|---|---|
| Карточка находки | `.card` | одна находка/факт в §2, §5 |
| Боль-якорь | `.card.anchor-card` | ровно один раз, в §4 — терракотовая полоса слева |
| Противоречие | `.card.tension` | пара полюсов + `.syn` для синтеза, §3 |
| Диаграмма | `.figure > svg + .figcap` | inline SVG, см. раздел 7 |
| Таблица со скроллом | `.table-scroll > table` | всегда, а не голый `<table>` — иначе едет мобильная раскладка |
| Статус-пилюля | `.st.st-done / .st-silent / .st-hole` | ячейка таблицы «что сделано / молчит / дыра» |
| Вердикт-тег | `.v.v-keep / .v-fix / .v-no` | первое слово в `.card` секции 5 |
| Лестница уровней | `ol.ladder > li > .num + текст` | §7, золотые ступени |
| P0/P1/P2-тег | `.tag.tag-p0 / .tag-p1 / .tag-p2` | план, §8 |
| Кикер/лид/мета | `.kicker`, `.lede`, `.meta` | только в `<header>` |

**Инлайн-семантика текста:** `<strong>` — терракота (акцент), `<em>` — приглушённый
(ремарка), `<code>` — путь/идентификатор. Не заводить новых цветовых классов текста —
трёх акцентов (`--terra`, `--forest`, `--gold`) достаточно на всю страницу.

## 6. Диаграммы — inline SVG, приёмы из разборов №9–10

- Кольцевая диаграмма состояния (разбор №10) — узлы-кружки на окружности,
  цвет обводки узла = статус (`var(--done)`/`var(--silent)`/`var(--hole)`),
  легенда трёх цветов внизу диаграммы. Хорошо для «карта петли/процесса со статусами».
  Layout best-effort — так же, как SVG-диаграммы в `html_deck_guide.md`, у SVG внутри
  этого формата не используется единый шаблон-генератор.
  Аналогичный layout можно повторно использовать как отправную точку для новой
  кольцевой диаграммы (структуру `<circle>`+`<text>` узлов копировать вручную).
- Все цвета SVG — `var(--paper-2)`, `var(--muted)` и т.д. напрямую (SVG видит
  CSS-переменные страницы в inline-режиме, в отличие от печати в
  `html_deck_guide.md`, где для печати их дублируют вручную — здесь печать не
  основной путь потребления, дублирование не обязательно).
- Обязателен `role="img"` + `aria-label` с текстовым описанием диаграммы.

## 7. Отношение к серии эволюционных разборов

Notebook-стиль — формат подачи для
[`evolutionary_analysis_guide.md`](evolutionary_analysis_guide.md); все файлы в
[`evolutionary_analyses/`](evolutionary_analyses/) используют его (начиная с №9;
№1–8 — более ранняя, уже не воспроизводимая версия палитры, см. раздел 9). Формат
не привязан жёстко к эволюционным разборам — годится для любого стратегического
текста-лонгрида, где нужны разделы, таблицы и вердикты, а не раскадровка.

## 9. История версий стиля

- **v1.0 (2026-07-12).** Стиль зафиксирован по итогам разбора №9 «Своя комната» и
  расширен в разборе №10 «Кольцо замкнулось» (статус-токены `--done/--silent/--hole`,
  `code`-стиль). Причина фиксации: аудит показал, что разборы №1–8 каждый раз
  собирали CSS заново в новой сессии агента и разъехались по палитре (зелёный
  `--accent:#14705F` на белой бумаге вместо тёплой кремовой с терракотой).
  Владелец решил: **старые файлы №1–8 не ретрофитить** (риск исказить готовый
  содержательный текст ради косметики) — стиль фиксируется на будущее, этот файл
  и `notebook_template.html` — единственный источник правды с этой даты.
- Более старая версия (№1–8) не документируется отдельным гайдом — исторический
  снимок, палитра восстанавливаема только чтением конкретных файлов.
