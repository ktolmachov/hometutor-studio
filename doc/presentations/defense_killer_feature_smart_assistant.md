---

## 🎯 Проблема: Когнитивная перегрузка выбора

### Типичная ситуация студента

```
Студент открывает учебное приложение:

❓ "Что мне делать сейчас?"
├─ Повторить карточки?
├─ Пройти квиз?
├─ Продолжить чат с тьютором?
├─ Задать новый вопрос?
└─ Посмотреть прогресс?

⏱️ 2-3 минуты на выбор → потеря мотивации → закрытие приложения
```

**Это называется "paradox of choice"** — чем больше опций, тем сложнее начать.

---

## ✨ Решение: Умный помощник с объяснением

### Что делает система

```
🧭 Умный следующий шаг

📌 Повторить
   
Почему сейчас: Локальная очередь SM-2: к повтору 12 карточек — 
интервал уже наступил; короткая сессия удержит факты и снижает 
риск забывания.

[Повторить] ← Primary action

Дополнительно: быстрый ответ, чат тьютора, quiz, flashcards 
и прогресс — всегда рядом с подсказкой.

[Вопрос] [Тьютор] [Quiz] [Прогресс] ← Secondary actions
```

**Ключевое отличие:** Система не просто предлагает действие, а **объясняет, почему именно сейчас**.

## 🧠 Как это работает (техническая сторона)

### Детерминированная логика приоритетов

```python
def build_smart_study_recommendation(...) -> SmartStudyRecommendation:
    """Детерминированный выбор primary + «почему сейчас» + вторичных действий (без LLM)."""
    
    # Приоритет 1: Карточки к повтору (SM-2)
    if flashcard_due_n > 0:
        return SmartStudyRecommendation(
            hint_kind="cards_due",
            primary_label_ru="Повторить",
            why_now_ru=f"Локальная очередь SM-2: к повтору {fc} карточек — "
                       "интервал уже наступил; короткая сессия удержит факты...",
            primary_nav="flashcards_review",
            secondaries=...
        )
    
    # Приоритет 2: Провал квиза
    if quiz_feedback_failed:
        return SmartStudyRecommendation(
            hint_kind="quiz_failed",
            primary_label_ru="Разобрать слабое место",
            why_now_ru="Сигнал последней мини-проверки: ответ не зачтён — "
                       "сейчас лучше разобрать слабое место с тьютором...",
            ...
        )
    
    # Приоритет 3: Адаптивный план
    if plan_primary_block:
        return SmartStudyRecommendation(
            hint_kind="adaptive_plan",
            primary_label_ru=f"Следовать шагу плана — {step}",
            ...
        )
    
    # Приоритет 4: Продолжить чат с тьютором
    if has_tutor_resume:
        return SmartStudyRecommendation(
            hint_kind="tutor_resume",
            primary_label_ru=f"Продолжить чат по теме «{topic}»",
            why_now_ru="Сейчас выгодно не терять контекст диалога...",
            ...
        )
    
    # Приоритет 5: Учить тему после ответа
    if has_last_answer_qa:
        return SmartStudyRecommendation(
            hint_kind="answer_ready",
            primary_label_ru="Учить тему",
            why_now_ru="Локально уже есть быстрый ответ по базе — "
                       "перенесите материал в учёбу темы...",
            ...
        )
    
    # Fallback: Безопасный старт
    return SmartStudyRecommendation(
        hint_kind="safe_default",
        primary_label_ru="Короткая учебная сессия с тьютором",
        why_now_ru="Нет сильных сигналов очередей — безопасный старт...",
        ...
    )
```

## 🎨 Визуальное оформление

### Композиция карточки

```
┌─────────────────────────────────────────────────────┐
│  🧭 Умный следующий шаг                             │ ← Заголовок
├─────────────────────────────────────────────────────┤
│  📌 [Primary Label]                                 │ ← Главное действие
│                                                      │
│  Почему сейчас: [Explanation]                       │ ← Объяснение
│                                                      │
│  [Primary Button]                                   │ ← CTA
│                                                      │
│  Дополнительно: [Secondary Actions...]             │ ← Альтернативы
└─────────────────────────────────────────────────────┘
```

### Цветовая схема

- **Заголовок:** Синий (#2196F3) — нейтральный, информационный
- **Primary button:** Зелёный (#4CAF50) — призыв к действию
- **Explanation:** Серый (#757575) — вспомогательный текст
- **Secondary buttons:** Светло-серый — доступны, но не отвлекают

## 🎓 Педагогическая основа

### Научные принципы

1. **Cognitive Load Theory** (Sweller, 1988)
   - Снижение extraneous cognitive load через guided choice
   - Студент фокусируется на обучении, а не на навигации

2. **Spaced Repetition** (Ebbinghaus, 1885)
   - Приоритет карточкам к повтору защищает от забывания
   - Интервалы SM-2 оптимизируют retention

3. **Retrieval Practice** (Roediger & Karpicke, 2006)
   - Провал квиза → немедленная recovery route
   - Активное воспроизведение сильнее пассивного чтения

4. **Mastery Learning** (Bloom, 1968)
   - Слабые концепты требуют повторения перед новым материалом
   - Адаптивный план строится на mastery score

## 💡 Почему это killer-фича

### 1. Решает реальную боль

**Проблема:** Студенты тратят 2-3 минуты на выбор, что делать дальше  
**Решение:** Система предлагает один лучший шаг с объяснением

### 2. Уникальна на рынке

**Конкуренты:** Либо нет рекомендаций (NotebookLM, ChatGPT), либо без объяснений (Duolingo)  
**Мы:** Единственные, кто объясняет "Почему сейчас"

### 3. Основана на науке

**Не магия:** Детерминированная логика на основе learning science  
**Прозрачность:** Студент понимает, почему система предлагает это действие

### 4. Local-first

**Приватность:** Вся логика работает локально, без облачных вызовов  
**Скорость:** Мгновенная рекомендация без задержек API

### 5. Accessibility-ready

**Инклюзивность:** Keyboard и screen-reader friendly  
**Universal Design:** Работает для всех пользователей
