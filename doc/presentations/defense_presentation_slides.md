## Архитектура: единое локальное состояние

**Streamlit, CLI и Telegram работают с одной SQLite/Chroma базой**

```
┌─────────────────────────────────────┐
│ Интерфейсный слой                   │
│ Streamlit UI · CLI · Telegram Bot   │
├─────────────────────────────────────┤
│ API слой — FastAPI :8000            │
│ /ask · /flashcards · /kb · /admin   │
├─────────────────────────────────────┤
│ Сервисный слой                      │
│ query_service · tutor · graph       │
├─────────────────────────────────────┤
│ Persistence слой                    │
│ SQLite · Chroma · provider.py       │
└─────────────────────────────────────┘
```

**Принципы:**
- Все входы используют одни сервисы
- LLM только через provider.py
- Приватность = архитектура

<!-- Слайд 8: RAG Pipeline -->

## RAG Pipeline: 5-ступенчатая обработка

**От вопроса студента до ответа с источниками**

```
┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐
│ CLASSIFY │ → │ REWRITE  │ → │ RETRIEVE │ → │ RERANK   │ → │ GENERATE │
│ Router   │   │ Enricher │   │ Hybrid   │   │ Cross-   │   │ LLM +    │
│ qa/over  │   │ улучшение│   │ BM25+Vec │   │ encoder  │   │ sources  │
└──────────┘   └──────────┘   └──────────┘   └──────────┘   └──────────┘
```

**Инновации:**

- Classify определяет тип запроса (qa/overview/synthesis)
- Rewrite улучшает формулировку для поиска
- Retrieve использует гибридный поиск (BM25 + Vector)
- Rerank точно оценивает релевантность
- Generate создает ответ строго по источникам

## Offline и приватность как архитектурное свойство

![](doc/screenshots/defense_pdf/pdf_slide_10.png)

**Ollama/OpenAI-compatible provider переключается конфигом**

### Local LLM vs Cloud LLM

| Критерий | Local (Ollama) | Cloud (OpenAI) |
|----------|----------------|----------------|
| Приватность | ✅ Данные локально | ⚠️ Передаются провайдеру |
| Стоимость | ✅ 0 ₽ API-cost | 💸 Зависит от модели |
| Offline | ✅ Полностью | ❌ Требует интернет |
| Качество | ⚙️ Зависит от модели | ✅ Высокое |

**Переключение:** `.env` без изменения кода

## Процесс разработки: от хаоса к управляемому циклу

**Backlog SSoT, role prompts, audit chain и воспроизводимые проверки**

### Было → Стало

| Метрика | Было | Стало |
|---------|------|-------|
| Команд вручную | 15+ | 1 |
| Переключений | 20+ | 0 (автопилот) |
| Время на пакет | 2-3 часа | 15-75 минут |
| Риск пропустить DoD | Высокий | Нулевой |

```
plan_next → accepted package
orchestration → worker prompts + DoD
audit_closed_packages → evidence replay
coverage_prompt → missing tests
```

**Результат:** Разработчик фокусируется на решениях, а не на процессе.
