# Гайд: полированный самодостаточный HTML-дек (переиспользуемый)

> **Что это.** Методика и шаблон для сборки презентаций в виде одного файла `.html`
> — без внешних зависимостей и без хрупких путей к картинкам. Открывается в любом
> браузере, листается с клавиатуры, печатается в PDF клавишей `F`. Подходит для
> **любой** презентации: защита, спринт-демо, отчёт, питч.
>
> **С чего начинать.** Копируй [`deck_template.html`](deck_template.html) — это
> пустой, готовый к заполнению каркас, который заодно служит галереей компонентов.
> Готовый пример «как выглядит наполненный дек» —
> [`defense_presentation_v5.html`](defense_presentation_v5.html).
>
> **Зачем именно самодостаточный HTML.** Картинки и matplotlib-пайплайны легко
> разъезжаются между репозиториями/папками → битые ссылки. Здесь вся графика —
> инлайновый SVG, всё оформление — внутренний CSS. Один файл = ничего не сломается.
>
> **Версия гайда:** 1.1 (2026-06-23). Шаблон: `deck_template.html`. Пример: `defense_presentation_v5.html`.
>
> **Ищешь формат для стратегического разбора-лонгрида, а не для раскадровки
> 16:9?** Это [`notebook_deck_guide.md`](notebook_deck_guide.md) — второй
> самодостаточный HTML-формат (зафиксированная палитра «тёплая бумага +
> терракота», восемь секций, компоненты вердиктов/лестниц). Используется всей
> серией [`evolutionary_analyses/`](evolutionary_analyses/). Не смешивать два
> формата в одном файле.

## 0. TL;DR — новый дек за 6 шагов

1. **Собери факты** из источников проекта (раздел 1) — никаких чисел «по памяти».
2. **Скопируй** `deck_template.html` → `<имя-дека>.html`.
3. **Замени `:root`-токены**, если нужна другая палитра/бренд (раздел 3).
4. **Наполни слайды** из библиотеки компонентов (разделы 4–6). Шаблон уже содержит по одному примеру каждого типа.
5. **Проверь** по чек-листу (раздел 8): печать, факты, влезает ли контент.
6. **Экспортируй PDF** (раздел 7) при необходимости.

Каркас неизменен: design-токены → слайды (`<section class="slide">`) → панель
навигации → `<script>`. Меняется только содержимое слайдов и, опционально, палитра.
**Счётчик слайдов считается из DOM автоматически** — N руками обновлять не нужно.

## 1. Сначала — факты, потом дизайн

Дек — витрина, но числа в нём должны быть **воспроизводимы из источников проекта**,
а не взяты из прошлой презентации. Для каждой метрики держи строку-источник
(команда / файл / прогон), чтобы её можно было перепроверить при защите.

**Как применять к своему проекту:** выпиши свои SSoT-источники в такую таблицу:

| Метрика | Где брать (команда / файл) |
|---|---|
| `<ваш KPI 1>` | `<команда grep / путь к файлу>` |
| `<ваш KPI 2>` | `<отчёт / лог прогона>` |
| … | … |

<details><summary><b>Пример — текущий проект hometutor</b> (резолвить пути: <code>hometutor</code> = CODE_ROOT, <code>hometutor-studio</code> = DOCS_ROOT)</summary>

| Метрика | Команда / источник | Репо |
|---|---|---|
| User Stories | `user_stories_index.json` → `items[].status` | studio |
| Пакеты закрыто | `grep -c "status: closed" doc/backlog_registry.yaml` | studio |
| Волны завершено | `grep -c "status: completed" doc/backlog_registry.yaml` | studio |
| Тест-функции | `grep -rc "def test_" tests/` (суммировать) | studio |
| FastAPI endpoints | `grep -rE "@router\.(get\|post\|put\|delete\|patch)" app/routers/ \| wc -l` | product |
| ADR | `grep -cE "^\| ?ADR-" doc/adr.md` | studio |
| Архитектура / стек | `docs/architecture.md`, `docs/technical_specification.md` | product |
| Local LLM throughput | последний benchmark-прогон (`doc/local_llm_*`) | studio |

> Свериться с актуальным: на 2026-06 у hometutor US 96/96 (не 87), тесты 3 011 (не 3008),
> docs пересобраны 2026-06-23. Не копируй числа из старого дека.
</details>

**Правило:** каждое число на слайде «Метрики» имеет источник. Eval-метрики на
demo-наборе — помечать явно («demo, N вопросов»).

## 2. Анатомия файла

Один `.html`, четыре блока, строго в этом порядке:

```
<head>
  └─ <style>  ─ design-токены (:root) + классы компонентов + @media print
<body>
  ├─ .progress          ─ полоса прогресса сверху
  ├─ .hint              ─ подсказка управления (правый верх)
  ├─ .stage > .deck     ─ контейнер 16:9, внутри все <section class="slide">
  ├─ .nav               ─ ‹ счётчик ›
  └─ <script>           ─ навигация, клавиши, клик по краям, печать
```

Каждый слайд — `<section class="slide">`. Первый имеет класс `active`. Скрипт
показывает по одному. **Никаких внешних `<link>`, `<img src="http...">`, шрифтов
с CDN** — иначе теряется самодостаточность.

---

## 3. Design-токены (`:root`) — единственное место правки темы

Вся палитра и геометрия — в CSS-переменных. Меняешь тему/бренд → меняешь только `:root`.

```css
:root{
  --bg:#0b1020; --bg2:#0e1430;            /* фон сцены */
  --panel:#141b38; --panel2:#1a2348;      /* карточки (градиент) */
  --ink:#eef2ff; --muted:#9aa6d4;         /* текст / приглушённый */
  --line:#283163;                         /* границы */
  --accent:#6d8bff; --accent2:#33e0c8;    /* акценты: индиго → циан */
  --gold:#ffcb6b; --good:#5ee0a0; --bad:#ff7a90;
  --grad:linear-gradient(120deg,#6d8bff,#33e0c8);  /* фирменный градиент */
  --shadow:0 20px 60px rgba(0,0,0,.45);
  --radius:18px;
  --font:'Segoe UI',system-ui,-apple-system,Roboto,Arial,sans-serif;
  --mono:'JetBrains Mono','Cascadia Code',Consolas,monospace;
}
```

**Перекраска под бренд:** поменяй `--accent`, `--accent2` и `--grad` на свои два
цвета — этого достаточно, чтобы дек сменил айдентику.

**Светлая тема для раздаток (печать).** Подмени токены:
`--bg:#f6f8ff; --bg2:#ffffff; --panel:#ffffff; --panel2:#f0f3ff;
--ink:#12183a; --muted:#5a648c; --line:#d7ddf2`. Градиент-акценты оставить —
они читаются на обоих фонах. (В шаблоне эта строка лежит закомментированной в `:root`.)

**Фирменный приём — градиентный текст** (заголовок-акцент, цифры-метрики):

```css
.accent{ background:var(--grad); -webkit-background-clip:text;
         background-clip:text; color:transparent; }
```

## 4. Сетка слайда и три типа

Каждый слайд: `kicker` (номер + рубрика) → `h2` (заголовок с `<span class="accent">`)
→ опц. `.sub` → `.body` (контент, скроллится) → `.foot` (бренд + подпись).

**Тип A — контент** (большинство):
```html
<section class="slide">
  <div class="kicker"><span class="num">06</span> Рубрика</div>
  <h2>Заголовок <span class="accent">с акцентом</span></h2>
  <div class="sub">Один абзац-подзаголовок (опционально).</div>
  <div class="body"> … компоненты … </div>
  <div class="foot"><span class="brand"><span class="dot"></span>проект</span><span>рубрика</span></div>
</section>
```

**Тип B — обложка** (`class="slide cover active"`): `.logo`, крупный `h1`, `.tag`,
ряд `.badge` в `.meta`.

**Тип C — разделитель/итог** (`class="slide divider"`): центрированный `.big`
+ ряд `.badge`. Радиальный фон уже в CSS.

Все три типа уже лежат заготовками в `deck_template.html`.

> **Высота — главный риск.** Слайд имеет фикс. аспект 16:9. Контент должен влезать
> без скролла на 1080p. `.body` скроллится как страховка, но цель — чтобы скролла
> не было. Меньше текста, больше карточек; длинные списки → 2–3 колонки `.grid`.

## 5. Библиотека компонентов (есть в шаблоне, копировать как есть)

| Компонент | Класс | Когда |
|---|---|---|
| Карточка | `.card` (+ `h3`, `p`, `ul`) | блок-контейнер, 2–3 в ряд |
| Сетка | `.grid.g2` / `.g3` | колонки |
| Цифры-метрики | `.stats > .stat (.v .l)` | KPI-слайд |
| Таблица | `<table>` | сравнения, статусы |
| Тезис/цитата | `.thesis` / `.thesis.small` | ключевая мысль |
| Моноблок-схема | `.flow` | ASCII-пайплайн, `white-space:pre` |
| Пилюли | `.pillrow > .pill` | теги / список фич |
| Маркеры | `ul > li` | квадратный градиент-буллет авто |

**Семантические инлайн-классы текста:** `.accent-t` (циан), `.gold-t` (золото,
«новое/важное»), `.muted`, `.yes`/`.no`/`.part` в таблицах (зелёный/красный/жёлтый
для ✅/✗/частично).

**Цифровой KPI-слайд** — самый сильный визуально:
```html
<div class="stats">
  <div class="stat"><div class="v">00</div><div class="l">подпись метрики</div></div>
  <!-- 4 или 8 штук в сетке 4×N -->
</div>
```

## 6. Диаграммы — инлайновый SVG, не картинки

Любая схема рисуется `<svg viewBox="0 0 W H">` прямо в слайде. Это и есть причина
самодостаточности. Паттерн (есть в шаблоне):

```html
<div class="diagram">
  <svg viewBox="0 0 800 200" width="100%" style="max-width:760px">
    <defs>
      <linearGradient id="g1" x1="0" x2="1">
        <stop offset="0" stop-color="#6d8bff"/><stop offset="1" stop-color="#33e0c8"/>
      </linearGradient>
    </defs>
    <rect x="40" y="78" width="160" height="44" rx="10" fill="#141b38" stroke="url(#g1)"/>
    <text x="120" y="105" fill="#eef2ff" text-anchor="middle">Узел</text>
    <path d="M200 100 L320 100" stroke="url(#g1)" stroke-width="2" fill="none"/>
  </svg>
</div>
```

Правила SVG:
- Цвета SVG **дублируют** значения токенов вручную (SVG не видит CSS-переменные
  при печати надёжно) — держи их синхронными с `:root`.
- `text-anchor="middle"`; стрелки `path` с `fill="none"`; циклические связи —
  `stroke-dasharray="5 5"`.
- **Уникальный `id` градиента на каждый SVG** (`g1`, `g2`…), иначе цвета конфликтуют.
- В примере `defense_presentation_v5.html` так нарисованы: split-диаграмма, цикл из
  7 узлов с обратной связью, трёхслойная схема.

> Реальные скриншоты **можно** добавлять, но только через `data:` base64
> (`<img src="data:image/png;base64,…">`), чтобы не зависеть от путей. Для большинства
> схем чистый SVG предпочтительнее — он векторный и печатается резко.

## 7. Навигация и печать в PDF

**JS (≈25 строк, уже в шаблоне).** Даёт: `←/→/PageUp/PageDown/Space` — листание;
`Home/End` — края; `F` — печать; клик по правой/левой половине дека — вперёд/назад;
полоса `.progress` и счётчик `.nav`. Число слайдов берётся из DOM. При смене слайда
сбрасывает `scrollTop` у `.body`.

**Печать в PDF — критичный блок `@media print` (уже в шаблоне):**
```css
@media print{
  @page{size:1280px 720px;margin:0}
  .stage{position:static;display:block}
  .deck{width:1280px;height:720px;max-height:none;aspect-ratio:auto}
  .slide{display:flex!important;position:relative;
    page-break-after:always;break-after:page;
    width:1280px;height:720px;border-radius:0;box-shadow:none;border:none}
  .nav,.progress,.hint{display:none!important}
}
```
Логика: в печати показываются **все** слайды (override `display:none`), каждый —
ровно одна страница 1280×720 (16:9), хром скрыт.

**Экспорт PDF:**
- Вручную: открыть в браузере → `F` (или Ctrl+P) → «Сохранить как PDF», поля
  «Нет», фон-графика **включена** (Chrome: «Background graphics»), размер из `@page`.
- Headless (Chrome/Edge), подставь свои пути:
  ```powershell
  & "C:\Program Files\Google\Chrome\Application\chrome.exe" `
    --headless --disable-gpu --no-pdf-header-footer `
    --print-to-pdf="<имя-дека>.pdf" `
    "file:///<абсолютный-путь>/<имя-дека>.html"
  ```

## 8. Чек-лист перед сдачей

- [ ] Каждое число на «Метриках» имеет источник; ничего «по памяти»
- [ ] Числа сверены с актуальными источниками (не скопированы из прошлого дека)
- [ ] Ни одного внешнего ресурса: нет `http(s)://`, CDN-шрифтов, внешних `<img>`
- [ ] Все слайды влезают без скролла на 1080p (проверить визуально)
- [ ] Первый слайд — `active`; счётчик стартует `1 / N` сам (N из DOM)
- [ ] У каждого SVG свой `id` градиента; цвета синхронны с `:root`
- [ ] Печать (`F`): все слайды, по одной странице, фон-графика на месте
- [ ] `.foot` подпись и `kicker .num` осмысленны и идут по порядку
- [ ] Заменены все плейсхолдеры шаблона (`ВАШ ПРОЕКТ`, `00`, `vX.X`, `Узел A`…)

## 9. Частые ошибки

| Симптом | Причина | Фикс |
|---|---|---|
| При печати белые слайды без фона | отключена фон-графика | в диалоге печати включить «Background graphics» |
| Контент уезжает за край | слишком много текста на слайде | резать текст, выносить в `.grid`/`.card`, дробить слайд |
| Градиент-текст невидим в Firefox-печати | `background-clip:text` | дублировать важное обычным `color` или печатать из Chrome/Edge |
| Два SVG «слиплись» по цвету | одинаковый `id` градиента | дать уникальные `id` |
| Счётчик показывает не то число | остался `<section>` без класса `slide` или лишний | N считается из `.slide`; проверь разметку |
