# Текст выступления для защиты

Здравствуйте. Я представляю проект **home-rag_v2** — персональный учебный ассистент, который работает с локальными учебными материалами пользователя.

## Слайд 1. Обзор продукта: проблема и решение

Главная проблема, которую решает проект, в том, что у студента часто есть много материалов: лекции, статьи, заметки, документация, но извлечь из них пользу сложно. Обычный поиск по файлам находит фрагменты, но не объясняет тему. ChatGPT может объяснить, но не знает именно мои материалы. Anki помогает повторять, но карточки нужно создавать вручную.

home-rag_v2 объединяет эти этапы в один учебный цикл: документы индексируются, пользователь задаёт вопрос, получает ответ с источниками, затем может перейти к разбору темы, квизу, карточкам и отслеживанию прогресса. Главная идея проекта — не просто ответить на вопрос, а довести пользователя до освоения темы.

## Слайд 2. Архитектура и стек технологий

На этом слайде показана архитектура системы. Она состоит из нескольких слоёв: интерфейсный слой, API, сервисная логика и persistence.

Пользователь может работать через Streamlit UI, CLI или Telegram-бота. API реализован на FastAPI. Основная логика находится в сервисах: query service, tutor orchestrator, learner model service, knowledge graph. Для хранения используются Chroma как векторный индекс и SQLite для прогресса, карточек и учебного состояния.

Важное архитектурное решение — local-first подход. Индекс и прогресс хранятся локально. Также все обращения к LLM и embeddings проходят через единый модуль provider.py, поэтому можно переключаться между локальной Ollama и cloud-провайдерами без изменения бизнес-логики.

## Слайд 3. RAG: сердце системы

Здесь показано, как работает RAG-пайплайн. Сначала документы из папки data проходят ingest: сканирование, парсинг, чанкинг, извлечение метаданных, embeddings и сохранение в Chroma.

Особенность проекта — двухуровневая индексация: отдельно индексируются документы целиком и отдельные фрагменты. Это нужно, потому что для точечного вопроса удобнее искать чанки, а для обзора темы или учебного плана — документы целиком.

Дальше показан query pipeline: classify, rewrite, retrieve, rerank и generate. Система сначала понимает тип запроса, при необходимости улучшает формулировку, ищет релевантные материалы, переранжирует их и только потом генерирует ответ с источниками. Это снижает риск галлюцинаций и делает ответ проверяемым.

## Слайд 4. Учебный цикл: вопрос -> квиз

На этом слайде показано, что после ответа работа не заканчивается. Пользователь получает ответ по своим материалам, видит confidence и источники, а затем может нажать «Учить эту тему».

После этого система создаёт квиз по теме ответа. Это важный момент: квиз проверяет не случайную тему, а именно то, что только что было объяснено. После результата можно создать flashcard и обновить mastery score. Так RAG превращается из чата в учебный инструмент.

## Слайд 5. Прогресс и рабочее пространство курса

Здесь показан слой прогресса. Система отслеживает, какие концепты пользователь уже узнаёт, какие может вспомнить, а какие умеет применять. Используется шкала recognition, recall, transfer и concept graduation.

Также есть workspace курса: можно активировать отдельную папку как отдельный курс и вести прогресс независимо. Это удобно, если пользователь учит несколько дисциплин или проектов параллельно.

## Слайд 6. Smart Study Router

Этот слайд объясняет, зачем нужен Smart Study Router. Когда в системе много режимов — вопрос, tutor, quiz, карточки, dashboard — пользователь может не понимать, что делать дальше.

Smart Study Router решает это: он анализирует состояние обучения и предлагает следующий шаг. Например, если квиз провален, он предлагает разобрать слабое место. Если давно не было проверки, предлагает mini-quiz. Если есть данные mastery, предлагает адаптивный план.

То есть система становится не просто набором функций, а учебным проводником.

## Слайд 7. Local LLM vs Cloud LLM

Здесь сравниваются два режима работы. Local LLM через Ollama даёт приватность, offline-режим и нулевую API-стоимость после установки моделей. Минус — качество и скорость зависят от железа.

Cloud LLM обычно быстрее и качественнее, но данные отправляются провайдеру и появляется стоимость запросов. В проекте это переключается через .env, без изменения кода. Также возможен гибридный режим: embeddings локально, генерация через cloud.

## Слайд 8. Процесс разработки

Этот слайд показывает не пользовательскую, а инженерную часть проекта. Раньше работа над пакетом требовала множества ручных шагов: открыть документы, выбрать команду, запустить тесты, проверить DoD.

В проекте был создан workflow router: он анализирует сложность задачи и выбирает маршрут разработки. Лёгкие задачи идут напрямую в исполнение, сложные — через orchestration. Это позволило сократить количество ручных действий и сделать процесс более воспроизводимым.

## Слайд 9. Идеи развития

Здесь перечислены направления развития. Первое — desktop installer и onboarding wizard, чтобы пользователь мог установить приложение не через терминал, а в несколько кликов.

Второе — исследование образовательной эффективности: сравнить retention, quiz scores и time to mastery у пользователей с системой и без неё. Третье — дальнейшая автоматизация разработки: event-driven watching и параллельная работа агентов.

## Слайд 10. Для кого проект

Основная аудитория — студенты с техническим бэкграундом, которым нужно учиться по собственным материалам. Также проект подходит пользователям, которым важна приватность: например, если материалы нельзя отправлять в облако.

Для новичков без опыта программирования пока есть барьер установки, поэтому нужен desktop installer. Для преподавателей в будущем потребуется multi-user режим и dashboard класса.

## Слайд 11. Сравнение с конкурентами

На этом слайде проект сравнивается с NotebookLM, Anki, ChatGPT с файлами и Obsidian. Каждый из этих инструментов силён в отдельной области: NotebookLM хорош для Q&A по документам, Anki — для повторения, Obsidian — для заметок, ChatGPT — как универсальный ассистент.

Отличие home-rag_v2 в том, что он объединяет весь учебный цикл: ответ по документам, tutor, quiz, flashcards, mastery tracking и адаптивный план. То есть ниша проекта — не просто чат и не просто карточки, а связанная система обучения.
