01 Проблема и решение

Куча документов → как извлечь пользу?

Проблема

  • Обычный чат-бот не знает ваших документов
  • Поиск по файлам даёт обрывки без объяснения
  • Карточки для повторения приходится делать вручную
  • Непонятно, что уже выучено, а что нет

Итог: прыгаешь между инструментами и теряешь нить.

Решение — один инструмент на весь цикл

Документы → Ответ → Объяснение → Проверка → Карточки → Усвоение ↑ Помощник подсказывает, что делать дальше

И всё это — на вашем компьютере: данные и модель остаются у вас.

Главное отличие: обычный поиск или чат останавливается на ответе. Здесь ответ — только начало; система доводит тему до усвоения.

02 Полный учебный цикл

Ответ — это первый шаг, а не финал

Обычная система «вопрос-ответ» на ответе заканчивается. Здесь с него всё начинается.

Ответ → Объяснение

Сначала ответ со ссылками на источники, затем разбор темы наводящими вопросами, а не сухой лекцией.

Проверка → Карточка

Короткий квиз на понимание; важное автоматически превращается в карточку для повторения.

Повторение → Усвоение

Повторение по расписанию; система отслеживает, что вы по-настоящему усвоили.

04 Как находится ответ и почему ему можно доверять

Ответ строится из ваших материалов

Почему ответам можно доверять

  • Опора на ваши материалы — ответ из ваших документов, а не из «общих знаний» модели
  • Ссылка на источник — под каждым фактом видно, откуда он, и это можно проверить
  • Честное «не знаю» — если ответа в материалах нет, система это говорит, а не выдумывает

06 Разработка с помощью ИИ

Продукт создан с помощью ИИ-инструментов

ИИ помогал на всех этапах — от планирования до кода, тестов и документации. Процесс воспроизводим и под контролем человека.

Планирование

Все задачи и их статусы ведутся в одном источнике (реестр backlog) — ничего не теряется и не дублируется.

Командная цепочка

Роли «PO → аналитик → архитектор → разработчик → тестировщик» проходятся с участием ИИ как воспроизводимый процесс.

Контроль качества

Задача закрывается только с пройденными тестами и обновлённой документацией — «Definition of Done» проверяется автоматически.

Локальный ИИ в деле

Локальная модель (qwen3-coder-next на llama.cpp) сама вносит правки в код: git apply + прогон тестов под контролем скрипта.

Принцип: «документ — закон, скрипт — движок, агент — руль». Меньше ручной рутины, решения задокументированы, качество проверяется автоматически на каждом шаге.
