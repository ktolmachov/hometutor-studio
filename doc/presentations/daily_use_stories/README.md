# Истории ежедневного использования

Второй жанр, компаньон витрины `docs/quickstart_demo.md` и разбора
[`../evolutionary_analyses/11_usage_scenarios.html`](../evolutionary_analyses/11_usage_scenarios.html).
Разница жанров (зафиксирована в разборе №11, §3 «Противоречия»):

- **Витрина** (`quickstart_demo.md`, `evolutionary_analyses/`) продаёт **момент** —
  «вот что умеет продукт», wow-кадр.
- **Истории ежедневного использования** (этот файл) учат **ритуалу** — «вот что я
  делаю каждый день», пошаговые инструкции + живые диалоги, мотивирующие новичка.

## Правила жанра

1. **Стиль — notebook v1.0** (см. [`../notebook_deck_guide.md`](../notebook_deck_guide.md)):
   палитра не меняется; добавлены два новых компонента специально для этого жанра —
   `.steps` (нумерованные шаги с кружком-цифрой) и `.chat`/`.msg.user`/`.msg.system`
   (живой диалог), оба собраны из уже существующих токенов, новых hex нет.
2. **Каждая история проверяется по коду или по уже отснятому сценарию витрины** —
   сноска `из чего собрано` под каждой историей обязательна (тот же принцип
   антигаллюцинации, что и во всей серии, и что подтвердил quality pass
   сценариев `31`-`35` в разборе №11).
3. **Диалоги — иллюстративный контент, не заявление о фиче.** Формулировки реплик
   придуманы для живости, но структура ответа тьютора (три панели понимания +
   «проверь себя») — настоящая, взята из `tutor_chat_response_render.py`.
4. **Известные шероховатости не замалчиваются.** История «Утро 2» прямо называет
   два разных счётчика due (Flashcards vs tutor-chat) как открытую проблему, а не
   скрывает её ради красивой истории — с сноской на `full_circle_visibility_plan.md`.
