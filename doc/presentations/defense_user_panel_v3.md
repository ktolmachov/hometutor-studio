## Раунд 1: Студенты и самоучки

### 📱 Студент Артём

**Вопрос 1:** Я привык учиться по YouTube и конспектам в телефоне. Зачем мне ещё один инструмент с установкой Python?

**Ответ студента:** Если у вас нет конспектов и вы учитесь только по видео — инструмент не для вас прямо сейчас. Он решает конкретную боль: «у меня есть свои материалы, и я хочу учиться по ним, а не по интернету». Если вы скачиваете лекции в PDF или делаете заметки — это ваш случай.

---

**Вопрос 2:** Есть мобильная версия? Я не буду сидеть с ноутбуком ради повторения карточек.

**Ответ студента:** Полноценной мобильной версии нет — это локальное приложение. Но Telegram-бот покрывает мобильный сценарий: `/review` — карточки дня, `/ask Что такое attention?` — ответ по вашим материалам. Нативное мобильное приложение — в дорожной карте, но не в текущем MVP.

---

**Вопрос 3:** Сколько времени уйдёт на первый запуск, прежде чем я смогу задать первый вопрос?

**Ответ студента:** По quickstart — около 15 минут: установка зависимостей, настройка `.env`, запуск, загрузка первых файлов, индексация. После этого — первый вопрос. Если использовать cloud LLM (API-ключ), Ollama устанавливать не нужно. Честно: это не «один клик», это разработческий инструмент с quickstart для людей, умеющих открыть терминал.

---

### 🧑‍💻 Самоучка Настя

**Вопрос 1:** Я и так делаю заметки в Obsidian, синхронизирую с Notion, там всё структурировано. Что ваш инструмент даёт сверх этого?

**Ответ студента:** Хорошая структура — не то же самое, что знание. Вы можете прекрасно организовать 300 заметок, но не помнить 70% из них через месяц. Мой инструмент добавляет: RAG-поиск (находит по смыслу, а не по папкам), квизы из ваших же материалов, SM-2 повторение и mastery tracking. Если у вас уже есть дисциплина и вы помните всё — нет смысла. Если нет — вот решение.

---

**Вопрос 2:** Можно ли подключить мои Obsidian-файлы напрямую, без копирования?

**Ответ студента:** Да. Указываете папку `data/` — туда можно положить симлинк на ваш Obsidian vault, или настроить `DATA_PATH` в `.env` на нужную директорию. `.md`-файлы обрабатываются нативно. Единственное: индексация нужна после каждого крупного обновления, но есть частичная переиндексация — только изменённые файлы.

---

**Вопрос 3:** Я использую Bear, Roam и несколько других приложений для разных типов заметок. Как это всё объединить?

**Ответ студента:** Любой формат, который умеет экспортировать в `.md`, `.txt`, `.pdf` или `.docx`, подойдёт. Идеальный сценарий — настроить автоматический экспорт из каждого инструмента в общую `data/`-папку, которую проект индексирует. Это не автоматическая синхронизация, это ручной или скриптованный импорт.

## Раунд 2: Профессионалы и скептики

### 👩‍🏫 Преподаватель Светлана

**Вопрос 1:** Я хочу дать студентам инструмент для самостоятельной работы. Они разберутся сами?

**Ответ студента:** При наличии quickstart и материалов — да, если у студента есть базовые навыки работы с терминалом. Для аудитории без технической подготовки порог входа высоковат. Самый реальный сценарий: преподаватель разворачивает общий экземпляр (Docker), студенты заходят по URL. Это возможно, но требует серверной настройки.

---

**Вопрос 2:** Есть ли групповой режим, чтобы видеть прогресс всего класса?

**Ответ студента:** Нет. Это single-user local-first инструмент. Групповой режим с дашбордом преподавателя — это следующий уровень архитектуры (мультиарендность, аутентификация, shared storage). В текущем MVP каждый студент работает со своим изолированным state.

---

**Вопрос 3:** Могу ли я загрузить материалы курса один раз для всех, а студенты будут использовать готовый индекс?

**Ответ студента:** Можно создать «официальный» Docker-образ с уже проиндексированными материалами и раздать его студентам. Тогда каждый запускает готовый контейнер со своим пустым `user_state.db` и общим векторным индексом. Это не штатная фича, но архитектурно реализуемо.

---

### 🔒 Безопасник Роман

**Вопрос 1:** Мои документы — внутренняя документация компании. Куда уходят данные?

**Ответ студента:** При Ollama + локальные embeddings — данные не покидают машину вообще. Нет обращений к внешним серверам. При cloud LLM (OpenAI, OpenRouter) — фрагменты документов передаются провайдеру для генерации. Это осознанный trade-off. Для конфиденциальных материалов — только Ollama.

---

**Вопрос 2:** Кто-нибудь ещё, кроме меня, может получить доступ к моим данным через API?

**Ответ студента:** API доступен только локально (`127.0.0.1:8000` по умолчанию — привязка к loopback). Аутентификация: `session_id` для UI и `X-API-Key` header для REST. Если не открывать порт наружу — данные доступны только с вашей машины. При деплое на сервер нужно настраивать TLS и авторизацию через Nginx.

---

**Вопрос 3:** Как защищены мои учебные данные (карточки, прогресс) от кражи через документ с вредоносным контентом?

**Ответ студента:** Есть `guardrails.py` и `input_validation.py` для защиты от prompt injection через пользовательский ввод. Для injection через сами документы (при индексации) — это более сложная атака, которая требует document-level policy. Сейчас это зона риска, которую я честно признаю. Полный audit безопасности — следующий уровень зрелости.

## Раунд 3: Сравнение и хейтеры

### 😤 Хейтер Игорь

**Вопрос 1:** Это просто ChatGPT, который читает PDF и выдаёт галлюцинации с умным видом. Докажите обратное.

**Ответ студента:** Три принципиальных отличия. Первое: ответ привязан к конкретным источникам — кликабельным фрагментам ваших документов, которые можно проверить. Второе: self-correction loop отклоняет ответ при слабом контексте и говорит «данных недостаточно» вместо фантазирования. Третье: это не разовый чат, а учебный цикл со SRS, mastery tracking и историей прогресса. ChatGPT забывает вас через сессию — система помнит, что вы изучили.

---

**Вопрос 2:** SM-2 придумали в 1987 году. Почему это «инновация»?

**Ответ студента:** SM-2 — не инновация. Это проверенная, эффективная база. Инновация — в интеграции: SM-2 карточки создаются автоматически из RAG-ответа и квиза, а не вручную. Mastery tracking соединён с retrieval-контекстом — вопросы по слабым местам приоритизируются. Умный маршрутизатор использует SM-2 данные для рекомендации следующего действия. Алгоритм старый, но пайп вокруг него — новый.

---

**Вопрос 3:** ИИ всё равно иногда ошибается. Как вы объясните пользователю, что ответ неверный?

**Ответ студента:** Через прозрачность источников. Каждый ответ показывает, из каких конкретных фрагментов он собран. Если источник — ваш документ с ошибкой, ответ будет неверным. Система не может быть умнее ваших материалов — это её честное ограничение. Confidence score — это не «правда», это «насколько хорошо нашли источники». Пользователь сам проверяет источник.

---

### 📊 Anki-фанат Кирилл

**Вопрос 1:** У меня 5 000 карточек, выработанная привычка, 99% retention. Зачем ломать это?

**Ответ студента:** Не нужно ломать. Если у вас работающий процесс — не трогайте. Мой инструмент решает другую проблему: откуда берутся карточки. Вы создаёте их вручную — это требует времени и дисциплины. Здесь карточки рождаются из ответов на ваши вопросы и результатов квиза. Если вы учите новое и тратите много времени на создание карточек — вот где помогу.

---

**Вопрос 2:** Можно ли импортировать мои Anki-карточки в вашу систему?

**Ответ студента:** Прямого импорта нет. Экспорт в Anki тоже. Это честное ограничение. Системы используют разные форматы SRS. В дорожной карте есть интеграция с AnkiConnect, но это не текущая фича.

---

**Вопрос 3:** Ваш SM-2 хуже оригинального Anki, который оптимизируется 30 лет. Почему вы его используете?

**Ответ студента:** Согласен, что Anki — лучший в классе SRS. Мой SM-2 — это baseline, не конкурент Anki в части повторений. Главная ценность не в том, что мой алгоритм лучше, а в том, что карточки возникают автоматически из учебного процесса и mastery привязан к RAG-контексту. Для продвинутого SRS — Anki лучше. Для полного учебного цикла без ручной работы — мой инструмент.

## Раунд 4: Специализированные сценарии

### 🎓 Аспирант Лейла

**Вопрос 1:** Я работаю с arxiv-PDF: формулы, таблицы, многоколоночная вёрстка. Нормально ли это обрабатывается?

**Ответ студента:** Честно: академические PDF — это самый сложный случай. Текстовый слой извлекается нормально, но формулы превращаются в нечитаемые символы, таблицы теряют структуру, многоколоночная вёрстка иногда мешает. Для правильных формул нужен LaTeX-парсер или специализированный PDF-extractor (например, GROBID). Это не текущая функция — это следующий уровень работы с научными материалами.

---

**Вопрос 2:** Моя диссертация строится на 80 источниках. Как масштабируется система?

**Ответ студента:** При 80 PDF среднего размера — работает нормально. BM25 строится in-memory (может расти), ChromaDB хранит векторы локально. Узкое место — время переиндексации при первом запуске и RAM при BM25 на большом корпусе. Есть частичная переиндексация — при добавлении новых источников не пересчитывается всё. Для 80 документов это реалистично; для 800 — нужна оценка на вашем железе.

---

**Вопрос 3:** Как цитировать источники — мне важно точное место в документе?

**Ответ студента:** Система показывает: название файла, номер чанка, краткий фрагмент текста. Точный номер страницы — это зависит от метаданных PDF. Для академического цитирования система даёт указатель «откуда», но не заменяет Zotero или Mendeley для управления ссылками.

---

### 💸 Экономный Павел

**Вопрос 1:** Сколько реально стоит в месяц при активном использовании?

**Ответ студента:** При Ollama — 0 рублей после установки моделей. Модели скачиваются один раз. Только электричество и RAM. При cloud LLM (OpenRouter/OpenAI): каждый вопрос — ~200–500 токенов контекст + ответ. При 50 вопросах в день и gpt-4o-mini (~$0.15/1M input) — меньше $1 в месяц. При синтезных запросах с большим контекстом — дороже. Есть token budget guard, который не даст случайно потратить много.

---

**Вопрос 2:** Можно ли ограничить использование дорогих моделей только для важных вопросов?

**Ответ студента:** Через `.env` можно задать разные модели для разных задач — например, лёгкую для классификации запроса и тяжёлую для генерации итогового ответа. Explicit routing по типу запроса не реализован в UI, но в конфиге это гибко. Для power user с аккаунтом на OpenRouter — есть возможность указывать конкретный дешёвый model per use case.

---

### 🌏 Международный пользователь Карлос

**Вопрос 1:** Мои материалы на испанском, английском и частично русском. Работает ли поиск корректно?

**Ответ студента:** Векторный поиск через `nomic-embed-text` работает на испанском и русском — но эта модель оптимизирована под английский; для mixed-language corpus лучше переключиться на `bge-m3` или `multilingual-e5` через `EMBED_MODEL` в `.env`. BM25 — без языковой нормализации, что снижает точность на морфологически богатых языках. Рекомендую тестировать на своих данных: базовый поиск работает, оптимальное качество — через смену embedding-модели.

---

**Вопрос 2:** Ответы будут на том же языке, что документ, или на языке вопроса?

**Ответ студента:** На языке вопроса — LLM отвечает на том языке, на котором его спрашивают. Фрагменты документа извлекаются на языке оригинала, но ответ синтезируется в соответствии с языком вопроса. Если хотите всегда получать ответ по-испански — задавайте вопрос по-испански.

## Итоговый вердикт пользовательской панели

### ✅ Что реально работает

- **Бесшовный учебный цикл:** вопрос → тьютор → квиз → карточка → повторение без переключений
- **Автоматические карточки** из ответов — экономия 80% времени создания
- **Local-first при Ollama** — полная приватность, нулевые расходы на API
- **Mastery tracking** — видно, что освоено, а что нет
- **Telegram-бот** — мобильный доступ к ключевым функциям

### ⚠️ Честные ограничения, которые нужно признать

- Нет мобильного приложения (есть Telegram-бот)
- Нет группового/многопользовательского режима
- Академические PDF с формулами парсятся частично
- Импорт/экспорт Anki — в дорожной карте, не сейчас
- Порог входа — нужно уметь открыть терминал

### 🎯 Для кого этот инструмент

| Подходит | Не подходит |
|---|---|
| Есть свои материалы (PDF, MD, DOCX) | Нет собственных материалов |
| Хочу учиться, а не только искать | Нужен только поиск по документам |
| Ценю приватность / offline | Важна нулевая настройка (онлайн-сервис) |
| Готов к установке через терминал | Нужно «один клик, никакого кода» |
| Хочу закрыть полный учебный цикл | Уже счастлив с Anki + ChatGPT раздельно |
