Часть 1 из 8

Суть: чего студент по-настоящему хочет от «картинки»

Студент не хочет иллюстрацию. Разложим по первым принципам — инфографика закрывает три потребности, и все три уже названы в документах продукта.

ПотребностьЧто это на самом делеГде это уже зафиксировано
Ориентация за 30 секунд Увидеть весь материал одним взглядом: его форму, границы, «за что хвататься». Прямой ответ на страх №3 — «утонуть в материале». Три страха студента — ядро продукта (см. разбор №2: время до первого инсайта)
Двойное кодирование Слова + пространственная структура строят более прочную ментальную модель — если картинка связана с сутью, а не украшает. docs/multimodal_konspekt_plan.md §5: dual coding, coherence, анти-паттерн «декоративные иллюстрации»
Точка возврата Картинка как карта: с неё можно прыгнуть в точное место материала — раздел · строка · минута видео. Суть hometutor из разбора-родоначальника: возврат в точное место лекции — главное отличие от ChatGPT/NotebookLM/Anki
Определение

Инфографика в hometutor — это не жанр иллюстрации, а сжатие материала до формы, которую мозг схватывает за секунды и с которой можно вернуться в материал. Красота — побочный эффект ясности. Всё, что не ведёт обратно в материал, — постер, а не инструмент.

Часть 3 из 8

Противоречия: три напряжения и их синтез

НапряжениеЛожный выборСинтез
Красота ↔ достоверность Либо эффектный постер, «нарисованный» LLM (красиво, но врёт), либо сухая таблица (честно, но не схватывается) Инфографика компилируется, а не рисуется. Данные детерминированы и уже прошли гейт из 10 проверок (100% evidence — разбор №3). LLM допускается максимум к подписям — и только через preview. Красота — из шаблона, правда — из данных.
Файл ↔ интерактив Либо статичная картинка (удобно шарить, но мертва), либо живой экран (ведёт в материал, но заперт в приложении) Самодостаточный HTML-файл — одновременно и файл, и интерактив. Паттерн уже доказан в этом же репо: build_kg_html = шаблон + JSON-плейсхолдеры + вшитый d3. Один файл, работает offline, шарится, кликается.
Local-first ↔ богатый рендер Либо отказаться от диаграмм (offline свято), либо CDN и image-модели (богато, но чужое) Вендорить библиотеки и рисовать SVG из данных. D3 уже вендорен — mermaid вендорится тем же жестом. Image-модель не нужна вовсе: невозможное достигается не генерацией пикселей, а шаблоном над данными.
Часть 4 из 8

Боль-якорь: за картинкой о продукте — из продукта

Факт из жизни

2026-07-11: чтобы получить инфографику и mind-map по материалам о самом hometutor, владелец продукта загружает их в NotebookLM. Продукт, чья суть — «материал становится учебным объектом», не умеет показать материал. Ни студенту, ни собственному автору.

Точная причина — не «нет фичи», а три разрыва в уже существующем коде:

Разрыв 1 — приёмник без передатчика

# app/ui/living_konspekt_reader.py:140 — рендерер ЖДЁТ диаграммы: _MERMAID_RE = re.compile(r"```(?:mermaid|flowchart).*?\n(.*?)\n```", ...) # app/prompts/_impl.py — grep mermaid|диаграм: 0 совпадений # ни smart-konspekt, ни тьютор, ни quiz не выпускают ни одного блока ```mermaid # схема появляется, только если студент сам впишет её в markdown руками

Разрыв 2 — живой экспорт выбрасывается, мёртвый предлагается

# app/ui/knowledge_graph_d3.py:590 — самодостаточный интерактивный HTML карты курса def build_kg_html(payload) -> str: # шаблон + JSON + вшитый d3 # :670 — единственное использование: внутренний буфер Streamlit-компонента. # Файл никогда не предлагается пользователю. grep download|export|png # по dashboards_graph.py: пусто # а экспорт-кнопка ВНУТРИ компонента (KG-04, template:699-706) отдаёт # knowledge_graph.svg — статичный снимок: без кликов, без переходов в материал

Разрыв 3 — двойной стандарт local-first

# d3 — вендорен: app/ui/assets/d3.v7.min.js (CDN только как fallback, :592) # mermaid — нет (living_konspekt_reader.py:253): import mermaid from 'https://cdn.jsdelivr.net/npm/mermaid@11/...' # offline: схемы молча не рендерятся — деградация без предупреждения

И поверх всего — тир-замок: даже готовая карта курса открывается только на уровне 3 (feature_registry.py:42), а ориентация «увидеть всё сразу» нужна в первый день — именно тогда страх «утонуть» максимален (разбор №2).

Часть 5 из 8

Вердикты

Позиция по каждому элементу — не обзор опций.

ЭлементВердиктПочему
D3-карта курса + build_kg_html ОСТАВИТЬ + ОТКРЫТЬ Лучший визуальный актив продукта. Одна кнопка «Скачать живую карту (HTML)» превращает внутренний буфер в первый шарящийся артефакт. Паттерн «шаблон + JSON» — образец для всей будущей инфографики.
SVG-экспорт KG-04 ОСТАВИТЬ Быстрый снимок полезен. Не развивать: будущее за живым HTML, не за мёртвым SVG.
Mermaid-рендерер ПОЧИНИТЬ Вендорить mermaid@11 по образцу d3 — иначе local-first продукт показывает схемы только при живом интернете.
Передатчик схем (US-M2.2) ПОСТРОИТЬ Кнопка «Схема раздела»: промпт-роль в app/prompts/, preview «принять/отклонить», запись только по подтверждению. Контракт готов с 2026-07-05 — нужен только код.
«Одна страница лекции» ПОСТРОИТЬ Детерминированная сборка: section_index (скелет) + концепты лекции из граф-payload + таймкоды sidecar + подсветка слабого. LLM не обязателен. Это и есть инфографика-продукт.
Plotly-слой прогресса ОСТАВИТЬ НА МЕСТЕ Это зеркало студента, не выжимка материала. Не смешивать жанры: на «одной странице лекции» mastery — тонкий overlay, не дашборд.
«Чистый вид» (print_view) ОБЪЕДИНИТЬ Готовая точка выдачи longform — научить отдавать не только .md, но и one-pager. Не строить новый экран.
Инлайнер картинок ОСТАВИТЬ Готовый транспорт для будущих слайдов (M2.1) и любых встроенных визуальных артефактов.
Растровая генерация «как NotebookLM» НЕ ДЕЛАТЬ Против local-first, недоказуемо (LLM-иллюстрация врёт без evidence), результат мёртв — не ведёт в материал. Невозможное делаем возможным другим путём.
Часть 6 из 8

Ценность: живая инфографика вместо красивого постера

NotebookLM / Canvahometutor (после плана)
Каждый элемент ведёт в материалнет — картинка конечна да — раздел · строка · минута видео (суть продукта)
Пересборка при изменении материалазаново руками автоматически — компилируется из данных
Достоверностьна совести генерации за гейтом: 10 проверок, 100% evidence
Offline / приватностьоблако локально, один самодостаточный файл
Знает, что ты уже знаешьнет да — mastery/decay overlay из существующего payload
Измеримое доказательство

Северная звезда: CTR возврата — доля кликов из инфографики, приводящих в раздел или на минуту видео. Это прямая метрика сути продукта («возврат в точное место»). Опоры: (1) число скачиваний живой карты/one-pager; (2) доля лекций, у которых one-pager есть; (3) вклад в TTFI из разбора №2 — новичок с картой ориентируется быстрее.

Часть 7 из 8

Путь: один золотой маршрут и лестница уровней

Открыл лекцию→ Сверху — «одна страница лекции»→ 30 секунд ориентации→ Клик по слабому месту→ Раздел / минута видео→ Петля обучения

Инфографика встраивается не как новый экран, а как верхний этаж уже существующих поверхностей: шапка Живого конспекта и «Чистый вид». Ни одной новой вкладки.

  1. L0
    Карта курса скачивается. Кнопка над готовым build_kg_html. Достигается за минуты, эффект «вау, это файл».
  2. L1
    Схема раздела по кнопке. Передатчик к готовому mermaid-приёмнику, через preview. Первый LLM-визуал — под контролем студента.
  3. L2
    «Одна страница лекции» собирается сама. Детерминированный билдер, без LLM. Появляется в шапке Живого конспекта.
  4. L3
    One-pager выдаётся наружу. «Чистый вид» отдаёт HTML-файл рядом с .md; им можно поделиться.
  5. L4
    «Моя карта». Тот же one-pager с mastery/decay overlay: зелёное — знаю, тлеющее — забываю. Данные уже в payload.
Часть 8 из 8

План: P0–P2 по соотношению «эффект / усилие»

P0 · ХОД 1

Отдать живую карту

Что: кнопка «Скачать интерактивную карту курса (HTML)» на вкладке графа — st.download_button над результатом уже существующего build_kg_html(payload).

Усилие: ~десяток строк + smoke-тест. Эффект: первый экспортируемый визуальный артефакт продукта; работает offline; шарится одним файлом.

P0 · ХОД 2

Передатчик к готовому приёмнику

Что: кнопка «Схема раздела» в Живом конспекте: промпт-роль в app/prompts/_impl.py (через provider-layer) → mermaid → preview «принять/отклонить» → запись только по подтверждению. В том же ходе — вендорить mermaid@11 в app/ui/assets/ по образцу d3.

Усилие: контракт US-M2.2 уже спроектирован. Эффект: впервые продукт сам производит визуальное объяснение; рендерер перестаёт зависеть от CDN.

P1

«Одна страница лекции»

Что: детерминированный билдер one-pager: скелет из section_index + концепты лекции из граф-payload + таймкоды из media sidecar + подсветка слабого. Рендер — по паттерну build_kg_html (шаблон + JSON). Выдача: шапка Живого конспекта + «Чистый вид» (download HTML рядом с .md). LLM не обязателен.

Эффект: закрывает страх «утонуть» в первый день; каждый блок — точка возврата.

P2

«Моя карта» + слайды

Что: mastery/decay overlay на one-pager («что я знаю / что тлеет») — данные уже в payload; затем слайды из M2.1 через готовый инлайнер картинок. Отдельно — телеметрия CTR возврата на дашборд метрик.

Эффект: инфографика становится персональной — то, чего не может ни один внешний генератор.

Почему это шедевр в простоте

Ни одной новой библиотеки визуализации, ни одной image-модели, ни одного нового экрана. Весь план — соединить уже написанные половины конвейера: данные, прошедшие гейт, и рендереры, ждущие данных. «Невозможное» — инфографика уровня NotebookLM локально — оказывается не задачей генерации картинок, а задачей одной кнопки download и одного промпта с preview.
