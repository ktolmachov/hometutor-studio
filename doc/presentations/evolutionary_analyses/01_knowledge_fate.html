Часть 1 · Суть

Чего по-настоящему хочет студент, когда учится

Студент хочет одного: чтобы усилие не пропадало. Каждый момент понимания — дорогая, трудно добытая вещь. Смысл системы с памятью не в карточках и не в SM-2 — а в обещании: «всё, что ты однажды понял, я сохраню, верну в нужный момент и покажу, как ты растёшь».

Отсюда первый принцип петли: понимание → след → возврат. Знание, не оставившее следа, для системы не существует. А для студента именно оно и было главным событием дня: он час разбирался с тьютором, ему объяснили, он понял — и наутро система смотрит на него как на чистый лист. Это предательство обещания, даже если ни одна строка кода не «сломана».

Часть 2 · Реальность

Пять хранилищ, четыре события, один гейт

Вид с высоты: «что я знаю» записано в пяти независимых местах, учебных событий — четыре, и только одно из них (квиз) доходит до концептной памяти.

записывается разрыв — след теряется чтение витриной

Инвентаризация: зачем каждое хранилище и что с ним не так

ХранилищеЗачем появилосьСильноеСлабое
quiz_mastery уровень освоения концепта после квизов единственный вход — через provenance-gate слеп ко всему, кроме квизов

tutor_sessions.mastery_level микробар уровня в чате тьютора дёшев, сессионный пятое независимое «что я знаю»
Что уже образцово — не трогать

Provenance-gate (fact_source_binding.require_fact_provenance): mastery меняется только от проверяемого события, и вход один — apply_quiz_outcome_to_learner_state(). Дисциплина «нет освоения без доказательства» — редкая и правильная. Flashcard-handoff (карточка → мгновенное объяснение тьютора без RAG-раунда) — направление «память → понимание» уже сделано отлично. Планировщик один: compute_flashcard_schedule строится поверх apply_sm2 — дубля алгоритма нет, вопреки первому впечатлению.

Часть 3 · Противоречия

Три напряжения — и синтез для каждого

Строгость доказательств ↔ полнота памяти

Гейт пускает в mastery только квиз-события — и это правильно. Но большинство реального обучения происходит в диалоге и чтении, которые бесследны. Синтез: два слоя памяти. Ниже гейта — лёгкий слой exposure («видел, объяснялось» — без изменения mastery); выше гейта — evidence (квиз, ревью). Гейт не ослабляется ни на йоту — под ним появляется пол.

Карточки как продукт ↔ карточки как интерфейс памяти

Сейчас карточный SM-2 — самодостаточный остров с прекрасной механикой. Но карточка — это представление концепта, а не отдельная сущность знания. Синтез: карточка получает связь с концептом (тег concept:), ревью пишет в оба слоя через ту же единую точку с новым типом provenance flashcard_review. Механика карточек не меняется — меняется то, что о ревью узнаёт весь организм.

Пять зеркал ↔ одна истина

Каждое хранилище появилось честно — для своей витрины. Но пять мест записи = пять шансов разойтись (случай «76 vs 89 концептов» из прошлого разбора — то же семейство болезней). Синтез: одна запись, много представлений. Не сливать таблицы силой — объявить quiz_mastery + spaced_repetition источником истины, остальное постепенно переводить в derived views.

Часть 4 · Боль-якорь

«Час занимался — наутро система считает, что я ничего не учил»

Точная причина в коде, не в теории

После каждого tutor-ответа learner model получает пустой результат:

# app/query_response_postprocessing.py:407
update_learner_model_after_interaction("local", "tutor", {}, session_id=…)

# app/learner_model_service.py:528 — вся ветка "tutor" целиком:
elif it == "tutor":
    profile.cognitive_load = max(0.1, float(profile.cognitive_load) − 0.05)

Итог часа диалога: ни mastery, ни записи в очередь повторения, ни карточки — только «когнитивная нагрузка −0.05». Единственный шанс оставить след — пройти inline-квиз после ответа; пропустил кнопку — след нулевой. Сохранить ответ как карточку тоже нельзя: add_flashcard() вызывается только из API-роутера, в UI — ни одного вызова.

Вторая створка той же боли: review_flashcard() (flashcard_service.py:798) обновляет SR карточки, лог и недельный счётчик — и не трогает ни spaced_repetition, ни quiz_mastery. Студент три раза подряд провалил карточку по теме X — граф знаний остаётся зелёным, SSR не узнаёт. Особенно обидно: карточка знает свой источник (source_path_from_card) — данные для связи есть, связи нет.

Часть 6 · Ценность

Коэффициент следа — измеримое доказательство

Уникальность hometutor — замкнутая петля с возвратом в точное место лекции. Этот разбор показывает: петля замкнута только для квиз-пути. Метрика прогресса — «коэффициент следа»: доля учебных событий, оставляющих след в памяти системы.

Событие обученияСлед сейчасЦель после P0–P1
Квиз (scoped / micro / inline)100%100% — не трогать
Повторение карточки50%100% — оба слоя (P0-1)
Диалог с тьютором0%>0 — exposure-след (P0-2) + карточка по кнопке (P1)
Чтение конспекта0%P2 — exposure от собранных фрагментов корзины

Проверяется из существующих данных: чат-сессии и agent_runs против записей в spaced_repetition / review-лога за ту же неделю — без новой телеметрии.

Часть 7 · Путь

Один золотой путь и лестница уровней

Спросил → понял → проверил себя (inline) → знание легло во все слои → через N дней пришла карточка → провалил → мгновенное объяснение (handoff — уже работает) → вернулся в точное место лекции → проверил снова. Один цикл, ни одного тупика.

  • L0Просто спрашиваю — и вижу: «система запомнила N концептов из нашего разговора» (exposure)
  • L1Прохожу проверки после ответов — концепты переходят из «видел» в «доказал»
  • L2Повторяю по одной очереди «Повторить сегодня» — карточки и концепты вместе
  • L3Вижу карту памяти: граф с двумя яркостями — бледное «встречал», яркое «подтвердил»
  • L4Управляю расписанием — expert-настройки SM-2 (уже существуют)
Часть 8 · План

P0–P2 по соотношению «эффект / усилие»

P0 · ХОД 1

Замкнуть ревью карточки в концептную память

review_flashcard() пробрасывает quality в apply_quiz_outcome_to_learner_state-путь с новым provenance-типом flashcard_review; концепт — из тегов карточки. Провал карточки впервые становится виден графу и SSR. Малое усилие: единая точка записи уже существует.

P0 · ХОД 2

Exposure-след тьюторского ответа

Вместо outcome={} — лёгкая запись «встречены концепты …» (без изменения mastery, ниже гейта). Система впервые «видит» диалоговое обучение; в чате появляется честная строка «запомнил за тебя: N концептов».

P1

Кнопка «→ в карточку» из ответа + concept-теги + одна очередь

Жест сохранения ответа карточкой (front = вопрос, back = teaching_summary, теги source/concept из источников ответа); генератор карточек пишет concept:-тег; витрины сводятся к одной очереди «Повторить сегодня».

P2

Одна истина + точный возврат из карточки

mastery_vector → derived view; карточка получает section-anchor (возврат в раздел·строку·таймкод, а не только в файл); «коэффициент следа» выводится на дашборд метрик.

Планка неизменна: гениальность — в простоте. P0 не добавляет ни одной новой сущности — он соединяет два уже построенных, образцовых механизма (гейт и карточный SM-2), которые просто не были знакомы друг с другом.
