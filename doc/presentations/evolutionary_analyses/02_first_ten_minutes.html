Часть 1 · Суть

Чего по-настоящему хочет новый студент в первые 10 минут

Не «настроить систему» и не «пройти тур». Он хочет одного доказательства: мои материалы оживают. Момент, когда система отвечает по его собственному конспекту и показывает, откуда взяла ответ, — это и есть первый инсайт. Всё до этого момента — долг, всё после — доверие.

Три первых принципа: ценность до конфигурации (цели, уровни, туры — после первого ответа), ни одного тупика (из любого состояния есть следующий шаг) и обещания исполняются немедленно — интерфейс первого дня не имеет права говорить «готовится», если ничего не готовится: новичку не с чем сравнить, он верит буквально.

Часть 2 · Реальность

Онбординг построен дважды — и слои не пересекаются

Инвентаризация удивляет: недостатка онбординга нет, есть изобилие несоединённого. Слой «обещаний» (двери, hero, туры) и слой «исполнения» (пути на диске, билдер артефактов, LLM-стек) разрабатывались с любовью — но смотрят в разные папки.

Заготовленный первый обзор (draft-ответы до первого вопроса) — правильный ответ локальному стеку, который не может быть быстрым в холодную.

Часть 3 · Противоречия

Три напряжения — и синтез для каждого

Мгновенная ценность ↔ тяжёлый локальный стек

Локальная LLM, индексация, холодный старт — честные минуты ожидания. Синтез уже изобретён в этом же коде: first-session артефакт с заготовленными ответами — ценность, вычисленная заранее, во время индексации. Осталось одно: чтобы билдер смотрел туда, куда двери кладут материалы. Не ускорять стек — соединить готовое.

Провести за руку ↔ не мешать спасению

Тур запускается чекбоксом в onboarding-диалоге — когда индекс ещё пуст. Его первый шаг уводит в «Быстрый ответ» (tutorial_guide.py:109-113) — прочь с Mission Control, где живёт единственный экран, способный дать новичку материалы. Синтез: тур начинается не с чекбокса, а с события — первого успешного ответа. Глава 1 тура и называется «Первый ответ»: пусть стартует от факта, а не от намерения.

Честность о сбоях ↔ не напугать новичка

Если LM Studio не запущена, баннер говорит: «SSR работает в template-режиме» — звучит безобидно. Реальный масштаб: из коробки оба fallback-уровня выключены (enable_llm_fallback=False, HOME_RAG_LLM_FALLBACK_ENABLED=false), и первый же вопрос новичка умрёт по таймауту. Синтез: не тонкая диагностика подсистем, а один преполётный светофор: «что будет работать прямо сейчас: ответы / карточка SSR / индексация» — честно и без жаргона.

Часть 4 · Боль-якорь

«Первый обзор курса готовится» — и не приходит никогда

Точная причина в коде, не в теории

Новичок нажимает «Попробовать демо» — единственную кнопку, обещающую «ответы с источниками без подготовки». Дальше происходит вот что:

# Дверь кладёт демо в data/demo (demo_sandbox.py:22-23):
def demo_target_dir() -> Path: return DATA_DIR / "demo"

# …а единственный билдер первого обзора сканирует только data/docs
# (ingestion_index_full.py:356-357, то же в ingestion_index_partial.py:272):
ing_sup.run_first_session_precompute_tail(docs_root=ing.DATA_DIR / "docs", …)

# При этом hero ВЫВОДИТ «курс demo» из путей индекса (course_cache.py:871-873)…
inferred = sorted({path.split("/", 1)[0] for path in files})

# …и честно обещает (mission_control_first_session.py:216-217):
st.info("Первый обзор курса готовится — ниже есть подсказка, с чего начать")

Загрузка файлов кладёт их в data/uploads, quickstart советует «положите в data/» — ни одна из трёх дверей и ни одна строка документации не ведёт в data/docs/, единственное место, где артефакт первого обзора вообще может родиться (list_course_candidates, course_cache.py:529). Коронный актив онбординга — миссия курса, seed-вопросы с черновиками ответов и цитатами — структурно недостижим через собственные двери продукта. А фраза «готовится» превращает разрыв в невыполненное обещание, которому новичок верит буквально.

Вторая створка: та же архитектурная болезнь, что в разборах №0 и №1 — два вычисления одной истины. «Что считать курсом» решают два независимых кода: билдер (data/docs + фильтр + ≥3 файла) и hero-резолвер (первый сегмент пути любого индексированного файла). Они дают разные ответы для одного и того же корпуса.

Часть 6 · Ценность

TTFI — время до первого инсайта, измеримое уже сейчас

Уникальность hometutor в первые 10 минут: первый инсайт из собственных материалов, локально, без аккаунтов и облака. Ни ChatGPT (нет твоего корпуса), ни NotebookLM (нет локальности), ни Anki (нет ответов) этого момента не дают. Доказательство — метрика TTFI: время от первого запуска до первого ответа с непустыми источниками.

СигналЕсть сейчасНе хватает
Выбор двериfirst_run_door_selected (ui_events)—
Завершение онбордингаonboarding_completed + ui_level—
Латентность ответаlatency_budget.jsonl по 5 поверхностям—
Первый заземлённый ответсобытия нетfirst_grounded_answer + TTFI = разница таймстампов

North star: медиана TTFI через демо-дверь ≤ 3 минут; через собственные материалы ≤ 10 минут на чистой машине. Порог провала: новичок завершил онбординг, но за первую сессию не получил ни одного ответа с источником.

Часть 7 · Путь

Один золотой путь и лестница уровней

Запуск → 30-секундный онбординг (только уровень интерфейса) → пустой индекс → дверь «Демо» → индексация с прогрессом (секунды: 20 КБ) → hero с миссией и seed-вопросами (после P0 он реально строится) → клик по seed-вопросу → wait-UX → первый ответ с источником → «Подстроить объяснения под вас?» → предложение тура. Ни одного тупика, ни одного невыполненного обещания.

  • L0Демо-инсайт: первый ответ с источником из демо-набора (≤ 3 мин)
  • L1Свой материал загружен и проиндексирован — «оно читает мои конспекты»
  • L2Первый ответ по собственному материалу — настоящий первый инсайт
  • L3Курс активирован — появляются обзор курса, scope и cockpit
  • L4Первая проверка пройдена — знание оставило след (мост к петле памяти, разбор №1)
Часть 8 · План

P0–P2 по соотношению «эффект / усилие»

P0 · ХОД 1

Один резолвер кандидатов — билдер видит то же, что hero

Кандидаты первого обзора вычисляются тем же кодом, что и scope hero (первый сегмент индексированных путей + фильтр техпапок), а не отдельным сканом data/docs. Демо- и upload-двери впервые приводят к заготовленному обзору с seed-вопросами. Коронный актив включается для 100% дверей.

P0 · ХОД 2

Честный статус вместо «готовится»

Если артефакта нет и сборка не запланирована — hero не обещает: либо молчит (seed-чипы уже дают следующий шаг), либо показывает CTA «Собрать первый обзор» с запуском сборки для активного scope. Обещание = запланированное действие.

P1

Тур после первого ответа + преполётный светофор + метрика TTFI

Чекбокс тура из onboarding-диалога переезжает в предложение после первого успешного ответа. LLM-баннер говорит правду о масштабе («ответы не будут работать»), одна строка-светофор. Событие first_grounded_answer и TTFI из существующих таймстампов.

P2

Чистая машина без растяжек + замороженный демо-обзор

local_start.ps1 перестаёт зависеть от пути машины автора. Демо-курс поставляется с готовым артефактом первого обзора в репозитории — hero мгновенен даже без единого вызова LLM.

Планка неизменна: гениальность — в простоте. P0 не строит ничего нового — он соединяет два перфекционистски сделанных слоя онбординга, которые сегодня смотрят в разные папки одного диска.
