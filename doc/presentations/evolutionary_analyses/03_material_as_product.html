Часть 1 · Суть

Чего по-настоящему хочет студент от своего материала

Не «файлов в индексе» и не «графа». Студент хочет, чтобы его лекции стали живым учебником: который отвечает, проверяет, помнит — и возвращает в точное место (раздел · строка · таймкод). Материал как продукт — это превращение сырца в организм, которому можно доверять.

Отсюда два первых принципа. Карта обязана соответствовать территории: если граф врёт или отстаёт от индекса, рушится всё, что построено выше — маршрутизация, план, mastery. И честность лучше покрытия — этот принцип в коде уже победил дважды (гейт графа, alignment v3.4: «честное отсутствие таймкода лучше уверенного промаха»), но честность пока живёт в сайдкарах и логах, не доходя до студента.

Три производственные линии и пять паспортов качества

Материал производят три независимые линии — и у каждой свой инструмент честности, живущий в своём углу.

Ещё два инструмента качества вне линий: source readiness (триаж сырых файлов: нативный ли PDF, нужен ли docling — source_readiness.py, отдаётся через API) и аудит дубликатов концептов (knowledge_graph_audit.py) — который запускается только ручным скриптом rebuild_knowledge_graph.py, никогда при штатном реиндексе.

Гейт графа: десять проверок перед публикацией

Компилятор графа (course_graph_compiler.py:699) не публикует бандл, пока не пройдены все проверки: ≥3 документов, ≥12 концептов, ≥10 семантических связей, ≥3 междокументных, 100% концептов и связей с evidence, ≤25% сирот, 0 висячих ссылок, 0 циклов prerequisites, 0 filename-fallback узлов, 100% документов участвуют. Провал любой — бандл остаётся в staging, активным продолжает служить предыдущий граф.

Часть 3 · Противоречия

Три напряжения — и синтез для каждого

Честность ↔ покрытие

Гейт строг: курс из 1–2 документов никогда не получит семантическую карту (порог min_documents ≥ 3), любой из 10 провалов блокирует публикацию целиком. Это правильно — но для студента выглядит как беспричинное отсутствие. Синтез: не ослаблять гейт, а превратить порог в лестницу: «добавьте третий документ — появится семантическая карта». Fail reasons уже написаны по-русски; им не хватает только сцены.

Автоматика ↔ операторская линия

Конспекты и таймкоды рождаются вне приложения (PowerShell + CUDA), граф — внутри, аудит дубликатов — в третьем, ручном скрипте. Синтез: не втаскивать ASR в приложение — операторская линия легитимна. Объединять надо не производство, а витрину: один «паспорт материала», агрегирующий готовые сайдкары всех линий.

Свежесть ↔ стабильность

При провале гейта система выбирает стабильность: индекс новый, граф старый. Выбор верный — но молчаливый: рассинхронизация фиксируется одной строкой в логе. Синтез: рассинхронизация — это статус материала, а не деталь вкладки графа. Карта может отставать от территории, но студент должен видеть, на сколько.

Часть 4 · Боль-якорь

«Лекция уже отвечает — а на карте знаний её нет, и никто не сказал почему»

Точная причина в коде, не в теории

Студент добавил лекцию 6 и переиндексировал. Быстрый ответ по ней работает. Карта, план и маршрутизация — нет:

# Развилка качества в момент реиндекса (knowledge_graph.py:947):
use_compiler = graph_llm_probe_ok() # LLM недоступна → эвристика

# Эвристика честно не проходит гейт (knowledge_graph_bundle.py:303-310):
"gate_passed": False, "fail_reasons": ["Heuristic fallback path — semantic gate не пройден"]

# Promote пропускается — след остаётся только в логе (graph_generation_paths.py:49-54):
if not staging_bundle_gate_allows_promote(...):
    logger.info("knowledge_graph_promote_skip | reason=gate_not_passed")

# Активным продолжает служить граф предыдущего поколения (knowledge_graph.py:999-1022)

Единственное место, где об этом узнаёт человек, — вкладка графа, глубоко и на языке разработчика: «для active generation promoted bundle не найден», expander «Последний staging graph не опубликован» (dashboards_graph.py:707-726). Mission Control, учебный план и SSR молча продолжают жить по старой карте. Та же самая недоступность LM Studio, которая в разборе №2 убивала первый ответ, здесь молча портит реиндекс — один корень, две беды.

Вторая створка: аудит дубликатов концептов не входит в штатный цикл. write_graph_audit_report вызывается только из ручного scripts/rebuild_knowledge_graph.py:107 — при обычном реиндексе дубликаты («LLM» ≈ «ЛЛМ» ≈ «языковая модель») не ловятся никем. А дубликат концепта — это размытое mastery: знание студента делится между двумя узлами (прямая связь с разбором №1).

Часть 6 · Ценность

Материал с доказуемым качеством — и индекс свежести карты

Уникальность hometutor здесь: материал с доказательствами. Каждый концепт карты обязан иметь evidence в тексте (гейт: 100%), каждый таймкод — честный или отсутствует (v3.4), каждый конспект несёт sha256 своих источников. NotebookLM и ChatGPT не дают ни gate-метрик, ни честных таймкодов, ни провенанса. Но доказательства, которых не видно, не создают доверия.

МетрикаИз чего считаетсяСтатус
Индекс свежести карты = документы в индексе − документы в опубликованном графе манифест индекса + docs_participating из quality report обе величины уже существуют; нигде не сопоставляются
Доля курса с конспектами konspekt_discovery.coverage_summary есть, показывается в topics tab
Честное видео-покрытие .media.json coverage (объединение интервалов) есть, живёт в сайдкаре
Дубликаты концептов knowledge_graph_audit считается только при ручном rebuild

North star: после каждого успешного реиндекса индекс свежести карты = 0; при провале гейта он виден на витрине — не в логе.

Часть 7 · Путь

Один золотой путь материала и лестница уровней

Положил лекцию → реиндекс → чанки в индексе, граф прошёл гейт, карта обновилась → (есть видео? операторская линия дала конспект и честные таймкоды) → студент открывает паспорт материала: «6 документов · карта покрывает 6 · конспекты 5/6 · видео-привязка честная на 86%» — и знает, чему верить.

  • L0Файлы отвечают — материал в индексе, вопросы работают
  • L1Карта опубликована — гейт пройден, каждый концепт с evidence
  • L2Конспекты покрывают курс — по каждой лекции есть смысловой слой
  • L3Видео привязано честно — таймкоды с confidence, возврат в момент лекции
  • L4Паспорт полный — материал «готов к обучению», все линии сошлись
Часть 8 · План

P0–P2 по соотношению «эффект / усилие»

P0 · ХОД 1

Видимый статус «карта отстаёт от материалов»

Индекс свежести карты (разница уже существующих величин) выводится на scope/context строку Mission Control и в шапку карты — человеческим языком: «Карта не включает 2 новых документа — Мало междокументных связей». Ноль новых вычислений: publish status + fail reasons уже есть.

P0 · ХОД 2

Аудит дубликатов в штатный reindex-хвост

write_graph_audit_report вызывается после каждой сборки бандла (он без LLM и дешёвый), отчёт кладётся рядом с quality report. Дубликаты концептов перестают жить незамеченными между ручными запусками скрипта.

P1

«Паспорт материала» + лестница вместо порога

Одна секция, агрегирующая готовые сайдкары: гейт-метрики, konspekt coverage, видео-покрытие, source readiness, дубликаты. Для курса <3 документов — не пустота, а «добавьте третий документ — появится семантическая карта».

P2

Свежесть конспекта + learner-language

Сравнение sha256 из frontmatter конспекта с текущими входами → бейдж «конспект устарел» (needs discovery: где дёшево пересчитывать). Language-pass по вкладке графа: «staging bundle» → человеческие слова.

Планка неизменна: гениальность — в простоте. P0 не добавляет ни одной новой проверки качества — он выносит на сцену доказательства, которые система уже собирает, и включает в штатный цикл аудит, который уже написан.
