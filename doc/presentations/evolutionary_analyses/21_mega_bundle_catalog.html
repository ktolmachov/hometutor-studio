Эволюционный разбор · №21 · Мега-бандл курсов

Один город без адресов

Владелец хочет мега-бандл: несколько курсов одной области, общий план, сквозной маршрут в Мнемополисе и каталог, в котором нельзя потеряться. Разбор показывает: мега-бандл в слое данных уже построен — единый граф, восемь слитых общих тем, сквозная память. Не хватает не «мощнее», а трёх честных вещей: границ курсов, адресов на остановках и одного экрана-каталога.

1Суть

Чего хочет человек, который учится на эксперта по нескольким курсам сразу

Не «пройти три курса». Эксперт отличается от прилежного студента тем, что у него в голове одна карта области, а не три конспекта. Курсы для него — источники света на одну и ту же территорию: второй курс ценен не новыми главами, а тем, что он повторно освещает уже знакомые темы под другим углом. Экспертность растёт на пересечениях.

Отсюда три первых принципа мега-бандла. Общее сливается: «RAG» из первого курса и «RAG» из второго — один концепт с одной памятью, иначе прогресс дробится. Границы сохраняются: внутри курса порядок уроков — авторский закон; между курсами такого закона нет, и рисовать его выдумкой нельзя. Адрес не теряется: «не потеряться в курсах» — это в любой момент знать, где я и где это лежит: курс → урок → раздел → минута. Каталог — это не список файлов, это адресная система города.

2Реальность

Два слоя: город уже един — двери всё ещё по одной

Данные-эталон. В D:\AI\app\data живут два реальных курса: «ИИ Агенты» — 5 уроков полного конвейера (конспект .md + транскрипт .txt + .mp4 + .segments.json + медиа-сайдкар .media.json с таймкодами) и «ИИ Агенты Deep» — пока 1 модуль. Плюс личный рабочий-конспект в living-konspekt/.

Слой данных: мега-бандл уже существует

Единый индекс и сквозной поиск. Реиндекс охватывает весь DATA_DIR; курс-фильтр ретривала — опциональный параметр (query_service.py:133: course_id=options.folder_rel or options.folder). Вопрос без активного курса уже ищет по всем курсам сразу.

Единый граф обоих курсов, gate пройден. Активная генерация (reset_home_rag_v2_2026_07_17T21_01): 13 документов обоих курсов, 82 концепта, 192 типизированные связи, 12 меж-документных, gate_passed=true (10 проверок компилятора, course_graph_compiler.py:712-747).

Побочный выигрыш мега-бандла: курс «Deep» из 1 модуля никогда не прошёл бы гейт в одиночку (min_documents ≥ 3, :730) — в общем бандле он уже на карте.

Общие темы уже слиты — их 8. Прямой запрос к снимку активного бандла (kg.sqlite → graph_snapshot): у 8 концептов документы из обоих курсов — Agentic Loop, Tools, Guardrails, Evals, AI Agent, LLM, Pipeline, RAG; 53 концепта только базового курса, 12 — только Deep. Компилятор уже делает работу «найти общее между курсами». Наружу это не показано нигде.

Память уже сквозная. Mastery/SM-2 живут по каноническому cid (fact_source_binding, ход разбора №8): «RAG», выученный в первом курсе, — это тот же «RAG» во втором. Пересечение курсов уже засчитывается памяти — молча.

Слой опыта: все двери — одно-курсовые

Scope — строго одна папка. activate_scope(folder_rel=…) (app/ui/study_scope.py:146-173) принимает один каталог; опции курсов — top-level папки индекса (course_cache.py:492-520). Выражения «вся область / все курсы» в UI не существует (grep «Все курсы» по app/ui — 0 совпадений).

Каталога нет — есть плитки и вкладка. Конспекты перечисляет scan_konspekts(course_dir) (konspekt_discovery.py:78) — только для активного курса (topics_tab). Экрана «курсы → конспекты → разделы» с поиском нет ни в одном view.

Агент не видит библиотеку. Из 9 read-only инструментов агента ни один не принимает курс и не перечисляет материалы: rag.search ищет чанки, graph.inspect — концепты, а konspekt.inspect смотрит личный workbench, а не библиотеку курсов (grep course|scope|folder по app/agent/tools_*.py — 0). «Найди раздел про guardrails в курсе Deep» — задача без инструмента.

Карта и 3D-зал уже показывают весь бандл — но узлы не знают, из какого они курса. Экран графа рендерит концепты активной генерации целиком (dashboards_graph.py:1675-1676); активный scope используется только для плашки свежести и due-фильтра (:1705-1728). Маршрут дня выбирается по worth из всего бандла (select_day_route, knowledge_graph_d3_analysis.py:386) — он уже межкурсовой, просто не подписан. Атрибута «курс» у узла нет.

3Противоречия

Три напряжения — и их синтез

простой каталог ↔ масштаб мега-бандла

Каталог должен быть простым («не потеряться»), но курсов будет больше, конспекты толстые, разделов сотни. Синтез: каталог — не новая сущность, а проекция уже готовых резолверов. Курсы уже перечисляет build_mission_control_course_options, конспекты — scan_konspekts (с паспортом качества №14 и бейджем свежести), разделы — section_index. Простота достигается адресом, а не сокращением содержимого. Ни одного нового хранилища.

мощь всего бандла ↔ фокус одного шага

Страх мега-бандла — «утонуть во всём сразу». Синтез: фокус уже построен — worth и select_day_route выбирают лучший следующий шаг по всему бандлу. Продукту не нужен выбор «учить один курс или все»: маршрут дня и так один. Ему не хватает только подписи «курс · урок» на каждой остановке, чтобы мощь не ощущалась хаосом.

граница курса ↔ сквозные темы

Внутри курса закон — авторский порядок уроков. Между курсами закона порядка нет, зато есть общие концепты. Синтез — метро Мнемополиса: курс = линия (жёсткая последовательность станций), общая тема = пересадочная станция. Линию не рвём и не сшиваем с чужой; между линиями ходим по пересадкам — восьми уже слитым темам. Это дословно закон vision №20 («линии метро = курсы»), у которого, как выяснилось, уже есть данные.

4Боль-якорь

«Не хочу потеряться в курсах» — а теряется сам продукт

Проверяемый факт (живой прогон, не теория). Прямой вызов lesson_floor_order(...) на снимке активного бандла даёт Мнемополису 13 этажей на 6 реальных уроков:

  • этаж 1 — «Модуль 1» продвинутого курса Deep, поставленный ПЕРЕД «урок_1 Введение» базового курса — прямо в бандле лежит связь lesson:…deep-modul-1… → precedes → lesson:…urok-1-vvedenie…, которой не существует ни в одной программе обучения;
  • каждый урок стоит дважды — конспект (.md) и транскрипт (.txt) стали отдельными «уроками»-этажами;
  • последний этаж — личный рабочий-конспект из living-konspekt/, вшитый в лестницу уроков как «урок №9999».

Точная причина в коде:

  • course_graph_compiler.py:157 — все документы бандла сортируются одним плоским списком; ключ _lesson_sort_key (:119-125) берёт первое число имени файла: у «Модуль 1» и «урок_1» оно равно 1, а тай-брейк по пути ставит «ИИ Агенты Deep/» раньше «ИИ Агенты/» (пробел < «/»);
  • :207-215 — zip(lesson_meta, lesson_meta[1:]) рисует precedes между соседями списка, не глядя на границу папки — так продвинутый курс становится «прологом» базового;
  • knowledge_graph_d3_analysis.py:297-309 — lesson_anchor_key срезает расширения «.md»/«.txt», но id уроков уже слагифицированы в «-md»/«-txt» — коллапс вариантов не срабатывает, урок раздваивается;
  • lesson-узел создаётся «на каждый документ курса» (course_graph_compiler.py:146) — включая личный конспект, который курсом не является.

Это и есть «потеряться»: первый же экран мега-бандла — маршрут и этажи — уверенно врёт про порядок области. Не потому, что данных мало, а потому, что у 13 документов нет понятия «курс»: одна безымянная куча, отсортированная по первой цифре имени файла.

5Вердикты

Позиция по каждому элементу

оставитьЕдиный граф-бандл + gate. Мега-бандл достаётся бесплатно: 8 слитых тем — доказательство, что merge работает; «Deep» из 1 модуля попадает на карту только благодаря общему бандлу.
оставитьСквозная память по cid. Ровно то, что нужно эксперту: пересечение курсов уже засчитывается. Не дублировать per-course.
оставитьworth + select_day_route. Межкурсовой фокус уже построен — механизм не трогать, добавить только адрес на остановке.
оставитьscan_konspekts + паспорт №14 + section_index. Готовые кирпичи каталога; им не хватает лишь одного экрана, где они встречаются.
починитьprecedes через границы папок. Цепочка уроков — только внутри top-папки курса. Дёшево: группировка перед существующим zip.
починитьРаздвоение урока (.md/.txt). Один урок = один lesson-узел: группировать документы урока по стему до расширения на этапе компиляции — тогда оба зеркала (Python и JS) наследуют правду без правок.
починитьЛичный конспект в лестнице уроков. living-konspekt/ — не курс; lesson-узел ему не положен.
починитьБезымянность узлов. Узел получает атрибут «курс» (top-папка) — аддитивное поле payload, домен-схема не ломается.
объединитьПлитки курсов + вкладка конспектов + поиск. Один экран-каталог «Библиотека»: курсы → конспекты → разделы → действие (открыть / активировать / спросить).
спрятать глубжеScope-переключатель. Из «главной двери» — в фильтр внимания: карта и Q&A уже умеют жить поверх всех курсов; активный курс остаётся дефолтным фокусом, а не границей мира.

НЕ делать

не делатьВторой «мега-граф» или хранилище каталога. Бандл уже один на все курсы; каталог — проекция файловой системы и индекса. Третья копия истины — типовая болезнь серии (№1, №2, №13).
не делатьLLM для порядка курсов и границ. Границы даёт файловая система, порядок внутри курса — номера уроков, порядок между курсами — явное решение владельца (метаданные), не генерация.
не делатьМульти-scope ретривала. Сквозной поиск уже есть (пустой scope); чинить нечего.
не делатьАвтослияние конспектов разных курсов в один текст. Слияние — на уровне концептов (уже работает), не текстов: перегенерация рушит якоря строк, сайдкары и живость (вердикт №14).
6Ценность

Библиотека, которая сама знает, чем её книги связаны

NotebookLM даёт чат по источникам — без памяти ученика. Anki даёт память — без карты области. Obsidian даёт карту — без памяти и без маршрута. hometutor с мега-бандлом — единственный из четырёх, где «RAG» из двух разных курсов — один узел с одной памятью и адресами обоих первоисточников. Уникальность — пересадочные станции: общие темы курсов с общей памятью и обратными адресами.

North star: «у каждого шага есть адрес» — доля остановок маршрута дня, подписанных «курс · урок»: 0% → 100%.

Проверить перенос

Короткий квиз подтверждает, что знание перенеслось между курсами.

8 минquiz
В маршруте нет остановок по этому запросу.

Принцип переноса референса: время доклада → адрес, спикер → курс, зал → линия, тема → концепт, звезда → pin. Плитка остаётся единицей сканирования; подробности открываются следующим уровнем, а не вкладываются внутрь карточки.

8Путь

Золотой путь и лестница уровней

Золотой путь одной сессии: открыть «Библиотеку» → увидеть свои курсы, их конспекты с паспортами и восемь общих тем → нажать «Маршрут дня» → пройти остановки с адресами («Guardrails · Deep, Модуль 1 · раздел 4»), пересадка между курсами — только через общую тему → из остановки — в раздел конспекта или отрезок лекции (ворота №19) → след уходит в сквозную память → вечером карта честно показывает, что область стала на N шагов ближе.

  1. 0Один курс. Сегодняшний режим: активный scope, вкладка конспектов, карта без имён курсов.
  2. 1Честные границы. Этажи = реальные уроки, precedes не пересекает папки, у каждого узла есть курс (P0-1).
  3. 2Библиотека. Один экран: курсы → конспекты → разделы, до любого материала ≤ 2 кликов (P0-2).
  4. 3Пересадочные станции. Общие темы видимы: бейдж «в 2 курсах», в зале — пересадка между линиями (P1).
  5. 4Агент-навигатор. «Найди, где в Deep про guardrails» — инструмент каталога у агента, ответ с адресом (P1).
  6. 5Мнемополис-метрополитен. Курс = линия метро, цвет линии, пересадки = общие темы — мир vision №20, волна W4 (P2).
