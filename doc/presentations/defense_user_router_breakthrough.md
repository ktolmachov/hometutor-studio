# Прорыв после пользовательской панели: Smart Study Router

Источник: [`defense_user_panel.md`](defense_user_panel.md) + паттерн [`team_workflow/workflow_router.md`](team_workflow/workflow_router.md).

## Что для пользователей самое ценное

Пользователи не покупают "RAG", "Chroma" или "LLM pipeline". Они хотят меньше хаоса в обучении.

Самое ценное:

1. **Одна учебная траектория вместо набора инструментов.**  
   Главная боль панели — фрагментация: ChatGPT для вопросов, Anki для карточек, Obsidian для заметок, отдельный планировщик для повторения.

2. **Проверяемые ответы с источниками.**  
   Пользователь боится выучить ошибку. Источники и confidence превращают ответ из "поверь модели" в "проверь по материалу".

3. **Переход от ответа к действию.**  
   Самый сильный момент продукта — кнопка "Учить эту тему": ответ становится началом tutor, quiz, flashcard и adaptive plan.

4. **Local-first и offline.**  
   Для корпоративных, экономных и privacy-sensitive пользователей это не деталь, а причина выбрать продукт.

5. **Честность ограничений.**  
   Пользователи лучше реагируют на честное "OCR пока нет", "learning outcomes ещё не доказаны", "локальная модель слабее cloud", чем на обещания магии.

## Чего больше всего не хватает

1. **Понятного следующего шага.**  
   Даже если функции сильные, пользователь может не понимать: спросить вопрос, пройти tutor, сделать quiz, повторить карточки, открыть dashboard или настроить индекс?

2. **Onboarding без терминала.**  
   Новичок не хочет выбирать Python, `.env`, provider, embeddings, reranker. Ему нужен guided setup.

3. **Доверия к эффективности.**  
   Нужны eval baseline, learning outcomes, retention metrics. Без этого критик справедливо скажет: "Интересно, но где доказательство?"

4. **Интеграции с существующими привычками.**  
   Power user не хочет бросать Anki/Obsidian. Ему нужен export/import, а не ультиматум "переходи полностью".

5. **Группового сценария.**  
   Преподаватель хочет dashboard класса, ручные квизы, прогресс студентов, ACL.

## Что взять от умного workflow-роутера

Smart Workflow Router сделал прорыв потому, что убрал вопрос "какой документ открыть и какой промпт запускать?". Его паттерн можно перенести в пользовательский учебный интерфейс.

| Workflow Router | Smart Study Router |
|---|---|
| Читает `backlog_registry.yaml` | Читает learning state: mastery, quiz ошибки, due cards, последние вопросы |
| Определяет состояние пакета | Определяет состояние ученика и темы |
| Выбирает следующий шаг | Предлагает лучшее учебное действие |
| Даёт `next_label`, `next_cmd`, `next_hint` | Даёт `next_action`, причину, кнопку и fallback |
| Скрывает матрицу решений | Скрывает сложность учебной методики |
| Одна команда `workflow.py` | Одна панель "Что делать дальше?" |

Главная формула:

> Пользователь не должен выбирать режим. Система должна объяснить, что лучше сделать сейчас и почему.

## Сильный продуктовый ход

Добавить в интерфейс блок **"Умная подсказка следующего шага"**.

Пример:

> Следующий лучший шаг: пройти короткий quiz по теме "retrieval pipeline".  
> Почему: вы получили хороший ответ с источниками, но ещё не проверяли понимание.  
> После quiz я создам карточки по слабым местам и обновлю план повторения.

Кнопки:

- **Начать quiz**
- **Сначала объяснить проще**
- **Создать карточки**
- **Открыть источники**
- **Пропустить**

## Состояния учебного роутера

| Состояние | Сигнал | Подсказка |
|---|---|---|
| `no_material` | Нет индекса или документов | "Загрузите первый материал" |
| `indexed_no_question` | Материал есть, вопросов не было | "Задайте первый вопрос по материалу" |
| `answer_ready` | Есть ответ с источниками | "Учить эту тему" |
| `low_confidence_answer` | Низкий confidence или слабые источники | "Уточнить вопрос или открыть источники" |
| `tutor_started` | Tutor начал тему | "Ответьте на 1 вопрос, чтобы проверить понимание" |
| `quiz_due` | Есть тема без проверки | "Пройти короткий quiz" |
| `weak_concept` | Ошибки в quiz / низкий mastery | "Разобрать слабое место и создать карточку" |
| `cards_due` | Есть карточки к повторению | "Повторить N карточек сегодня" |
| `mastery_stale` | Давно не было проверки | "Проверить retention через mini-quiz" |
| `plan_ready` | Есть данные mastery | "Открыть адаптивный план на неделю" |

## Что это изменит в восприятии

До:

> Инструмент мощный, но я не понимаю, куда нажимать и что делать дальше.

После:

> Он сам ведёт меня по обучению. Я вижу следующий шаг и понимаю, почему он выбран.

Это превращает продукт из набора функций в учебного проводника.

## Как показать на защите

Добавить к Killer Demo один момент после ответа с источниками:

> Система сама предлагает: "Учить эту тему", потому что ответ найден, источники достаточные, но понимание ещё не проверено.

После quiz:

> Теперь подсказка меняется: "Создать карточку по слабому месту", потому что я ошибся в вопросе про reranking.

После flashcard:

> Теперь следующий шаг — "Повторить завтра" или "Открыть adaptive plan".

Финальная фраза:

> Умный роутер уже улучшил процесс разработки. Следующий уровень — такой же роутер для обучения: не пользователь ищет режим, а система ведёт его к следующему полезному действию.

## MVP без большой перестройки

Начать можно без сложного ML:

1. Собрать `StudyState` из уже существующих данных: индекс, последние ответы, quiz results, flashcard due dates, mastery.
2. Написать rule-based `recommend_next_study_action(state)`.
3. Вернуть структуру:

```json
{
  "next_action": "quiz_due",
  "label": "Пройти короткий quiz",
  "reason": "Вы изучили тему, но ещё не проверили понимание",
  "primary_button": "Начать quiz",
  "secondary_actions": ["Объяснить проще", "Открыть источники"]
}
```

4. Показать блок в Streamlit dashboard и после RAG-ответа.
5. Логировать принятие/отклонение подсказок для будущей адаптации.

## Почему это сильнее обычного roadmap

Desktop installer снижает входной барьер. Eval baseline повышает доверие. Export в Anki помогает power users.

Но **Smart Study Router** меняет сам опыт использования: пользователь чувствует, что система понимает его учебный контекст и ведёт дальше. Это и есть "восторг от умной подсказки".
