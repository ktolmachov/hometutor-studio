## Волна `wave-kg-node-worth` (P0)

### A1 «Цена на узле» — due + novel + честная шапка

- **Problem.** Узел не называет ни одной причины остановки: карточки «к повторению»
  собираются при каждом рендере и выбрасываются; «новое для тебя» не вычисляется; шапка
  зала показывает «89 концептов» при подписи «76 концептов» под тем же iframe.
- **Evidence.** `app/ui/knowledge_graph_d3.py:649` — `get_due_reviews(limit=20)`
  собираются; `:678` — передаются в `build_kg_payload`; `:500` — параметр принят и в теле
  не используется ни разу. Новизна: `mastery_vector`/`decay_vector`/`mastery_history`
  уже в payload (`:507-510`, `:600-602`) — отсутствие концепта во всех трёх словарях
  нигде не читается. Шапка: `knowledge_graph_d3_template.html:305` рендерит `STATS.total`
  (концепты + 13 lesson-узлов) с подписью «концептов», тогда как Streamlit-подпись уже
  переведена на `total_concepts` (`dashboards_graph.py:1093-1096`, комментарий B1) —
  фикс №0 дошёл до всех поверхностей, кроме самой шапки зала.
- **Proposed.**
  1. В `build_kg_payload`: маппинг due_reviews → `{concept_id: n_due}` (совпадение по
     каноническому cid — тот же резолвер, что в скелете), поле `due` на узле; поле
     `novel = cid not in mastery_vector | decay_vector | mastery_history` (для
     не-lesson узлов).
  2. Тултип и вкладка «✦ Что учить» в шаблоне: строки-причины «🃏 ждут N карточек»,
     «✨ новое для тебя» рядом с уже существующими «mastery … · открывает N концептов»
     (template:489).
  3. Шапка: `STATS.total` → `STATS.total_concepts` + отдельная плитка «уроков N»
     (`STATS.total_lessons` уже считается, `_kg_counters_from_skeleton:360`).
- **Files.** `app/ui/knowledge_graph_d3.py`,
  `app/ui/assets/knowledge_graph_d3_template.html`; тесты: расширение теста payload
  (поля due/novel), тест счётчиков шапки (если есть snapshot-тест шаблона — проверить).
- **DoD.** На живой базе узел с ожидающими карточками показывает их число; нетронутый
  концепт помечен «новое»; на экране графа одно число концептов (76) и явное «уроков: 13»;
  0 новых запросов к БД (due уже собирались).
- **Doc-sync.** `docs/user_guide.md` (экран графа).
- **Effort.** ~День.

### A2 «Маршрут дня по ценности» — W(узел) + связный тур вместо голого BFS

- **Problem.** KG-05 «Маршрут» — невзвешенный BFS по unlocks/prereqs
  (кратчайший по хопам); «Что учить» — фильтр без ранжирования. «Идеального маршрута
  с достойными остановками» не существует как функции.
- **Evidence.** `_bfsPath`, `knowledge_graph_d3_template.html:679-689`; фильтр
  `n.frontier` `:896`.
- **Proposed.**
  1. `worth(node)` — чистая функция в `app/ui/knowledge_graph_d3_analysis.py`
     (рядом с build_decay_vector/build_graph_health): взвешенная сумма из
     due (A1), decay, frontier, novel (A1), normalized reach/centrality. Веса —
     константы модуля с докстрингом; юнит-тесты на граничные случаи.
  2. Поле `worth` на узле в payload; в route-панели шаблона — кнопка
     «Авто: маршрут дня»: топ-K узлов по worth, соединённых в prereq-порядке
     (топологический порядок скелета уже есть), рендер той же анимацией остановок
     KG-05; у каждой остановки — причина одной строкой (наибольшее слагаемое W).
  3. Существующий ручной режим «старт → цель» остаётся без изменений.
- **Files.** `app/ui/knowledge_graph_d3_analysis.py`, `app/ui/knowledge_graph_d3.py`,
  `app/ui/assets/knowledge_graph_d3_template.html`; тесты на worth() и на выбор
  топ-K остановок.
- **DoD.** «Маршрут дня» на живой базе ведёт через узел с максимальным due, а не мимо
  него; каждая остановка подписана причиной; при пустом личном слое (новый пользователь)
  маршрут честно строится по novelty+reach и не падает.
- **Doc-sync.** `docs/user_guide.md`.
- **Effort.** ~День-два.
- **Dependencies.** A1.

**Статус:** `done` (2026-07-15 polish + audit) — `build_kg_3d_html` +

- **Problem.** 3D в продукте — 0 строк (grep three/THREE/WebGL/3d-force по app/);

- **Proposed.** Новый шаблон `kg_3d_template.html` + `build_kg_3d_html(payload)` по
  образцу `build_kg_html`: вендоренный three.js (+3d-force-graph или собственный
  минимальный рендер) в `app/ui/assets/` по образцу `_load_d3_source()`; Z-раскладка —
  этажи-уроки (lesson-узлы уже канонически определяются `is_lesson_node`), высота
  над этажом / свечение = `worth` (A2); кнопка «⬇ Скачать 3D-зал» рядом с
  «⬇ Скачать живую карту» (`dashboards_graph.py:1100-1107`); «полёт по маршруту» —
  камера по остановкам «Маршрута дня». CDN запрещён (болезнь №6) — нет вендора, нет фичи.
- **Files.** `app/ui/assets/` (three vendored, новый шаблон),
  `app/ui/knowledge_graph_d3.py` (или новый тонкий `knowledge_graph_3d.py`),
  `app/ui/dashboards_graph.py` (кнопка).
- **DoD.** Скачанный файл открывается офлайн, показывает этажи и остановки, полёт
  проходит маршрут A2; payload не расширяется под 3D отдельной схемой (тот же объект).

## Рекомендованный порядок

A1 → A2 → B1 → B2 → C2 → C1.

**North star:** время от открытия карты до старта достойной остановки — минуты → секунды;
каждая остановка маршрута отвечает «почему я здесь» одной строкой.

**Метрики.** Узлов с названной причиной остановки: 0 → все остановки маршрута.
due_reviews: выбрасываются → поле узла. «Новое для тебя»: неотличимо от «0%» → поле novel.
Чисел «концептов» на экране: 2 → 1. Маршрут: кратчайший по хопам → максимум ценности.
