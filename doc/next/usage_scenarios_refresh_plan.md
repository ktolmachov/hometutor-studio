# Usage Scenarios Refresh Plan

**Дата:** 2026-07-12 · **База анализа:** hometutor HEAD `c374892` «194»
**Источник:** эволюционный разбор №11 «Витрина обещаний: сценарии против реальности»
(`../presentations/evolutionary_analyses/11_usage_scenarios.html`).

План материализует вывод разбора №11: витрина сценариев сильна механикой, но
нуждается в честной свежести и закрытии слепых зон. Механику пайплайна не менять:
YAML остаётся SSoT, `scripts/generate_demo_doc.py` и demo workflow остаются
единственным путём публикации.

## Контекст

- В `doc/scenarios/` уже 30 YAML-манифестов сценариев.
- `docs/quickstart_demo.md` — витрина, собранная из YAML + screenshots/meta.
- Главная уязвимость: freshness gap и перекос покрытия. Сценарии успешны, но
  не показывают честный сбой, HF demo-гостя, экспорт живой карты, возврат в
  таймкод и privacy/offline-проверку как самостоятельные обещания.
- После Full Circle P0 переснять/обновить сценарии 06 и 30: они сейчас могут
  обещать больше видимости, чем было на момент съёмки.

## P0 — два хода, YAML-only

### A1. Штамп свежести витрины

- **Problem:** читатель не видит, насколько сценарий свеж относительно HEAD.
- **Proposed:** `scripts/generate_demo_doc.py` выводит в шапку документа и каждого
  сценария строку вида `снято на HEAD <sha> · отстаёт на <N> коммитов`, используя
  уже существующие `meta.json` и git metadata.
- **Scope:** одна правка вывода; YAML-схему и capture workflow не менять.
- **Extra:** у сценариев с опережающими обещаниями (06, 30) добавить явную сноску
  до пересъёмки после Full Circle P0.
- **DoD:** quickstart demo показывает freshness stamp; сценарии 06/30 честно
  помечены; strict demo validation не регрессирует.

### A2. Пять манифестов слепых зон

Добавить новые YAML по существующей схеме:

1. честный сбой LLM;
2. гость HF Spaces;
3. экспорт живой карты;
4. возврат в таймкод видео;
5. приватность / offline-проверка.

- **Scope:** только новые манифесты и, если нужно, разделы `doc/user_scenarios.md`.
- **Capture:** съёмка через studio/demo pipeline позже; манифесты не ждут съёмки,
  чтобы попасть в SSoT.
- **DoD:** новые сценарии проходят YAML/schema validation; покрытие архетипов из
  круглого стола пользователей становится 5/5 на уровне каталога.

## P1 — второй жанр и синхронизация

- Создать парный документ «Истории ежедневного использования»: не демо-кадры, а
  дни/ритуалы, живые диалоги и переход от момента к привычке.
- Связать gap-анализ с `.kiro/specs/user-scenarios-refresh`, если этот слой есть
  в checkout: один список слепых зон должен питать и каталог, и витрину.
- Переснять сценарии 06/30 после Full Circle P0-1/P0-2.

## P2 — видео и автоматизация

- Нарезка видео из `narration` полей YAML-манифестов.
- CI/локальный reminder: freshness gap превысил порог, например 50 коммитов.

## Метрики

| Метрика | Сейчас | Цель |
|---|---:|---:|
| Freshness gap | ~192 коммита | видимый stamp + пересъёмка при gap > 50 |
| Покрытие архетипов стола №3 | 2/5 | 5/5 |
| Сценарии честного сбоя | 0 | >=1 |
| Таймкод-возврат как уникальность | нет | отдельный сценарий |
| Обещания, опережающие продукт | >=2 | 0 после Full Circle P0 + пересъёмки |

## Kill switch

Если актуализация требует менять механику конвейера, структуру YAML или обходить
существующий demo workflow, остановиться. Сила витрины — в неизменной механике:
новые манифесты + один честный слой вывода.

## Prompt helper

Промпт для агент-сессии актуализации собирается скриптом
`doc/scenarios/generate_scenario_refresh_prompt.py`. Скрипт не вызывает LLM: он
только печатает постановку задачи для агента, который сам сверяет манифесты с
HEAD и предлагает новые сценарии.
