## Волна `wave-audio-first-sound` (P0)

### A1 «Подать звук» — аудиодорожка лекции + слушаемый раздел/плейлист

- **Problem.** Продукт не отдаёт ни секунды звука: 0 × `st.audio` во всём `app/`
  (единственный аудио-виджет — `st.audio_input`, `app/ui/sidebar.py:526`); плейлист
  «🎧 Мои N минут» для локального видео рендерит подписи вместо действия.
- **Evidence.** `app/ui/living_konspekt_media.py:141-143` — `_playlist_video_url`
  возвращает `None` для `LocalVideoSource`; `:137-138` — пункт становится caption.
  Данные готовы: сайдкар урока_2 — 120 разделов, все с `t_start`/`t_end`
  (v3.4, 0 low_confidence). Замер 2026-07-13: `ffmpeg -vn -c:a copy` отделяет
  аудио 6:26-видео за 0,29 с, 4,5 МБ против 29 МБ.
- **Proposed.**
  1. Офлайн-шаг в существующем конвейере (`scripts/build_media_sidecar.py` или
     `Run-MediaKonspektBatch.ps1`): после сайдкара извлекать `<урок>.m4a`
     (`ffmpeg -vn -c:a copy`), путь по конвенции рядом с mp4 (`with_suffix(".m4a")`) —
     схема сайдкара НЕ меняется, файл обнаруживается по имени. Нет ffmpeg —
     пропуск с подсказкой (образец: `remux_to_mp4`, `scripts/transcribe_media.py:233-235`).
  2. `_render_media_panel` (`living_konspekt_media.py`): если аудиофайл существует —
     кнопка/чекбокс «🎧 Слушать раздел» → `st.audio(path, start_time=t_start,
     end_time=t_end)` (сигнатура проверена в установленном Streamlit:
     `media.py:71-80`). Рендер за кликом — тот же паттерн, что у `st.video`
     (комментарий о цене полного чтения файла, `:276-279`).
  3. `render_playlist_panel`: пункты локального видео при наличии `.m4a` становятся
     слушаемыми (последовательность плееров), отбор/порядок
     `playlist_items_from_rows` не трогается.
- **Files.** `scripts/build_media_sidecar.py` (или batch-скрипт),
  `app/ui/living_konspekt_media.py`; тесты `tests/test_media_sidecar.py` /
  `tests/test_living_konspekt_view_smoke.py` (+ новый узкий тест обнаружения
  `.m4a` по конвенции).
- **DoD.** На курсе «ИИ Агенты» раздел урока_2 слушается из Живого конспекта;
  «Мои N минут» звучит для локальных видео; без ffmpeg/без `.m4a` — прежнее
  поведение без ошибок.
- **Doc-sync.** `docs/user_guide.md` (Живой конспект), `docs/multimodal_konspekt_plan.md`
  (новый шаг конвейера), поправка ADR-0002 — см. A2.
- **Effort.** Часы.

### A2 «Выпуск в дорогу» — один m4a из корзины

- **Problem.** Слушать «в дороге» нечем: все 15+ `st.download_button` в UI отдают
  текст/HTML/JSON; артефакта-аудиофайла не существует.
- **Evidence.** Грепы HEAD 207: `st.download_button` ×15 (образец паттерна —
  `print_view.py:46-53`); производителей аудио 0.
- **Proposed.** Кнопка «Скачать выпуск (m4a)» рядом с «🎧 Мои N минут»: нарезка
  отрезков корзины из уже извлечённых `.m4a` (A1) + `ffmpeg concat` без
  перекодирования → один файл + текстовое оглавление таймкодов (тот же
  `playlist_items_from_rows`). Нет ffmpeg в PATH — кнопка заменяется честной
  подсказкой. Требуется поправка ADR-0002: мандат системного ffmpeg
  «remux only» → «remux + аудио-операции» (одна строка, новых зависимостей нет).
- **Files.** `app/ui/living_konspekt_media.py` (+ возможно тонкий helper
  `app/media_audio.py` для нарезки/склейки с тестами на команду, без запуска ffmpeg
  в юнитах); `docs/adr/0002-asr-dependency-strategy.md`.
- **DoD.** Из корзины с ≥2 фрагментами скачивается один играющий m4a с оглавлением;
  время «корзина → файл» на демо-курсе — секунды.
- **Doc-sync.** ADR-0002, `docs/user_guide.md`.
- **Effort.** ~День.
- **Dependencies.** A1.

## Волна `wave-audio-voice-and-trace` (P1)

### B1 След «прослушано» → learner state

- **Problem.** Прослушивание бесследно — повтор греха №1 («чтение бесследно») в новом
  канале.
- **Proposed.** Событие «раздел прослушан» (клик ▶ / завершение фрагмента) пишет
  exposure-след тем же слоем, что закрыл якорь №1 (слой ниже provenance-gate,
  mastery не трогает). Раздел уже знает свои концепты через section_index/граф.
- **Effort.** Часы—день. **Dependencies.** A1.

### B2 Extra `[tts]` — озвучка разделов без медиа

- **Problem.** Разделы без видео (и конспекты без лекций) не звучат; единственный TTS —
  `speak()` в динамики сервера без файла (`app/voice_service.py:106-118`).
- **Proposed.** Optional extra `[tts]` в `pyproject.toml` по образцу `[asr]`
  (кандидат: piper / silero, локально); функция «текст раздела → аудиофайл в кэш» +
  тот же плеер из A1. `speak()` в динамики не расширять; форма «текст → файл →
  плеер/скачивание» — единственная поддерживаемая.
- **Doc-sync.** Новый короткий ADR (TTS dependency strategy) по образцу ADR-0002.
- **Effort.** Дни. **Dependencies.** A1 (плеер).

### B3 Унификация ASR — `voice_service` на faster-whisper

- **Problem.** Две независимые ASR-линии; студентский голос уходит в облако Google
  (`recognize_google`, `app/voice_service.py:69,96`) при докстринге «(локально)» —
  против local-first.
- **Proposed.** `transcribe_audio_bytes`/`listen_microphone_once` переводятся на
  faster-whisper из уже существующего extra `[asr]` (graceful: extra не установлен —
  прежнее честное сообщение об отсутствии зависимости). `recognize_google` удаляется.
  Заодно решает жалобу Telegram-обработчика на форматы (`telegram_handlers.py:298-303`).
- **Effort.** День. **Dependencies.** нет (независим от A1/A2).

## Волна `wave-audio-reach` (P2)

### C1 Telegram-доставка выпуска

- **Proposed.** Команда бота «выпуск корзины» → `answer_audio` с m4a из A2: телефон —
  родной плеер дороги. Требует A2.

### C2 Диалоговый подкаст (opt-in)

- **Proposed.** Новая промпт-роль (сценарий диалога из ПРОВЕРЕННЫХ фрагментов, с
  section-ссылками) по образцу `section_diagram` в `PROMPT_ROLE_CONTRACT`
  (`app/prompts/_impl.py:1189-1192`) + TTS из B2 двумя голосами. Строго opt-in,
  строго после уровней 0–3 лестницы. Вердикт разбора: НЕ первый ход
  (путь NotebookLM — галлюцинации в уши без возврата в источник).

### C3 «Сегмент дня в ушах»

- **Proposed.** Аудио-вариант US-M4.1 (`docs/multimodal_konspekt_plan.md`): ротация
  Course Cockpit предлагает 3–5-минутный доверенный фрагмент в наушники.

- Переходы «из уха в раздел» (обратный адрес работает).
- Доля прослушанных разделов со следом в learner state (после B1).
