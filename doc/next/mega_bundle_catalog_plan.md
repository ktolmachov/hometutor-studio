# Mega bundle: candidate plan («Один город без адресов» — границы курсов, адреса, каталог)

**Источник:** эволюционный разбор №21
(`doc/presentations/evolutionary_analyses/21_mega_bundle_catalog.html`), 2026-07-18,
тема владельца «мега-бандл нескольких курсов: общий план, маршрут в Мнемополисе по всем
конспектам, простой каталог с ИИ-навигацией».

**Evidence-база и её свежесть.** Проверено на hometutor @ `8623c3476` «291»
(2026-07-18). Все `файл:строка` ниже — актуальный HEAD; при реализации сверять по
имени функции, не по номеру строки. Данные-эталон: `D:\AI\app\data` — курс
«ИИ Агенты» (5 уроков, полный конвейер) + курс «ИИ Агенты Deep» (1 модуль) +
`living-konspekt/рабочий-конспект.md`; активная генерация графа
`reset_home_rag_v2_2026_07_17T21_01` (снимок `kg.sqlite`).

**Тезис разбора.** Мега-бандл в слое данных уже существует: единый индекс (сквозной
Q&A при пустом scope), единый граф-бандл обоих курсов (13 документов, 82 концепта,
gate ✓), **8 общих тем уже слиты компилятором** (Agentic Loop, Tools, Guardrails,
Evals, AI Agent, LLM, Pipeline, RAG — документы обоих курсов у одного концепта),
память сквозная по каноническому `cid`. Слой опыта — одно-курсовый: scope = одна
папка, каталога «курсы → конспекты → разделы» нет, у агента нет инструмента
библиотеки, а там, где бандл прорывается наружу (этажи/маршрут), он не знает границ
курсов и врёт.

**Боль-якорь (живой прогон, не grep).** Прямой вызов `lesson_floor_order(...)`
(`app/ui/knowledge_graph_d3_analysis.py:312`) на снимке активного бандла →
**13 этажей на 6 реальных уроков**: этаж 1 — «Модуль 1» продвинутого курса Deep,
поставленный ПЕРЕД «урок_1 Введение» базового (в `typed_relations` буквально лежит
`lesson:…deep-modul-1…-txt → precedes → lesson:…urok-1-vvedenie…-md`); каждый урок
раздвоен (конспект `.md` и транскрипт `.txt` — отдельные «уроки»); последний этаж —
личный рабочий-конспект («урок №9999»).

**North star (одна на весь план):** «у каждого шага есть адрес» — доля остановок
маршрута дня, подписанных «курс · урок»: 0% → 100%. Прокси-факты DoD: этажей в зале
13 → 6; `precedes` через границы папок 2 → 0; общих тем видно 0 → 8.

## P0-1 «Курс — гражданин графа»: границы и честные этажи

- **Problem.** Компилятор не знает понятия «курс»: 13 документов бандла — одна
  плоская куча, отсортированная по первой цифре имени файла. Отсюда ложный
  межкурсовой prerequisite, раздвоенные уроки и личный конспект в лестнице уроков.
- **Evidence.**
  - `app/course_graph_compiler.py:119-125` — `_lesson_sort_key` берёт первое число
    стема («Модуль 1» = «урок_1» = 1), тай-брейк `path.casefold()` ставит
    «ИИ Агенты Deep/» раньше «ИИ Агенты/» (пробел < «/»);
  - `:157` — `sorted(documents_grouped.items(), key=_lesson_sort_key)` без
    группировки по папке; `:207-215` — `zip(lesson_meta, lesson_meta[1:])` рисует
    `precedes` между соседями списка сквозь границу папок;
  - `:146` — lesson-узел «на каждый документ курса» → `.md` и `.txt` одного урока —
    два узла; `living-konspekt/рабочий-конспект.md` тоже получает lesson-узел;
  - `app/ui/knowledge_graph_d3_analysis.py:297-309` — `lesson_anchor_key` срезает
    расширения «.md»/«.txt», но id уже слагифицированы в «-md»/«-txt» — коллапс
    вариантов не срабатывает (подтверждено живым прогоном: 13 этажей).
- **Proposed.** Чинить в компиляторе (одна правда — оба зеркала, Python
  `lesson_floor_order` и JS `computeLessonOrder` в `kg_3d_template.html`, наследуют
  её без правок):
  1) группировать документы урока по стему до расширения (md+txt → один
     lesson-узел; у узла оба документа в `documents`);
  2) цепочку `precedes` рисовать отдельно внутри каждой top-папки курса
     (группировка перед существующим zip — чистая арифметика);
  3) `living-konspekt/` (и любые не-курсовые корни) — без lesson-узла и вне
     цепочки; фильтр по существующему резолверу курсовых папок;
  4) аддитивное поле `course` (top-папка) на lesson- и концепт-узлах payload —
     render-contract расширяем (прецедент №18: домен-схема заморожена, рендер-поля
     добавляемы); карта/зал подписывают этаж и остановку курсом.
- **Files.** `app/course_graph_compiler.py` (группировка/сортировка/zip),
  `app/knowledge_graph_payload.py` или место сборки NODES (поле `course`),
  `app/ui/knowledge_graph_d3_analysis.py` + `app/ui/assets/kg_3d_template.html` —
  только если подпись курса рендерится там; тесты на группировку и границы.
- **DoD.** Пересборка реального бандла (`scripts/rebuild_knowledge_graph.py`) →
  0 `precedes` через границы top-папок; lesson-узлов 6 (5 базовых + 1 Deep);
  рабочий-конспект вне лестницы; `lesson_floor_order` на новом снимке → 6 этажей,
  этаж 1 = «урок_1 Введение»; у узлов заполнен `course`; юнит-тест: два курса +
  посторонняя папка → цепочки не пересекаются.
- **Kill switch.** Потребовалось менять домен-схему бандла не-аддитивно, новое
  хранилище или LLM для порядка уроков — стоп (всё решается арифметикой путей).
- **Effort.** ~день. **Priority.** P0. **Dependencies.** нет.

## P0-2 «Библиотека — один экран»: каталог из готовых резолверов

- **Problem.** «Не потеряться в курсах» негде: конспекты видны только у активного
  курса, до материалов другого курса пути нет (только смена scope), поиска раздела
  нет. Владельцу нужен простой каталог всех курсов и конспектов.
- **Evidence.**
  - scope строго одно-папочный: `app/ui/study_scope.py:146-173`; опции курсов —
    top-папки: `app/course_cache.py:492-520`; «Все курсы» в UI не существует
    (grep по `app/ui` = 0);
  - конспекты перечисляет `scan_konspekts(course_dir)`
    (`app/konspekt_discovery.py:78`) — вызывается только для активного курса
    (`topics_tab`); паспорт качества и staleness уже готовы (№14);
  - разделы уже индексируются `section_index`; сквозной Q&A уже работает при
    пустом scope (`app/query_service.py:133` — `course_id` опционален).
- **Proposed.** Новый view «Библиотека» (FeatureSpec по образцу существующих):
  1) список курсов из `build_mission_control_course_options` (индексированные +
     кандидаты `needs_reindex`);
  2) раскрытие курса → конспекты этого курса через `scan_konspekts` с паспортом
     №14 и бейджем свежести — БЕЗ активации scope;
  3) раскрытие конспекта → разделы (`section_index`) со ссылкой «открыть в
     читалке»;
  4) действия: «Открыть конспект» / «Сделать активным курсом» (существующий
     `activate_scope`) / «Спросить» — префилл Q&A с фильтром выбранного курса или
     по всем (folder=None). 0 LLM, 0 новых хранилищ — чистая проекция.
- **Files.** новый `app/ui/library_catalog.py` (узкий view),
  `app/ui/feature_registry.py` (FeatureSpec + tile), переиспользуемые:
  `app/course_cache.py`, `app/konspekt_discovery.py`, `app/section_index.py` —
  только чтение.
- **DoD.** С одного экрана видны оба курса и все конспекты; до любого конспекта
  любого курса ≤ 2 кликов без смены активного курса; «Спросить по курсу Deep»
  отвечает с источниками из Deep; тест видимости view + тест «каталог не пишет»
  (никаких мутаций scope/state при просмотре).
- **Kill switch.** Появился собственный кэш/БД каталога или третий резолвер «что
  такое курс» — стоп (проекция двух существующих: пути индекса + файловая система).
- **Effort.** ~день. **Priority.** P0. **Dependencies.** нет (P0-1 не требуется,
  но вместе дают полный эффект).

## P1 Пересадки + адреса + агент-навигатор

- **Problem.** 8 слитых общих тем невидимы; остановки маршрута безадресны; агент
  не может выполнить «найди раздел про guardrails в Deep».
- **Evidence.** Общие темы: прямой запрос к `kg.sqlite` активной генерации — у 8
  концептов документы обоих курсов; маршрут: `select_day_route`
  (`knowledge_graph_d3_analysis.py:386`) уже межкурсовой, подписи курса нет;
  агент: 9 read-only инструментов без каталога (`app/agent/tools_*.py`, grep
  `course|scope|folder` = 0; `konspekt.inspect` смотрит workbench).
- **Proposed.** 1) Бейдж «встречается в N курсах» на карточке концепта (произвести
  из `documents` узла — арифметика готовых путей) + подсветка пересадки в зале;
  2) адрес «курс · урок» на остановках маршрута дня и в плане (поле `course` из
  P0-1); 3) read-only инструмент `catalog.list` агента (курсы → конспекты →
  разделы; та же проекция, что P0-2) + golden-кейс «найди раздел» в
  `eval_data/agent_scenarios_golden_v1.json`.
- **DoD.** Общая тема кликабельно показывает оба первоисточника; каждая остановка
  маршрута несёт адрес (North star = 100%); агент отвечает на «где про X в курсе Y»
  адресом раздела без галлюцинаций (golden-тест).
- **Effort.** дни. **Priority.** P1. **Dependencies.** P0-1 (адреса), P0-2
  (проекция каталога для инструмента).

## P2 Метрополитен и порядок области

- **Problem.** Мнемополис не различает линии-курсы визуально; порядок между
  курсами («Deep после базового») существует только в голове владельца.
- **Proposed.** 1) Линии метро в 3D-зале: курс = линия с цветом, пересадки = общие
  темы (vision №20, волна W4 — данные готовы после P0-1/P1); 2) необязательные
  метаданные порядка курсов от владельца (например, `course.yaml` в папке курса
  или выбор в каталоге; дефолт — «курсы независимы», связь только через
  пересадки); 3) межкурсовой план области: weekly/adaptive план получает
  course-метки шагов и умеет предлагать пересадку («эта тема закрыта в базовом —
  углубить в Deep»).
- **DoD.** Экспортный зал показывает 2 линии и ≥1 пересадку на реальных данных;
  порядок курсов, если задан, меняет только рекомендации, не рисует precedes.
- **Effort.** волна. **Priority.** P2. **Dependencies.** P0-1, P1.

## НЕ делать (вердикты разбора, зафиксированы)

1. **Второй «мега-граф» или хранилище каталога** — бандл уже один на все курсы;
   каталог — проекция файловой системы и индекса (третья копия истины — типовая
   болезнь серии №1/№2/№13).
2. **LLM для порядка курсов и границ** — границы даёт файловая система, порядок
   внутри курса — номера уроков, порядок между курсами — явное решение владельца.
3. **Мульти-scope ретривала** — сквозной поиск уже есть (пустой scope).
4. **Автослияние конспектов разных курсов в один текст** — слияние на уровне
   концептов уже работает; перегенерация текстов рушит якоря строк/сайдкары
   (вердикт №14).

**Правило границы:** `backlog_registry.yaml` не редактируется этим планом —
только кандидаты; промоут — решением владельца.

**Рекомендованный порядок:** P0-1 → P0-2 → P1 (бейдж → адреса → агент) → P2.
