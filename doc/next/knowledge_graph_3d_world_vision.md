## 1. Аудит качества текущего скина (живой прогон @270)

Memory Run parity состоялся: route-кадр, топбар, карточка остановки, интерьер,
onboarding, export-деградация — работают и узнаваемы. Ниже — что отделяет «хорошо»
от «шедевра». Все пункты воспроизведены на living-артефакте, не по диффу.

### P0 — дефекты композиции (видны каждому за первые 30 секунд)

- **Q1. Нижние иконки-кнопки перекрывают подпись оси «СЕГОДНЯ»** (desktop, tablet,
  mobile: видно «СЕГ…НЯ» за кнопками). Ось — часть ориентационного контракта №17,
  её нельзя закрывать. Fix: кнопки сдвинуть/подложка, или подпись рисовать выше линии
  кнопок.
- **Q2. Компас: «ВХОД» продублирован на двух лучах** (восток и юг), север = «КУРС».
  Оси компаса не считываются. Fix: одна семантика (КУРС / ВХОД / СЕГОДНЯ / ДАЛЬШЕ или
  N-E-S-W со сносками), без дублей.
- **Q3. Mobile 390×844: активная остановка №1 — за левым краем кадра** (виден только
  callout), стопы 5–6 — за правым; компас наполовину обрезан и конфликтует с ←/→;
  «ДАЛЬШЕ» под кнопками. `fitRouteCamera` не удерживает маршрут в узком кадре.
  Это нарушение UI DoD №18 («390×844 … canvas не пустой, без обрезки») — самый
  серьёзный пункт аудита.
- **Q4. Desktop: маршрут прижат к верхней половине кадра**, нижняя половина сцены —
  пустая чернота. Fit-камера не использует высоту кадра; в reference тропа заполняет
  перспективу. Fix: вертикальный centring/наклон камеры в `fitRouteCamera`.

### P1 — шероховатости, ломающие «шедевр»

- **Q5. Несогласованные стили действий:** «Внутрь» — кнопка, «Открыть раздел» —
  подчёркнутая ссылка. Одна пара вторичных действий — один стиль (обе как
  `kgx-link-btn`).
- **Q6. Интерьер: статус сцены просвечивает сквозь шапку «ВНУТРИ …»** (полупрозрачный
  верх панели), низ панели пустует — интерьер выглядит недостроенным. Fix: непрозрачная
  шапка; нижнюю зону займут связи/quiz-история (см. §7 W5).
- **Q7. Сервисный футер «12 узлов · маршрут 6/6 · режим route · embedded»** — строка
  для разработчика, а не для ученика. Убрать в `?`-диалог или title.
- **Q8. Зигзаг хвоста маршрута (4→5→6):** №6 выше и «за» №5, дуга ломается. Реальные
  данные не обязаны рисовать красивую кривую — значит кривую надо защищать рендером:
  монотонное сглаживание маршрутной полилинии (только презентация порядка 1→N, домен
  не трогается).
- **Q9. Мелочи статуса:** в «Созвездии» статус сцены остаётся «Сцена: маршрут дня»;
  кольцо прогресса при 0/6 неотличимо от отсутствия (нужен видимый трек).

### Примечания (не дефекты, но зафиксировать)

- Onboarding в embedded открывается всегда при первом открытии вкладки
  (`SHOW_ONBOARDING || isEmbedded`, `kg_3d_template.html:2472`) — хост не может
  подавить. Осознанно? Если да — задокументировать.
- Проба с рёбрами `{type: …}` вместо `{relation_type: …}` дала пустое «Созвездие» и
  «Связей нет» **молча**. Реальный payload корректен, но тихое проглатывание неизвестной
  схемы рёбер скроет будущий рефакторинг. Предложение: console.warn + счётчик
  «рёбер отброшено» в `?`-диагностике.

**Прорыв уровня качества = W0 (Q1–Q9) + W1 «Рассвет» (§8).** Полировка без новой
механики уже поднимет ощущение на класс; «Рассвет» добавит первый признак живого мира.

## 2. Название: «3D-зал» мир перерос

Кандидаты:

| Название | За | Против |
|---|---|---|
| **Мнемополис** (рекомендую) | город памяти; вмещает районы, линии курсов, этажи, туман, рассвет; уникальное слово — presence в UI и разборах; рус./лат. написание живут оба | нужно один раз объяснить (одной строкой в onboarding) |

| Город знаний | понятен сразу | генерик, не присвоить |

Иерархия имён: **Мнемополис** — весь мир; **Memory Run** — ежедневный забег (маршрут

историческим названием вкладки до промоута.

## 3. Принцип мирового класса: «мир — честная проекция твоей памяти»

Уникальность hometutor — сочетание, которого нет ни у кого из «геймификаторов»:
граф знаний + spaced repetition + quiz-история + RAG + **локальный** LLM.
Отсюда один закон мира, из которого следует всё остальное:

> **Каждый объект, свет, угроза и звук мира — проекция реального сигнала данных
> обучения. В Мнемополисе нет ни одной декорации «просто так».**

- Туман — это `decay_vector`. Рассвет — это прогресс дня. Фонарь — пройденный quiz.
  Артефакт ◆ — раздел в Живом конспекте. Стройплощадка — свежая индексация.
- Обратное тоже верно: **никакой выдумки**. Нет данных — нет объекта. Поэтому мир
  не может «наврать» и не обесценится как случайная игра.
- Всё работает офлайн и локально (local-first): и мир, и его ИИ-хранитель
  (provider layer, `LOCAL_STRICT`). Полная деградация без LLM обязательна.

Duolingo рисует сову поверх прогресс-бара; Мнемополис рисует **сам прогресс** как
город. Это и есть решение мирового класса, уникальное для приложения.

## 4. Карта районов: интеграция всех разделов приложения

Каждый район = существующий view из `app/ui/feature_registry.py`. Мир — это hub;
дверь района открывает view (механика `PENDING_CURRENT_VIEW_KEY` уже работает в
`start`-действии G1). Ничего нового в домене — только маппинг и образы.

| Район (образ) | View продукта | 3D-образ (canvas 2D) | Сигнал данных |
|---|---|---|---|
| **Врата / Вокзал** | Mission Control | портальная рама на входе тропы, табло «сегодня» | day_route, стрик |
| **Магистраль дня** | Адаптивный план | неоновая тропа Memory Run (есть) | worth, day_route |
| **Арена испытаний** | Интерактивный Quiz | амфитеатр с прожекторами; свет = последние score | quiz_results |
| **Оранжерея памяти** | Flashcards / SR | сад: карточка = растение, вянет по кривой забывания, оживает поливом-повторением | SR due, decay |
| **Кузница конспекта** | Живой конспект | верстак с ◆-артефактами; крафт конспекта из разделов | workbench |
| **Архив-башня** | Темы / Поиск / Объяснить файл | стеллажи-этажи уроков (26 этажей уже в сцене) | lessons, floors |
| **Обсерватория** | Прогресс / Метрики / Аналитика | башня: высота = уровень, огни = XP дня, витрины = бейджи | gamification_service |
| **Маяк Тьютора** | Чат с тьютором / Быстрый ответ | маяк; луч приглашает «спроси» | ask-pipeline |
| **Линии курсов** | Курс / Course Cockpit | линии метро сквозь районы; активный курс — светящаяся линия («курс →» уже на сцене) | active_course |
| **Летопись** | История | зал хроник; реплей недели (G4.2) | mastery_history |

Правила карты районов:
- районы появляются в режиме «Вся карта» и «Созвездие»; **первый route-кадр остаётся
  чистой тропой** (ориентация №17 неприкосновенна);
- район = дверь (1 клик → view) + состояние (свет/износ по данным) — не новая логика;
- у каждого района есть «обратный портал» (§5).

## 5. «Быстро и просто вернуться в 3D»

1. **Глобальная кнопка «🌆 В Мнемополис»** в сайдбаре на каждой странице (одна строка
   в `sidebar.py` через `PENDING_CURRENT_VIEW_KEY` → Knowledge Graph/3D). Мир становится
   home-экраном привычки.
2. **Возврат с трофеем:** после завершения квиза / сессии карточек — в итоговой карточке
   CTA «Вернуться в мир». На возврате мир **показывает изменение**: остановка зажигает
   ✓-фонарь, сегмент тропы зеленеет, туман отступает (данные уже приходят через
   payload/mastery_history при rerun — нужна только анимация дельты «было → стало»).
3. **Контекст сохраняется:** мир открывается на той остановке, откуда ушли
   (`selected_concept_id` уже в session-state механике G0).

Петля закрывается: мир → действие → результат → мир показал последствие. Это ядро
«живости» — мир реагирует на каждый поступок ученика.

## 6. ИИ-агент в мире: «Хранитель Мнемо» — все сценарии

Технический контракт для ВСЕХ сценариев (жёсткий):
- только через provider layer (`app/provider.py`), профили LOCAL_STRICT/BALANCED;
- LLM пишет **только текст/команды сцены**, никогда — в домен (mastery, SR, корзину);
- кэш по `(snapshot_date, concept_id, сценарий)` — повторный рендер не жжёт токены;
- обязательная деградация: без LLM каждый сценарий имеет статическую замену
  (мир полностью функционален офлайн);
- бюджет: ≤N вызовов на сессию, circuit-breaker уже есть в провайдере.

### A. Экскурсовод (нарративный Memory Run)
Вместо «Тур = автопролёт»: Хранитель ведёт забег с рассказом. Вход: day_route +
worth_reason + mastery + названия. Выход: 6 связок по 1–2 предложения («Начнём с
Study session agent — он для тебя новый, а RAG после него станет очевиднее…»).
Кэш на день. Деградация: текущие worth_reason (уже в карточке).

### B. Хранитель памяти (сводка угроз)
По `decay_vector` + SR due: «Туман подступает к RAG (забывание 42%) — 3 карточки,
~2 минуты». Кнопка → `start` flashcards. Выход LLM — только формулировка; список
угроз считается детерминированно из данных. Деградация: список без прозы.

### C. Собеседник в интерьере узла
В интерьере (U3) — поле «Спросить об этом»: вопрос уходит в существующий ask-pipeline
со scope=концепт (retrieval по его разделам), ответ с источниками рендерится в
интерьере. Это главная дверь: из игрового мира — в ядро продукта (RAG-тьютор).
Деградация: кнопка ведёт в «Чат с тьютором» с преселектом темы.

### D. Квестмейстер
Оборачивает маршрут дня в микро-квест: одна строка цели утром («Сегодня — мост от
retrieval к агентам: 6 остановок, финал в Петле памяти») + существующие бейджи
как награда (никакой новой валюты). Деградация: «N из M» как сейчас.

### E. Архитектор достройки
После ingestion новых документов новые концепты появляются как **стройплощадка**
(каркас/леса вместо узла). Хранитель предлагает: к какому кварталу отнести, какие
связи заметны (текстовое предложение, человек подтверждает действием в существующих
инструментах). Мир растёт вместе с базой знаний — видимая причина её пополнять.
Деградация: каркасы без предложений.

### F. NL-управление миром (scene-DSL)
Командная строка мира: «покажи всё про агентов», «где я слабее всего», «маршрут
повторения на 10 минут». LLM переводит запрос в **scene-DSL** — JSON строго из
view-команд: `{filter, focus, scene_mode, overlay, route_override}`. Сцена исполняет;
неизвестные команды отбрасываются. Схема валидируется как action-envelope G0
(version/source/whitelist). Домен недостижим по построению — DSL не содержит
пишущих команд. Деградация: без строки, существующие кнопки.
`route_override` — презентационный (подсветить путь), доменный day_route не меняет.

### G. Летописец
Недельная хроника: реплей `mastery_history` по сцене (G4.2) + 3–4 предложения от
Хранителя («отстроено 4 квартала, туман забрал Guardrails — вернись туда») + «фото
недели» (G4.3, privacy DoD из плана-предшественника обязателен). Деградация: реплей
без текста.

### H. Голоса антагонистов
Короткие реплики Тумана/Призрака/Разлома (§7) — по 1 строке, генерятся пачкой на
день и кэшируются. Тон: уважительный юмор, **никогда не стыдящий** («Я подожду. Я
всегда жду» — туман). Деградация: фиксированный набор реплик в шаблоне.

## 7. Антагонисты: честные опасности мира

Закон §3 распространяется на врагов: **антагонист = визуализация реального врага
обучения**, не случайный монстр. Ровно три (больше — шум):

### Туман Забвения (враг №1 — кривая забывания)
- **Данные:** `decay_vector` (уже в шаблоне с G2).
- **Образ:** серая дымка над кварталами с высоким decay; **подпись узла теряет
  буквы** по мере забывания («Guardrails» → «Gu…rd…s») — самый честный хоррор
  этого мира; radial-gradient + альфа, canvas 2D.
- **Опасность:** окутанная остановка «не видна» — вход требует развеять туман.
- **Развеять:** мини-повтор — 2–3 SR-карточки концепта (`start` flashcards с
  преселектом). Это retrieval practice, педагогически честно.
- **Не блокирует:** кнопка «войти сквозь туман» всегда доступна (угроза = приглашение,
  не запрет).

### Призрак Уверенности (враг №2 — иллюзия знания)
- **Данные:** давний ✓ (старое quiz-событие) + высокий decay = «ты уверен, что
  помнишь?».
- **Образ:** полупрозрачный двойник узла с ✓-отблеском, дрожащий контур.
- **Опасность:** стоит на тропе перед остановкой, «выдаёт себя за пройденное».
- **Развеять:** 1 контрольный вопрос через существующий quiz-пайплайн (результат —
  честное quiz-событие в домене, как обычно; LLM домен не трогает).

### Разлом (враг №3 — незакрытые пререквизиты)
- **Данные:** `precedes`-рёбра в непройденные концепты (уже в EDGES).
- **Образ:** трещина в тропе перед остановкой; мост достраивается прохождением
  prereq-квиза.
- **Не блокирует:** «перепрыгнуть» можно всегда — но теперь ученик видит, **почему**
  тяжело: порядок знания стал осязаемым.

Правила дозировки (kill switch ориентации жив):
- в **первом route-кадре** антагонистов нет — максимум тихие маркеры на остановках;
  полные образы живут в «Созвездии», «Вся карта» и интерьере;
- одновременно активна максимум одна «сценка» угрозы;
- `prefers-reduced-motion` — все антагонисты статичны;
- тумблер «Спокойный мир» выключает антагонистов целиком (доступность/вкус);
- каждая угроза конвертируется в конкретное действие ≤2 минут — иначе её нельзя
  показывать.

## 8. Игровые элементы мотивации (витрина первой валюты, не вторая)

XP, уровни, daily/quiz-стрики и бейджи **уже существуют**
(`app/gamification_service.py`). Мир их не дублирует и не вводит новые — он их
**материализует**:

- **Рассвет** (ключевой образ): цвет неба мира = прогресс дня. 0/6 — глубокая ночь;
  каждый пройденный стоп — небо теплеет; 6/6 — рассветное золото над городом.
  Один градиент, ноль новых данных, максимум эмоции. Экран «6 из 6» становится
  наградой сам по себе.
- **Фонари:** пройденная остановка зажигает фонарь навсегда (в рамках снимка дня) —
  тропа за спиной светится, виден «след труда».
- **Отстроенный квартал:** все концепты урока ≥80% mastery — портальная рама района
  загорается. Долгосрочная цель без таймеров и монет.
- **Стрик = непрерывность света:** daily_streak подсвечивает магистраль N ночей
  подряд; пропуск — свет гаснет (но ничего не «теряется» — punish-механик нет).
- **Обсерватория:** высота башни = уровень (Apprentice → …), витрины = бейджи.
- **Реплей недели + фото дня** (G4) — то, чем хочется поделиться.

Запрещено (без изменений): монеты, магазин, лидерборды, «жизни», потеря прогресса.

## 9. 3D-образы режимов — показать сразу

| Режим/оверлей | Образ | Реализация (canvas 2D) |
|---|---|---|
| Маршрут | **Тропа рассвета** — чистая тропа, небо по прогрессу | градиент фона от `done/route.length` |
| Созвездие | **Квартал под фонарём** — конус света над активным домом + соседи | radial light + приглушение вне конуса |
| Вся карта | **Город с обзорной башни** — террасы этажей, линии курсов как метро света | существующие floors + подсветка рёбер курса |
| След памяти | **Аврора** — золотая лента вчерашнего пути в небе над тропой | текущий trace, поднятый и размытый |
| Дозор (новый) | **Туман** — decay-слой поверх любого режима | §7, overlay как memory_overlay |
| Стройка (новый) | **Леса** — каркасы новых концептов после ingestion | пунктирный контур узла + ромб-кран |
| Хроника | **Ускоренные сутки** — реплей недели: закат→ночь→рассвет | G4.2 + образ рассвета |

Всё — без WebGL: градиенты, альфа, свечения; reference уже доказал достаточность
canvas 2D.

## 10. Ревизия kill switch (v3) — что меняется и что нет

| Правило | Было (v1/v2) | Стало (v3) |
|---|---|---|
| Доменная схема payload | заморожена | **заморожена** (без изменений) |
| Новые хранилища | стоп | **стоп** |
| Вторая валюта | стоп | **стоп** (мир — витрина первой) |
| Первый route-кадр | ≤8 подписей, без сущностей | **неприкосновенен**; антагонисты/районы — не в нём |
| LLM-вызовы | стоп | **разрешены** по контракту §6: provider layer, кэш, бюджет, деградация, read-only для домена |
| Частицы/декор | запрещены | запрещён **бессмысленный** декор; атмосфера обязана кодировать данные (туман=decay, небо=прогресс) |
| Three.js/WebGL | стоп | стоп, пока canvas 2D достаточен (пересмотр — отдельным решением владельца) |

Ослабление LLM-запрета — главное решение этой ревизии: мир без Хранителя остаётся
скином, с Хранителем становится продуктовой петлёй.

## 11. Волны

| Волна | Содержание | Приоритет | Effort |
|---|---|---|---|
| **W0 Полировка** | Q1–Q9 из §1 (fit-камеры, компас, ось, стили, интерьер, футер, сглаживание тропы) | P0 | день |
| **W1 Рассвет и фонари** | небо по прогрессу + фонари пройденных (§8) — максимум эффекта из имеющихся данных | P0 | день |
| **W2 Туман Забвения** | первый антагонист (decay уже в шаблоне) + «развеять» → flashcards + тумблер «Спокойный мир» | P1 | 1–2 дня |
| **W3 Хранитель: A+B** | экскурсовод + сводка угроз, контракт §6 (кэш/бюджет/деградация) — первые LLM-сценарии | P1 | волна |
| **W4 Районы и порталы** | карта районов (§4) в local/all + «🌆 В Мнемополис» из сайдбара + возврат с трофеем (§5) | P2 | волна |
| **W5 Интерьер-собеседник + NL** | сценарии C (ask в интерьере) + F (scene-DSL) | P2 | волна |
| **W6 Призрак, Разлом, Летопись, Стройка** | антагонисты 2–3 + G (хроника, G4-линия) + E (архитектор) | P3 | волна |

Каждая волна самоценна; мир не ломается, если следующая не случится. Gate V2′
(structural + viewport + live DOM) распространяется на каждую волну; DoD любой
UI-волны — живой прогон running-артефакта.

**Метрики уровня «мирового класса».**
- От открытия мира до первого действия (start/collect/развеять) — ≤10 секунд.
- Возврат в мир после квиза/карточек — норма, а не исключение (петля §5 замкнута).
- Завершаемость Memory Run (6/6 → рассвет) растёт после W1.
- Инвариант уникальности: **ни одного элемента мира без сигнала данных** (§3) —
  проверяется на каждом ревью волны.

**Не делать.**
- Аватар игрока, открытый мир-бродилка, экономика/магазин, punish-механики.
- Антагонисты, которые блокируют контент или стыдят.
- LLM-запись в домен; LLM без кэша/бюджета/деградации.
- Новые режимы навигации сверх `route|local|all` + оверлеи.
- Второй источник правды для прогресса (всё — из payload/gamification_service).
