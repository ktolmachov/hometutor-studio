# Lecture presence: candidate plan («Маршрут лекции» — чекпоинты во время урока)

**Источник:** эволюционный разбор №19
(`doc/presentations/evolutionary_analyses/19_lecture_presence.html`), 2026-07-18,
тема владельца «сложное — простыми словами, скучное — не усыпляет».

**Evidence-база и её свежесть.** Проверено на hometutor @ `b9eee3ef3` «287»
(2026-07-18), рабочее дерево чистое. Все `файл:строка` ниже — актуальный HEAD;
при реализации сверять по имени функции, не по номеру строки.

**Тезис разбора.** Продукт — превосходный собеседник ПОСЛЕ лекции (scaffolding,
CTA «Объясни проще», 9 quiz-модулей, MotivationCoach) и пустое кресло ВО ВРЕМЯ
неё: присутствие на 123-минутном уроке — один бит `mark_listened`
(`app/ui/living_konspekt_media.py:356-366`, «no effect on mastery»). Таймкоды
`.media.json` работают только как двери «в лекцию» (конспект → минута, карточка →
минута), дверей «из лекции в петлю» — ноль: grep `t_start` по всем девяти
quiz-модулям — 0 совпадений. Ход: **познакомить две готовые подсистемы** —
clipped-плеер, который уже умеет останавливаться на границе раздела
(`st.audio(start_time, end_time)`, `living_konspekt_media.py:396-399`), и
scoped-квиз, который уже умеет спрашивать по ограниченному тексту без retrieval
(`generate_scoped_quiz_from_content`, `app/quiz_scoped.py:182`). Позиция студента
известна по конструкции (отрезок сам остановился), телеметрия не нужна.

**Данные-эталон.** Реальный урок_2
(`DATA_DIR/ИИ Агенты/урок_2_как_агент_думает_и_действует.media.json`,
schema_version 1): 120 разделов, 75 с `t_start/t_end`, 59 semantic-блоков,
длительность ~123 мин. Разметка до слайда («Слайд 9: Tokenizer» = 284–673 сек).

**North star (одна на весь план):** «глубина лекции с подтверждением» — доля
таймкодированных разделов урока, закрытых воротами (правильный ответ после
отрезка). Сейчас неизмерима (плеер глух, вопросов к минуте не существует);
цель P0 — метрика существует и у урок_2 достижима медиана ≥ 70%.

## P0-1 «Маршрут лекции»: отрезки + ворота

- **Problem.** Студент засыпает к середине лекции и не может пересказать вторую
  половину; петля вовлечения (quiz/CTA/override) стартует только после лекции,
  когда студент сам приходит с вопросом.
- **Evidence.**
  - Присутствие во время лекции = 1 бит: `app/ui/living_konspekt_media.py:356-366`
    (`mark_listened` при клике «🎧 Слушать раздел», далее глухие
    `st.video`/`st.audio` — `:379`, `:397`);
  - вопросов к минуте не существует: `t_start` не принимает ни один из
    `quiz_service/quiz_micro/quiz_adaptive/quiz_scoped/quiz_parse/quiz_stats/
    quiz_micro_receipt/user_state_quiz/living_konspekt_scoped_quiz` (grep = 0);
  - скука распознаётся только по явному «скучно» в чате:
    `app/prompts/_impl.py:1007` (EMOTIONAL OVERRIDE), ярлык профиля
    `app/ui/learner_profile_panel.py:75-81`.
- **Proposed.** Режим «Пройти лекцию» в Living Konspekt поверх существующих rows:
  1) группировка смежных разделов с `t_start` в отрезки ~8–12 мин — чистая
     арифметика по готовым `t_start/t_end` (разделы без таймкода и
     `low_confidence` — честно вне маршрута, как в
     `app/ui/living_konspekt_next_steps.py:122`);
  2) отрезок играет через существующий `_render_local_audio_player`
     (`living_konspekt_media.py:382-399`) с `start_time/end_time` границ отрезка —
     плеер сам останавливается, позиция известна по конструкции;
  3) ворота: 1–2 вопроса из `generate_scoped_quiz_from_content`
     (`app/quiz_scoped.py:182`; контекст — текст разделов отрезка по
     `line_start/line_end` из сайдкара, ≥120 символов гарантированы размером
     раздела), `adaptive_level` — из существующего маппинга
     (`app/quiz_adaptive.py:223-227`);
  4) результат — в существующий quiz-стек (запись как у scoped-квиза, XP через
     `scoped_quiz_xp_reward`, `app/quiz_scoped.py:105`), прогресс прохождения —
     `st.session_state` в рамках сессии.
- **Files.** `app/ui/living_konspekt_media.py` (или новый узкий
  `app/ui/living_konspekt_lecture_route.py` рядом), `app/quiz_scoped.py`
  (параметризация без изменения контракта), `app/media_sidecar.py` — только
  чтение; новых схем/хранилищ нет.
- **DoD.** На урок_2 виден маршрут из отрезков; отрезок играет и останавливается
  на границе; после остановки — ворота; правильный ответ открывает следующий
  отрезок и начисляет XP; разделы без таймкодов показаны как «вне маршрута»;
  тест на группировку отрезков (чистая функция, без LLM) + тест «ворота получают
  текст именно этого отрезка».
- **Kill switch.** Потребовался кастомный JS-плеер/телеметрия позиции — стоп;
  потребовалось новое хранилище или офлайн-предгенерация вопросов — стоп;
  LLM для группировки отрезков — стоп (арифметика готовых t_start).
- **Effort.** ~день. **Priority.** P0. **Dependencies.** нет (все кирпичи в HEAD).

## P0-2 «Проще + переслушать» с адресом

- **Problem.** «Объясни проще» спасает точку, но не отрезок: кнопка
  (`app/ui/query_tab_answer_section.py:194-196`) шлёт «Объясни проще и на
  интуитивном уровне» про последний ответ чата — без привязки к месту лекции,
  где студент потерялся.
- **Evidence.** CTA-стек сквозной, но безадресный: приоритет №1 в промпте
  (`app/prompts/_impl.py:677`), fallback-наборы
  (`app/tutor_orchestrator.py:461,465`), рекомендация после провала квиза
  (`app/learning_plan_adaptive.py:206-212`). Обратный переход «в минуту» уже
  реализован в карточках: `app/ui/flashcards_review_view.py:756,818`
  (`start_time=citation.start_seconds`).
- **Proposed.** При провале ворот P0-1 показывать два действия:
  1) «Объясни проще» — существующий CTA, но с контекстом: текст разделов отрезка
     (по `line_start/line_end`) уходит в запрос тьютора;
  2) «Переслушать отрезок» — повторный clipped-плеер с теми же
     `t_start/t_end` (кирпич из `flashcards_review_view.py:818` уже показывает,
     как переиспользовать плеер вне конспекта).
- **Files.** тот же UI-модуль маршрута из P0-1; `app/ui/query_tab_answer_section.py`
  не трогается (черновик вопроса формируется в модуле маршрута).
- **DoD.** Провал ворот → обе кнопки видимы; «проще» приходит тьютору с текстом
  отрезка (проверяется тестом на состав черновика); «переслушать» открывает плеер
  с границами отрезка.
- **Kill switch.** Как у P0-1; плюс: если «проще с адресом» потребовал менять
  контракт пайплайна/оркестратора — стоп, достаточно черновика вопроса с текстом.
- **Effort.** часы. **Priority.** P0. **Dependencies.** P0-1.

## P1 Ставка до отрезка + глубина в user_state

- **Problem.** Ворота после отрезка проверяют, но не создают ставку ДО него;
  North star-метрика живёт только в session_state и умирает с сессией.
- **Evidence.** predict-then-listen — противоречие «лёгкость ↔ усилие» разбора:
  трудный вопрос до фрагмента = generation effect; лёгкое восстановление после
  провала уже кодифицировано (level_downgrade, `app/prompts/_impl.py:1017`).
  Персистентность — только через хелперы `app/user_state*.py` (конвенция).
- **Proposed.** 1) Перед отрезком — один вопрос-предсказание (recognition,
  существующий micro-quiz механизм с `{mastery_level}`,
  `app/prompts/_impl.py:1127`); тот же вопрос входит в ворота после отрезка.
  2) «Глубина лекции с подтверждением» пишется через user_state-хелперы и видна
  на дашборде прогресса. 3) Несданные ворота → карточки существующим handoff
  (механизм №1 B1, `app/flashcard_handoff.py`).
- **DoD.** Метрика глубины переживает рестарт; несданные ворота наутро в SM-2.
- **Effort.** дни. **Priority.** P1. **Dependencies.** P0-1.

## P2 Скука без признания + связка с маршрутом дня

- **Problem.** Скука распознаётся только по слову «скучно» в чате
  (`app/prompts/_impl.py:1007`) — засыпающий студент в чат не пишет.
- **Proposed.** Два провала ворот подряд = поведенческий сигнал скуки →
  существующий override-канал (MotivationCoach предлагает сменить формат:
  карточки / 3D-зал / перерыв). Шапка маршрута лекции объясняет «зачем этот урок
  сегодня» из worth (`app/ui/dashboards_graph.py:1846,1912-1913`).
- **DoD.** Сигнал не требует новых слов от студента; предложение смены формата
  появляется в момент второго провала.
- **Effort.** волна. **Priority.** P2. **Dependencies.** P0-1, P1.

## НЕ делать (вердикты разбора, зафиксированы)

1. **Кастомный плеер с телеметрией позиции** — новый JS-компонент/канал ради
   данных, которые clip-конструкция даёт бесплатно (типовой kill switch).
2. **Realtime-детекция скуки** (паузы/скорость/камера) — слежка вместо ставки.
3. **LLM-вставки «оживляжа» в материал лектора** — перегенерация нарушает
   вердикт №14 (слои поверх, не вместо).
4. **Офлайн-предгенерация вопросов ко всем разделам** — новое хранилище и цикл
   инвалидации; scoped-квиз on-demand достаточен и всегда свеж к mastery.
5. **«Сократи лекцию» / авто-выжимка** — сжатие уже есть (конспект); боль не в
   длине, а в отсутствии ставки.

**Правило границы:** `backlog_registry.yaml` не редактируется этим планом —
только кандидаты; промоут — решением владельца.
