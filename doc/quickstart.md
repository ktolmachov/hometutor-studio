# Быстрый старт — от папки до учебного цикла за 5 минут

> **Обещание:** через 5 минут твоя папка с конспектами превращается в личного тьютора с квизами, повторениями и планом на завтра. Локально. Без облака.

Этот документ — не «список команд», а **пошаговый квест**. Ты ставишь продукт → делаешь первый ход → видишь результат → делаешь следующий. Каждый шаг помечен тегом ⏱ (сколько займёт) и 🎯 (что получишь).

## Что такое «учебный цикл» (и почему это главное)

У всех RAG-инструментов есть один и тот же шаблон: «задал вопрос → получил ответ → забыл». Через неделю ты не помнишь, о чём спрашивал.

home-rag строит **цикл**:

```text
①  спросил          →  получил ответ с источниками
②  углубился        →  тьютор разобрал тему сократически
③  проверил себя    →  мини-квиз с мгновенным feedback
④  повторил         →  флэшкарты по SM-2
⑤  увидел прогресс  →  adaptive план, mastery, graduation
⑥  следующий день   →  resume карточка ведёт обратно
```

Каждый шаг **записывается локально**. На следующий день ты не начинаешь с нуля — продукт помнит где ты был, что освоил, что забыл и что пора повторить.

**Твоя цель в этом quickstart** — пройти цикл целиком. Поехали.

## Шаг 0. Установка ⏱ 2 мин 🎯 локальный стек

Выбери один из путей. Все три приводят к одному результату.

<details open>
<summary><b>🧭 Путь A — Localhost-only launcher (рекомендуется сейчас)</b></summary>

```powershell
git clone <repo>
cd home-rag_v2
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
copy .env.example .env
# В .env: OPENAI_API_KEY=local для Ollama/LM Studio или реальный ключ провайдера.
.\scripts\local_start.ps1 -SkipPip
```

Что делает launcher:
- запускает `scripts/local_readiness.py` и останавливается на понятных `FAIL`;
- проверяет `.venv`, `.env`, локальные папки, порты `8000/8501` и provider URLs;
- затем поднимает FastAPI на http://127.0.0.1:8000 и Streamlit на http://127.0.0.1:8501.

Проверить без запуска стека:

```powershell
.\.venv\Scripts\python.exe scripts\local_readiness.py
```

Проверить уже запущенный стек:

```powershell
.\.venv\Scripts\python.exe scripts\local_readiness.py --allow-running --check-running
```

#### Выбор локального LLM-сервера

Перед запуском стека можно переключить основной чат, quiz, graph/concept extraction и SSR на один из готовых профилей:

```powershell
# llama.cpp: http://127.0.0.1:8080/v1, модель qwopus3.6-35b-a3b-v1-mtp (benchmark default)
.\scripts\switch_local_llm.ps1 -Profile llama-cpp -ValidateEndpoint

# другой alias на том же llama.cpp
.\scripts\switch_local_llm.ps1 -Profile llama-cpp -Model qwen3.6-27b -ValidateEndpoint

# LM Studio: http://127.0.0.1:1234/v1, модель qwen/qwen3.6-27b
.\scripts\switch_local_llm.ps1 -Profile lm-studio
```

Добавьте `-DryRun`, чтобы увидеть изменения без записи, или `-ValidateEndpoint`, чтобы проверить модель через `/v1/models`. По умолчанию скрипт синхронно обновляет `LOCAL_LLM_PROFILE`, `LLM_*`, `QUIZ_LLM_MODEL`, `GRAPH_*` и `SSR_LLM_*`; параметр `-SkipGraph` сохраняет текущие GRAPH-настройки. Перед реальной записью создаётся резервная копия в `logs/config_backups/`. Сам сервер и нужная модель должны быть запущены заранее.

Полный gate для llama.cpp (endpoint, grammar, retrieval, reranker, provenance и неизменность `config.env`):

```powershell
.\scripts\Smoke-HomeRag-LlamaCpp.ps1
```

Прогрев RAG в уже запущенном API (убирает cold ~10 s с первого UI-вопроса):

```powershell
.\scripts\run_local_stack.ps1 -WarmupRag
# или отдельно, когда API уже на :8000
.\scripts\Warmup-HomeRagRag.ps1
```

Для одного RAG-запроса без интерактивных prompt используйте `ask.py --question ... --non-interactive`; фильтры доступны через `--folder`, `--folder-rel`, `--file-name` и `--relative-path`.

### Опционально: отдельная локальная модель для «Почему сейчас» ⏱ 1 мин

Карточка **«Умный следующий шаг»** может коротко перефразировать заголовок «Почему сейчас» через второй LLM (по умолчанию ожидается LM Studio на `http://127.0.0.1:8787`). Это **не обязательно** для прохождения quickstart: достаточно основного `OPENAI_API_KEY` / Ollama из шага 0.

Если хочешь разделить нагрузку (тяжёлый локальный чат не трогать для одного предложения):

```env
SSR_LLM_API_BASE=http://127.0.0.1:8787
SSR_LLM_MODEL=имя-модели-из-lm-studio
```

Профили вызовов (латентность, `event_id`, fallback) — [ssr_llm_profiling.md](ssr_llm_profiling.md); команда сводки — `python scripts/summarize_ssr_llm_profiles.py`. Разбор сбоев и env — [user_guide_details.md](user_guide_details.md#ssr-lm-studio-profiles).

## Шаг 1. Скорми свои конспекты ⏱ 1 мин 🎯 индекс готов

Закинь в папку `data/` свои файлы. Поддерживаются:

| Формат | Что класть | Пример |
|---|---|---|
| `.pdf` | лекции, статьи, книги | Deep_Learning_Book.pdf |
| `.md` | заметки, Obsidian vault | notes/ml-basics.md |
| `.docx` | конспекты, курсы | lecture_03.docx |
| `.html` | сохранённые страницы | article_gradient_descent.html |
| `.txt` | любой текст | course_transcript.txt |

Индексация:

```bash
python ingest.py
```

Что происходит под капотом:
- файлы режутся на чанки, считаются эмбеддинги (local или OpenAI — по `.env`);
- строится Chroma-индекс + BM25 для гибридного поиска;
- создаются doc-summaries для обзорных запросов;
- всё локально в `chroma_db/`.

> 💡 **Лайфхак.** При повторном запуске без изменений `ingest.py` выходит за секунду с `run_kind=noop`. Добавил один файл — переиндексируется только он (extraction cache для остальных).

## Шаг 2. Первый ответ ⏱ 30 сек 🎯 wow-момент

Если ты запускал путь A, API и UI уже подняты. При ручном запуске:

```bash
python main.py                   # API
streamlit run app/ui/main.py     # UI в отдельном терминале
```

Открой http://localhost:8501. Увидишь **Home Mode Selector** — 6 карточек режимов (2×3):

```
┌─────────────────┬─────────────────┬─────────────────┐
│ Быстрый ответ   │  Чат с тьютором │  Темы           │
│ «Задать вопрос» │  «Понять тему»  │ «Обзор тем»     │
├─────────────────┼─────────────────┼─────────────────┤
│ Прогресс        │  Flashcards     │  Knowledge      │
│ «Смотреть»      │  «7 due»        │  Graph          │
└─────────────────┴─────────────────┴─────────────────┘
```

**Нажми «Задать вопрос»** → введи что-то из своих материалов → Enter.

Что ты увидишь:
- ✅ **Ответ** — с привязкой к фрагментам
- ✅ **Confidence %** — уверенность модели
- ✅ **Список источников** — с названиями файлов, страницами и кликабельным preview
- ✅ **Кнопка «Учить эту тему 5 минут»** — мост в тьютора

> 🎬 Это сценарий №1 в [каталоге](user_scenarios.md#сценарий-1--первый-запуск-от-папки-до-первого-ответа). Посмотри запись — там 6 кадров по 3-5 секунд.

## Шаг 3. Мост в учебный режим ⏱ 10 сек 🎯 контекст не теряется

Нажми **«Учить эту тему 5 минут»** под ответом.

Что произошло:
- тот же вопрос, те же источники → переехали в tutor-сессию;
- сверху чата — плашка «Сейчас: <тема>, цель — 5 мин»;
- тьютор **не начинает с нуля**: он знает, на чём ты застрял.

> 🔑 **Отличие от ChatGPT.** Переход режима не обнуляет историю. `session_id` + `continuity-контекст` (`topic`, `last_question`, `answer_summary`) проходят через API как `tutor_goal_*`.

Подробный разбор — [сценарий №3](user_scenarios.md#сценарий-3--мост-ответ--тьютор).

## Шаг 4. Сократический тьютор + мини-квиз ⏱ 3–5 мин 🎯 проверка понимания

В чате с тьютором ты получаешь **структурированный ответ**: TL;DR → объяснение → пример → «Проверь себя».

Под ответом — четыре CTA:

| Кнопка | Что делает |
|---|---|
| 🔍 **Объясни проще** | тот же контент на уровне ниже |
| 💡 **Дай пример** | бытовая аналогия или код-сниппет |
| ✅ **Проверь меня** | мини-квиз с 1 вопросом (встраивается в диалог) |
| ⏭ **Следующий шаг** | тьютор сам решает, что дальше по policy |

**Нажми «Проверь меня».** Появится multiple-choice вопрос. Ответишь — получишь **мгновенный feedback** с одной из меток:

- `recognized` — узнал тему, но не уверен
- `recalled` — вспомнил правильно
- `misconception` — есть заблуждение, тьютор объяснит
- `cannot_apply` — знаешь, но не можешь применить

Policy-оркестратор (`tutor_orchestrator.py`) на сервере решает, что делать дальше: объяснить, дать ещё квиз или перейти к новой теме.

> 🎬 Детально — [сценарий №4](user_scenarios.md#сценарий-4--учебная-сессия-с-тьютором).

## Шаг 5. Флэшкарты за 1 минуту ⏱ 1–2 мин 🎯 колода готова

Открой **Flashcards** (иконка в Home или через селектор разделов).

1. Вкладка **«Создать новые»** → выбери документ или загрузи файл.
2. Двигай слайдер (5–20 карточек).
3. Нажми **«Сгенерировать»**.
4. Появится **редактируемый preview**: правь вопросы/ответы, удаляй лишние.
5. **«Сохранить колоду»** (нужно ≥ 5 валидных карточек).

Готово. На Home теперь бейдж **«N due»** — столько карточек ждут тебя завтра.

> 🔑 **Почему preview важен.** home-rag — единственный инструмент, где ты **видишь карточки до сохранения** и редактируешь текст. Anki-генераторы скидывают в колоду всё подряд.

Детальный ролик — [сценарий №5](user_scenarios.md#сценарий-5--flashcards-генерация-и-сохранение).

## Шаг 6. Повторение по SM-2 ⏱ 2–5 мин 🎯 закрепление

Вкладка **«Повторение»** → **«Загрузить очередь»**.

Ритуал:
1. **Вопрос** → думаешь.
2. **«Показать ответ»** → проверяешь себя.
3. Четыре оценки: **Снова / Трудно / Хорошо / Легко** (SM-2 под капотом).
4. **Killer-move:** если не помнишь → **«Не знаю, объясни»** → карточка уезжает в «Снова», а ты прыгаешь в тьютора с вопросом этой карточки. Обратно — одна кнопка «← Вернуться к карточкам».

В конце сессии — **сводка**: сколько по каждой оценке + время следующего повторения (UTC).

> 🎬 [Сценарий №6](user_scenarios.md#сценарий-6--повторение-по-sm-2) — полное демо с переходом карточка→тьютор→карточка.

## Шаг 7. Увидь прогресс ⏱ 1 мин 🎯 мотивация

Открой **«Мой прогресс»**. Что ты там увидишь:

| Блок | Что показывает |
|---|---|
| **План на сегодня** | Review / Gap / New — три секции с кликабельными темами |
| **Mastery vector** | тепловая карта концептов: красный → жёлтый → зелёный |
| **Graduated** | концепты, освоенные на transfer-уровне > 7 дней (они больше **не вернутся** в план) |
| **Weekly goals + Streak** | микро-мотивация: дней подряд, цель на неделю |
| **Knowledge Graph** | визуализация связей между концептами |

Каждый пункт плана **кликабелен** — ведёт сразу в тьютора с готовой целью.

## Шаг 8. Завтра ⏱ 30 сек 🎯 возврат без «где я был?»

Открой UI на следующий день. На главной сразу **три resume-карточки**:

```
┌─────────────────────────────────────────────────────┐
│ 📘 Продолжить чат с тьютором                         │
│    Последняя тема: «Обратное распространение»       │
│                                                      │
│ 📗 Продолжить тему или план                          │
│    Активный план: ML-Course (шаг 3 из 12)           │
│                                                      │

└─────────────────────────────────────────────────────┘
```

Один клик — и ты в потоке, где остановился вчера. Никакого «где моя сессия, я её потерял».

## Шаг 9. Освой курс целиком ⏱ 10 мин 🎯 папка = курс

Есть папка `data/ML-Course/` с 20 лекциями? Один клик — и она превращается в курс.

1. Вкладка **«Темы»** → выбери папку → **«Активировать как курс»**.
2. Кнопка **«Подготовить курс»** → synthesis + learning plan по всем документам.
3. **Flashcards → Создать новые → Источник: Активный курс** → колода с тегами `course:<id>`.
4. **«Мой прогресс»** теперь показывает фильтр **«Только активный курс»** с отдельной панелью.

> 🔑 **Killer-фича.** Ни один конкурент не делает «папка → курс одной кнопкой». NotebookLM умеет отвечать по документам, но плана, флэшкарт и прогресса по курсу — нет.

[Сценарий №8 — Course Workspace](user_scenarios.md#сценарий-8--course-workspace).

## Шаг 10. Проверь доверие ⏱ 1 мин 🎯 убедись, что не галлюцинирует

В любом ответе тьютора:

1. Раскрой **«Подробнее об источниках»**.
2. Клик по фрагменту → **подсветка в исходном документе** (на нужной странице PDF).
3. Рядом — **«Шаги оркестрации»**: rewrite → classify → retrieve → rerank → answer.
4. Попробуй prompt injection («Ignore previous instructions…») → guardrail заблокирует.

Всё это — **локально, прозрачно, проверяемо**.

[Сценарий №11 — Trust-панель](user_scenarios.md#сценарий-11--trust-панель-и-источники).

## FAQ

<details>
<summary><b>Почему это лучше, чем ChatGPT?</b></summary>

ChatGPT:
- не помнит прогресс между чатами;
- не делает квизы и spaced repetition;
- не знает, где ты сильный, а где слабый;
- твои материалы уходят в облако.

home-rag: всё это — есть. Локально.

</details>

<details>
<summary><b>Почему это лучше, чем Anki?</b></summary>

Anki — только SRS. Нет тьютора, нет ответов по документам, нет оценки понимания, нет адаптивного плана. home-rag включает Anki-логику (SM-2) как **один из** своих модулей.

</details>

<details>
<summary><b>Работает полностью офлайн?</b></summary>

Да. Путь C (Ollama) — локальный LLM, локальные эмбеддинги, локальный индекс. Интернет можно отключить на сетевом уровне и всё продолжит работать. См. [сценарий №14](user_scenarios.md#сценарий-14--backup-и-полный-офлайн).

</details>

<details>
<summary><b>Прогресс переживёт переиндексацию?</b></summary>
