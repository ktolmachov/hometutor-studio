# US-19.5 — Home Hub Visual Hierarchy

**Epic**: Epic 19: UX Breakthrough Wave  
**Priority**: P2  
**CJM Stage**: #13 Home mode selection  
**Status**: open  
**Covered by**: ux-home-hub-navigation-polish (planned)

## User Story

**Как** Learner,  
**Я хочу** видеть чёткую визуальную иерархию на главном экране,  
**Чтобы** быстро находить нужный режим и понимать приоритеты.

## Context

Главный экран — это точка входа в систему. 6 режимов должны быть видны сразу, но с чёткой визуальной иерархией: что важно сейчас (due flashcards), что можно продолжить (resume card), что доступно всегда. Hover states и micro-interactions делают интерфейс живым.

## Acceptance Criteria

### AC1: Grid Layout
**GIVEN** главный экран открывается  
**WHEN** режимы отображаются  
**THEN** система показывает режимы в стабильном scan-friendly layout

### AC2: Hover State
**GIVEN** Learner наводит курсор на режим  
**WHEN** hover активируется  
**THEN** система показывает понятный Hover_State/Focus_State

### AC3: Micro-Interaction
**GIVEN** Hover_State активен  
**WHEN** анимация воспроизводится  
**THEN** включается restrained Micro_Interaction без layout shift

### AC4: Due Badge Highlight
**GIVEN** Due_Badge > 0  
**WHEN** Flashcards режим отображается  
**THEN** система выделяет режим через accent color

### AC5: Resume Card Priority
**GIVEN** есть незавершённая сессия  
**WHEN** главный экран загружается  
**THEN** система показывает Resume_Card как первый элемент

### AC6: Mode Selection Transition
**GIVEN** Learner выбирает режим  
**WHEN** клик происходит  
**THEN** система применяет transition-анимацию 300ms

### AC7: Visual Hierarchy
**GIVEN** все режимы отображаются  
**WHEN** learner сканирует экран  
**THEN** Visual_Hierarchy использует размер, цвет и позицию для приоритизации режимов

## Definition of Done

- [ ] Stable scan-friendly layout works
- [ ] Hover/focus states and restrained micro-interactions work
- [ ] Due badge highlighting works
- [ ] Resume card priority logic works
- [ ] Transition animation (300ms) works
- [ ] CSS enhancements applied
- [ ] Unit tests для home hub components
- [ ] Visual hierarchy clarity validated
