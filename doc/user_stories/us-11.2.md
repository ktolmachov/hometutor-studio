### US-11.2 - Retrieval confidence is explained honestly (P1)

**As a** learner or reviewer checking an answer,
**I want** the UI and API to call the signal `retrieval_confidence` and explain what it means,
**so that** I do not mistake retrieval coverage for probability of truth.

**Acceptance:**
- Given an answer contains a retrieval quality signal,
- When I inspect the answer, API payload, or docs,
- Then the primary name is `retrieval_confidence`, not generic `confidence`.
- And the explanation says it is an explainability signal about source coverage/retrieval quality, not a truth guarantee.
- And backward-compatible API behavior is documented and tested while old clients migrate.
- And public docs avoid absolute quality claims unless they cite a concrete eval run.

**Related:** `wave-quality-defense-governance-trust`, `epoch-qbi-terminology-limitations-sync`.
