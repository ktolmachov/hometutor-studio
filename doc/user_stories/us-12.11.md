### US-12.11 - Local-first data governance and deletion (P2)

**As a** privacy-conscious local user or maintainer,
**I want** documented, confirmation-gated deletion of local user data and clear cloud-inference boundaries,
**so that** local-first claims remain operationally true and reviewable.

**Acceptance:**
- Given I request complete data deletion,
- When the deletion flow runs with explicit confirmation,
- Then local index data, user-state stores, logs, history, feedback, and cost logs are removed or reported as failed.
- And a verification mode checks for residual configured storage targets.
- And documentation explains local-first storage, cloud LLM context transmission, and known limits without overstating production readiness.
- And new persistent user-data locations must update this deletion coverage.

**Related packages:** `epoch-qbi-data-deletion-governance`, `epoch-qbi-terminology-limitations-sync`.
