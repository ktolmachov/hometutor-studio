### US-16.7 — Добавить домашнее задание к теме/курсу и сгенерировать пошаговую инструкцию (LLM) (P1)

**As a** Learner или автор курса,  
**I want** зафиксировать формулировку ДЗ в scope темы/курса и получить воспроизводимую пошаговую инструкцию (плейбук) рядом с курсом — без смены вкладок, с шагами в формате «что сделать» и критерием самопроверки,  
**so that** я не теряю поток между табами и остаюсь в Course Mode с опорой на материалы папки.

**Acceptance Criteria:**

- Given активный курс/тема в терминах Epic 16,
- When пользователь создаёт ≥1 домашнее задание, привязанное к курсу и/или теме,
- Then можно запросить разбор по шагам: структура шагов, на каждом — действие и критерий готовности (без готового финального ответа по умолчанию или с явным режимом «только подсказки»).

- And **вся доставка v1 — только Streamlit** (основной UI приложения): создание ДЗ, плейбук, генерация шагов, сохранение прогресса проверяются в Streamlit. **Telegram-бот, CLI и отдельные HTTP-клиенты в scope v1 не входят** (возможный follow-up — отдельная история).

- And плейбук отображается **inline** с контекстом курса/темы (панель / stack на узких экранах), без обязательного перехода на другую вкладку только для чтения ДЗ и шагов.

- And доступен режим **«кратко»** для продвинутых (меньше промежуточных подсказок).

- And прогресс по шагам **сохраняется между сессиями** (resume).

- And где уместно, инструкция использует retrieval по материалам курса (источники/цитаты в духе инварианта продукта).

**Out of scope (v1 пакета):** микро-чекпоинты, streak, «серия ДЗ» (идея 03); **также вне v1:** сценарии Telegram/CLI (не part of acceptance для этого пакета).
