### US-12.6 — Playwright E2E metrics для browser-level качества (P1)
**As a** maintainer, меняющий E11 UI или flow,
**I want** browser-level Playwright tests с traces/artifacts,
**so that** я вижу, что guided CTA, Progress, expert controls и 5-minute loop работают как пользовательский сценарий, а не только как unit/API tests.

**Acceptance:**
- Given PR меняет E11 UI/flow,
- When CI запускает Playwright smoke,
- Then `npx playwright test` проходит на offline/fixture state без live LLM.
- And CI сохраняет HTML report, trace/video/screenshot artifacts для провалов.
- And nightly/live job может запускать 5-minute loop с `OPENAI_API_KEY`, но не блокирует обычный PR без ключей.
- And MCP Playwright / Chrome DevTools MCP используются только локально как "глаза агента"; CI-gate остаётся обычным Playwright test suite.
