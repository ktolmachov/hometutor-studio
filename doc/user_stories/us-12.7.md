### US-12.7 - Defense-grade reproducible eval baseline (P1)

**As a** project defender or maintainer,
**I want** one reproducible eval flow with a defense dataset, retrieval-mode metrics, and baseline artifacts,
**so that** quality claims are backed by run-id evidence instead of assertions.

**Acceptance:**
- Given the defense eval dataset exists,
- When I run the eval flow,
- Then the result includes a run id, timestamp, config snapshot, git commit when available, and serialized JSON.
- And the dataset covers qa, keyword, overview, synthesis, negative, and injection categories.
- And retrieval comparison can report recall@k, MRR, hit rate, and latency across configured modes.
- And baseline comparison produces actionable deltas without replacing existing AQE/router gates.

**Related packages:** `epoch-qbi-golden-eval-dataset`, `epoch-qbi-retrieval-mode-comparison`, `epoch-qbi-baseline-regression-gate`.
