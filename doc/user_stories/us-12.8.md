### US-12.8 - Stage-level cost and latency transparency (P1)

**As a** maintainer diagnosing slow or expensive answers,
**I want** stage-by-stage latency, token, cost, and budget-violation evidence,
**so that** I can explain whether a regression came from classification, retrieval, rerank, generation, or judging.

**Acceptance:**
- Given a query has completed,
- When I inspect traces or run the cost/latency report,
- Then each measured pipeline stage has duration, budget status, and cost/token metadata where available.
- And budget violations are surfaced through a report or metrics endpoint with stage name and overage.
- And configuration for budgets flows through project settings helpers, not direct env reads.
- And this complements, rather than duplicates, the existing p95 latency CI gate.
