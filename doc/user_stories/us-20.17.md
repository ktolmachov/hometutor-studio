### US-20.17 - Tiered Explanation Gate

**As a** learner reading «why now» on the Smart Study Router card,  
**I want** rich explanations when needed and instant template text when sufficient,  
**so that** the card stays fast without losing trust.

**Acceptance Criteria:**

- Given simple local evidence (at most two ledger signals and no contrastive block required), When the SSR card renders, Then the explanation uses the template path without an LLM call and profiles as `template_only`.
- Given complex evidence (contrastive «why not others», learning-debt label with steering conflict, or three or more ledger signals), When the SSR card renders, Then LLM enrichment may run with existing cache, timeout, and template fallback on failure.
- Given blended simple and complex fixtures on the L2 eval set, When latency is measured, Then explanation p95 is below 2 seconds and helpful-rate is not below the L2 baseline.
- Given the tier gate chooses template or LLM, When trace or feedback metadata is recorded, Then the tier decision is visible and does not imply LLM was used when it was not.
- Given the tier gate runs, When recommendations are computed, Then `hint_kind` and `primary_nav` match the rule-only baseline (explanation-only scope).
