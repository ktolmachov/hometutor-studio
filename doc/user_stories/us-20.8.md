### US-20.8 - Local Route Confidence Ledger

**As a** learner,  
**I want** to inspect the local evidence behind the router recommendation,  
**so that** I understand which learning signals shaped the next action.

**Acceptance Criteria:**

- Given a recommendation is shown, a compact evidence view lists local signals such as due cards, failed quiz, answer-ready state, retrieval confidence, or skip memory.
- Given the recommendation uses no cloud-only scoring, the evidence view does not imply external profiling.
- Given a signal is missing, the UI labels it as unavailable instead of inventing confidence.
