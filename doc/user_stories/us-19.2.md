# US-19.2 — Seamless Tutor Handoff (MoT #3 Transition to Tutor)

**Epic**: Epic 19: UX Breakthrough Wave  
**Priority**: P0  
**CJM Stage**: #3 Transition to Tutor  
**Status**: closed  
**Covered by**: epoch-us19-2-tutor-handoff-ux

## User Story

**Как** Learner,  
**Я хочу** плавно переходить от ответа к тьютору с сохранением контекста,  
**Чтобы** не терять нить обучения.

## Context

Переход от Q&A к обучению — критический момент. Большинство RAG-систем заканчиваются на ответе. hometutor начинается с него. Кнопка «Учить эту тему» должна передавать полный контекст: вопрос, источники, найденные концепты, confidence score.

## Acceptance Criteria

### AC1: Context Payload Transfer
**GIVEN** Learner нажимает "Учить эту тему 5 минут"  
**WHEN** происходит переход в tutor-режим  
**THEN** система передаёт полный Context_Payload

### AC2: Complete Context
**GIVEN** Context_Payload создаётся  
**WHEN** данные сериализуются  
**THEN** payload включает исходный вопрос, тему, источники и Confidence_Score

### AC3: Visual Continuity
**GIVEN** переход между режимами  
**WHEN** анимация воспроизводится  
**THEN** система сохраняет Visual_Continuity через transition-анимацию 300-500ms

### AC4: Context Summary
**GIVEN** переход завершён  
**WHEN** tutor-режим открывается  
**THEN** UI показывает краткое резюме переданного контекста

### AC5: Connection Highlight
**GIVEN** tutor-сессия начинается  
**WHEN** первый шаг отображается  
**THEN** система подсвечивает связь между исходным ответом и текущим tutor-шагом

### AC6: Incomplete Context Handling
**GIVEN** Context_Payload неполный  
**WHEN** валидация обнаруживает пробелы  
**THEN** система запрашивает недостающие данные у Learner

## Definition of Done

- [ ] ContextObject serialization/deserialization работает
- [ ] Transition animation 300-500ms
- [ ] Context summary отображается
- [ ] Unit tests для handoff logic
- [ ] Integration test для complete flow
- [ ] Incomplete context handling протестирован
