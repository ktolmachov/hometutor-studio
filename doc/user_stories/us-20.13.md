### US-20.13 - Local Route Simulator (what-if preview)

**As a** learner with an active Smart Study Router recommendation,  
**I want** to preview what the router would suggest if I prioritized another visible study mode right now,  
**so that** I understand tradeoffs before overriding the primary next step.

**Acceptance Criteria:**

- Given the SSR card shows a primary action and at least one visible secondary alternative, When the learner opens a what-if preview for that alternative, Then the UI shows a counterfactual primary action and a short reason derived from local deterministic state without cloud-only scoring.
- Given local state is insufficient for an honest comparison, When preview is requested, Then the UI states the limitation plainly instead of inventing confidence.
- Given preview is open, Then tutor, quiz, flashcards, and dashboard remain reachable as secondary or persistent entry points.
- Given preview or an offline maintainer fixture run executes, Then learner DB and runtime primary recommendation do not change without an explicit separate user action.
- Given a documented fixture catalog exists, When a maintainer runs the offline simulator entrypoint, Then the output includes learning-state label, chosen route, reason trace, and key local signals for regression review.
