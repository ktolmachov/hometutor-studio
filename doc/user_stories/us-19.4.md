# US-19.4 — Flashcard Session Analytics (MoT #12 Flashcard Review)

**Epic**: Epic 19: UX Breakthrough Wave  
**Priority**: P1  
**CJM Stage**: #12 Flashcard Review  
**Status**: open  
**Covered by**: ux-foundation-parsers-contracts + ux-mastery-celebration-analytics (planned)

## User Story

**Как** Learner,  
**Я хочу** видеть детальную статистику после сессии повторения,  
**Чтобы** понимать свой прогресс и планировать следующие шаги.

## Context

После завершения flashcard review сессии learner должен видеть не просто "сессия завершена", а детальную аналитику: распределение оценок, скорость обучения, прогноз удержания знаний, расписание следующих повторений. Это превращает механическое повторение в осознанный процесс.

## Acceptance Criteria

### AC1: Session Analytics Overlay
**GIVEN** Learner завершает flashcard review сессию  
**WHEN** последняя карточка оценена  
**THEN** система показывает Session_Analytics overlay

### AC2: Grades Distribution
**GIVEN** Session_Analytics отображается  
**WHEN** статистика рассчитывается  
**THEN** overlay включает распределение оценок: Again/Hard/Good/Easy (количество и %)

### AC3: Learning Velocity
**GIVEN** сессия завершена  
**WHEN** метрики вычисляются  
**THEN** система отображает Learning_Velocity (карточек/минуту)

### AC4: Retention Prediction
**GIVEN** карточки повторены  
**WHEN** SM-2 intervals рассчитываются  
**THEN** система показывает Retention_Prediction для повторённых карточек

### AC5: Review Schedule Timeline
**GIVEN** Session_Analytics активна  
**WHEN** расписание строится  
**THEN** система визуализирует следующее расписание повторений (timeline на 7 дней)

### AC6: Difficult Topics Suggestion
**GIVEN** сессия содержит ≥3 оценок "Again"  
**WHEN** анализ завершён  
**THEN** система предлагает Primary_CTA "Разобрать сложные темы"

### AC7: History Persistence
**GIVEN** Session_Analytics создана  
**WHEN** overlay закрывается  
**THEN** статистика сохраняется в истории и доступна через Progress tab

## Definition of Done

- [ ] Analytics overlay renders with all metrics
- [ ] Grades distribution chart works
- [ ] Retention timeline (7 days) displays
- [ ] Difficult topics suggestion (≥3 "Again") triggers
- [ ] Insufficient data handling (<5 cards) works
- [ ] Stats persist through existing user_state APIs
- [ ] Unit tests для analytics overlay
- [ ] Property tests для metrics calculations
