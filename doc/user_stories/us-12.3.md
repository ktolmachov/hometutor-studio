### US-12.3 — Интеграционные тесты валидируют реальный retrieval (P1)
**As a** разработчик, меняющий логику retrieval,
**I want** интеграционные тесты, которые строят реальный индекс и проверяют качество ответов,
**so that** я ловлю регрессии retrieval до того, как они дойдут до пользователя.

**Acceptance:**
- Given markdown fixture-файлы по 5+ темам,
- When я запускаю `pytest -m integration tests/test_integration_retrieval.py` (или `pytest -m integration` по всему дереву; в дефолтном `pytest` без `-m integration` они не выполняются — см. `pytest.ini`),
- Then 5–10 тестов проверяют релевантность источников, полноту ответов и file-фильтрацию,
- And тесты завершаются за < 3 мин.
