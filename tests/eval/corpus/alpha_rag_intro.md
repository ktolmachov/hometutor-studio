# Введение в RAG

**RAG_ALPHA_UNIQUE_7741**

Retrieval-Augmented Generation (RAG) — это подход к работе с LLM, при котором модель получает доступ к внешней базе знаний.

## Зачем нужен RAG

LLM имеют ограниченную память и могут "галлюцинировать" — выдумывать факты. RAG решает это:
1. **Извлечение** — поиск релевантных фрагментов из базы документов
2. **Подмешивание контекста** — найденные фрагменты добавляются к запросу
3. **Генерация** — LLM отвечает с опорой на реальные источники

## Базовый пайплайн

```
Пользовательский вопрос → Извлечение контекста → Генерация ответа с источниками
```

Ключевое преимущество: ответы привязаны к документам, легко проверить достоверность.
